<objective>
## Goal
Render fog of war on the game board: visible hexes normal, fogged hexes (previously seen) dimmed with no units, shrouded hexes (never seen) dark. Switch human player to fog-of-war filtered state.

## Purpose
Visual fog of war completes the player-facing experience — enemies disappear when out of sight, creating tactical uncertainty and information asymmetry.

## Output
- Human player sees fog-of-war filtered state
- Hex overlays: visible (normal), fogged (dimmed terrain, no units), shrouded (dark)
- Client-side seen_hexes tracking persists across turns
</objective>

  <name>Task 1: Switch to fog-of-war state and track seen hexes</name>
  <files>norrust_love/main.lua, norrust_love/state.lua</files>
  <action>
    In state.lua:
    - Add `fog = { seen = {}, visible = {}, enabled = true }` to the state variables
    - seen is a table keyed by "col,row" → true (persists across turns)
    - visible is a table keyed by "col,row" → true (rebuilt each frame from snapshot)
    - enabled flag for debug toggle

    In main.lua love.draw (around line 656):
    - When in PLAYING mode and fog.enabled:
      - Call `norrust.get_state_fow(vars.engine, 0)` instead of `norrust.get_state(vars.engine)`
      - After getting state, rebuild fog.visible from state.visible_hexes:
        ```lua
        fog.visible = {}
        if state.visible_hexes then
            for _, vh in ipairs(state.visible_hexes) do
                local key = vh.col .. "," .. vh.row
                fog.visible[key] = true
                fog.seen[key] = true  -- union into persistent set
            end
        end
        ```
    - When not in PLAYING mode or fog not enabled: use norrust.get_state() as before
    - Pass fog table in the draw context (ctx.fog)

    In main.lua scenario load / scenario reset:
    - Clear fog.seen = {} when a new scenario loads (on "scenario_loaded" event or equivalent)

    Do NOT change AI state fetching — AI uses norrust.get_state() or its own planning.
  </action>
  <verify>luajit -bl norrust_love/main.lua /dev/null 2>&1 && luajit -bl norrust_love/state.lua /dev/null 2>&1</verify>

<task type="auto">
  <name>Task 2: Render fog and shroud overlays on board</name>
  <files>norrust_love/draw_board.lua</files>
  <action>
    In draw_board.lua, after terrain hexes are drawn (around line 166) and before units:

    Add fog overlay pass:
    ```lua
    -- Fog of war overlays
    if ctx.fog and ctx.fog.enabled then
        for row = 0, ROWS - 1 do
            for col = 0, COLS - 1 do
                local key = col .. "," .. row
                local cx, cy = hex.to_pixel(col, row)
                if not ctx.fog.visible[key] then
                    if ctx.fog.seen[key] then
                        -- Fogged: previously seen, currently not visible
                        love.graphics.setColor(0, 0, 0, 0.5)
                    else
                        -- Shrouded: never seen
                        love.graphics.setColor(0, 0, 0, 0.8)
                    end
                    love.graphics.polygon("fill", hex.polygon(cx, cy, HEX_RADIUS))
                    love.graphics.setColor(1, 1, 1, 1)
                end
            end
        end
    end
    ```

    The unit filtering is already handled by the snapshot (invisible enemies not in state.units).
    No changes needed to unit drawing — it just draws what's in the filtered state.

    Use the same HEX_RADIUS and hex.polygon already used for terrain rendering.
  </action>
  <verify>luajit -bl norrust_love/draw_board.lua /dev/null 2>&1</verify>
  <done>AC-2, AC-3, AC-4 satisfied: shroud, fog, and visible hexes render correctly</done>
