<objective>
## Goal
Enforce attack adjacency, implement defender retaliation, and add win/loss detection so a
complete match can be played to a conclusion.

## Purpose
Phase 3 left combat one-directional and infinite (no win condition). This plan closes the
loop: attacks are range-limited, retaliation makes combat symmetrically dangerous, and a
win screen appears when one faction's last unit dies. After this plan a full match is
playable from spawn to victory.

## Output
- `game_state.rs`: `ActionError::NotAdjacent`; adjacency check in Attack; retaliation block
- `gdext_node.rs`: `-7` code for NotAdjacent; new `get_winner()` bridge method
- `game.gd`: `_game_over` flag; `_check_game_over()` call after attack; win message in `_draw()`
</objective>

<acceptance_criteria>

## AC-1: Adjacent-Only Attack Enforcement
```gherkin
Given two units are placed more than 1 hex apart
When apply_attack() is called between them
Then it returns -7 (NotAdjacent) and no damage is dealt to either unit
```

## AC-2: Defender Retaliation
```gherkin
Given faction 0's fighter attacks faction 1's fighter at an adjacent hex
When the attacker's strikes resolve and the defender survives
Then the defender immediately retaliates: its attacks resolve against the attacker's
  terrain defense, applying damage to the attacker (and removing the attacker if HP reaches 0)
  And the defender's `attacked` flag is set to true after retaliation
```

## AC-3: Win/Loss Detection and Display
```gherkin
Given an attack kills the last unit of a faction
When the GDScript calls get_winner() after the attack
Then it returns the winning faction ID (0 or 1)
  And "Faction N wins!" is displayed centred on screen in large yellow text
  And all further input (clicks, key presses) is ignored
```

    2. In `apply_action()`, inside the `Action::Attack` arm, make these changes in order:

       a. **After the attacker validation block**, fetch attacker position:
          ```rust
          let attacker_pos = state.positions[&attacker_id];
          ```

       b. **After cloning the attack definition** (where `defender_pos` is fetched), add
          the adjacency check immediately after `defender_pos` is obtained:
          ```rust

              return Err(ActionError::NotAdjacent);
          }
          ```

       c. **After the attacker-strikes / defender-damage block** (including the dead-defender
          purge), add the retaliation block:
          ```rust
          // Retaliation: defender strikes back if still alive and has attacks
          if state.units.contains_key(&defender_id) {
              let ret_attack = state.units.get(&defender_id)
                  .and_then(|d| d.attacks.first())
                  .cloned();

              if let Some(def_attack) = ret_attack {
                  // Terrain defense for the attacker (defender is striking them)
                  let ret_defense = {
                      let a = state.units.get(&attacker_id).unwrap();
                      let terrain_id = state.board.terrain_at(attacker_pos).unwrap_or("");
                      a.defense.get(terrain_id).copied().unwrap_or(a.default_defense)
                  };
                  // ToD modifier from defender's alignment
                  let ret_tod = {
                      let d = state.units.get(&defender_id).unwrap();
                      tod_damage_modifier(d.alignment, time_of_day(state.turn))
                  };
                  let ret_damage = resolve_attack(
                      &mut state.rng,
                      def_attack.damage,
                      def_attack.strikes,
                      ret_defense,
                      ret_tod,
                  );
                  // Mark defender as having attacked (retaliation counts)
                  state.units.get_mut(&defender_id).unwrap().attacked = true;
                  // Apply retaliation damage to attacker
                  if state.units.contains_key(&attacker_id) {
                      let attacker_hp = {
                          let a = state.units.get_mut(&attacker_id).unwrap();
                          a.hp = a.hp.saturating_sub(ret_damage);
                          a.hp
                      };
                      if attacker_hp == 0 {
                          state.units.remove(&attacker_id);
                          state.positions.remove(&attacker_id);
                      }
                  }
              }
          }
          ```

       Avoid: modifying the EndTurn or Move arms. Do not add a new `ActionError` variant
       anywhere other than the enum definition.

  <action>
    **In `norrust_core/src/gdext_node.rs`:**

    1. Add `get_winner()` #[func] after `get_reachable_hexes()`:
       ```rust
       /// Returns the winning faction (0 or 1) when one faction has no units left.
       /// Returns -1 if the game is ongoing or no game exists.
       #[func]
       fn get_winner(&self) -> i32 {
           let Some(state) = self.game.as_ref() else { return -1 };
           let has_0 = state.units.values().any(|u| u.faction == 0);
           let has_1 = state.units.values().any(|u| u.faction == 1);
           match (has_0, has_1) {
               (true, false) => 0,
               (false, true) => 1,
               _ => -1,
           }
       }
       ```

    **In `norrust_client/scripts/game.gd`:**

