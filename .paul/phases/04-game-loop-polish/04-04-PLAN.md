<objective>
## Goal
Add a village terrain type (healing = 8), a `get_terrain_at()` bridge method so GDScript
can query terrain per-hex, expand the board to 8×5, and place two village hexes in the
centre of the map.

## Purpose
The current 5×5 board is too small for meaningful strategy and all terrain is determined
by a hardcoded checkerboard in GDScript — Rust is not the source of truth for rendering.
A larger board with contested villages gives players a strategic objective (capture the
healing hexes), and using `get_terrain_at()` makes the renderer Rust-driven.

## Output
- `data/terrain/village.toml`: id=village, healing=8, defense=40, movement_cost=1
- `gdext_node.rs`: `get_terrain_at(col, row)` #[func] — queries board terrain, returns ""
  for unset hexes
- `game.gd`:
  - `BOARD_COLS = 8`, `BOARD_ROWS = 5`
  - `COLOR_VILLAGE = Color(0.72, 0.60, 0.25)` (gold-tan)
  - terrain setup uses `set_terrain_at()` for every hex; village placed at (3,1) and (4,3)
  - `_draw()` queries `get_terrain_at()` per hex for color
  - units repositioned: blue at (1,2), red at (6,2)
</objective>

    Add a new `#[func]` after `set_terrain_at()`:
    ```rust
    /// Returns the terrain id at offset (col, row), or "" if none is set.
    #[func]
    fn get_terrain_at(&self, col: i32, row: i32) -> GString {
        let Some(state) = self.game.as_ref() else { return "".into() };
        state.board.terrain_at(Hex::from_offset(col, row)).unwrap_or("").into()
    }
    ```

    Avoid: changing any other methods. The village healing will work automatically
    because the existing healing infrastructure reads from board.healing_map, which


    2. Add village color constant after COLOR_FOREST:
       ```gdscript
       const COLOR_VILLAGE   = Color(0.72, 0.60, 0.25)  # gold-tan
       ```

    3. Replace the terrain setup loop in `_setup_rust_core()`:
       ```gdscript
       # Grassland/forest checkerboard base
       for col in range(BOARD_COLS):
           for row in range(BOARD_ROWS):
               var terrain = "forest" if (col + row) % 2 == 1 else "grassland"
               _core.set_terrain_at(col, row, terrain)
       # Village hexes — contested healing positions in the centre
       _core.set_terrain_at(3, 1, "village")
       _core.set_terrain_at(4, 3, "village")
       ```

    4. Update unit spawn positions (further apart on the wider board):
       ```gdscript
       _core.place_unit_at(1, "fighter", 30, 0, 1, 2)  # faction 0 (blue)
       _core.place_unit_at(2, "fighter", 30, 1, 6, 2)  # faction 1 (red)
       ```

    5. In `_draw()`, replace the hardcoded terrain color formula with a Rust query:
       ```gdscript
       # 1. Terrain hexes
       for col in range(BOARD_COLS):
           for row in range(BOARD_ROWS):
               var center = _tile_map.map_to_local(Vector2i(col, row)) + _tile_map.position
               var terrain_id = _core.get_terrain_at(col, row)
               var color: Color
               if terrain_id == "village":
                   color = COLOR_VILLAGE
               elif terrain_id == "forest":
                   color = COLOR_FOREST
               else:
                   color = COLOR_GRASSLAND
               draw_polygon(_hex_polygon(center, HEX_RADIUS), [color])
       ```

    Avoid: changing `_setup_tilemap()`, `_center_camera()`, `_draw_units()`,
    `_build_unit_pos_map()`, `_input()`, or any other function.
    The tile_set.tile_size and HEX_RADIUS constants remain unchanged — the existing
    HEX_CELL_W/H values are correct for any board size.
