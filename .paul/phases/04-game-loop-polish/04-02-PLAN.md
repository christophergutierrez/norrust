<objective>
## Goal
Expose unit exhaustion state to GDScript (dim circles for units that have moved or attacked),
and implement per-turn terrain healing (units recover HP at the start of their faction's turn
based on the terrain they occupy).

## Purpose
After 04-01 a complete match is playable but there is no visual feedback for which units have
already acted, making it hard to track game state. Per-turn healing adds a recovery mechanic
that rewards good positioning and makes matches longer/more strategic.

## Output
- `schema.rs`: `healing: u32` field on `TerrainDef` (serde default 0)
- `board.rs`: `healing_map: HashMap<String, u32>` + `set_healing()` + `healing_for()` methods
- `game_state.rs`: Healing applied in `EndTurn` to newly-active faction units, capped at `max_hp`
- `gdext_node.rs`: `get_unit_data()` extended to 7-tuple (adds `moved` + `attacked` as 0/1);
  `set_terrain_at()` also populates board healing map from registry
- `data/terrain/grassland.toml`: `healing = 2`
- `data/terrain/forest.toml`: `healing = 0`
- `game.gd`: stride updated 5→7; exhausted units drawn at alpha 0.4
</objective>

<acceptance_criteria>

## AC-1: Exhausted Unit Visual Indicator
```gherkin
Given a unit that has moved or attacked this turn
When _draw() renders the board
Then that unit's circle is drawn at alpha 0.4 (dimmed)
  And units that have not yet acted are drawn at full opacity (alpha 1.0)
  And after pressing E (end turn) all circles return to full opacity
```

## AC-2: Per-Turn Terrain Healing
```gherkin
Given a unit standing on grassland (healing = 2) with HP below max_hp
When the active faction's turn begins (EndTurn is called, faction flips to them)
Then that unit's HP increases by 2, capped at max_hp
  And the updated HP is visible in the unit circle on the next draw
```

## AC-3: Healing Does Not Exceed Max HP
```gherkin
Given a unit at full HP standing on healing terrain
When their turn begins
Then their HP remains at max_hp (no overheal)
```

</acceptance_criteria>

    **In `norrust_core/src/board.rs`:**

    2. Add `healing_map: HashMap<String, u32>` to the `Board` struct (after `terrain` field).

    3. In `Board::new()`, initialize it: `healing_map: HashMap::new()`.

    4. Add two new methods to `impl Board`:
       ```rust
       /// Store the healing value for a terrain type.
       pub fn set_healing(&mut self, terrain_id: impl Into<String>, healing: u32) {
           self.healing_map.insert(terrain_id.into(), healing);
       }

       /// Return healing per turn for a terrain id (0 if unknown).
       pub fn healing_for(&self, terrain_id: &str) -> u32 {
           self.healing_map.get(terrain_id).copied().unwrap_or(0)
       }
       ```

    **In `norrust_core/src/game_state.rs`:**

    5. In `Action::EndTurn` arm, **after** the flag-reset loop and before `Ok(())`, add healing:
       ```rust
       // Heal units of the newly-active faction based on their terrain.
       let active = state.active_faction;
       let to_heal: Vec<u32> = state.units.iter()
           .filter(|(_, u)| u.faction == active)
           .map(|(id, _)| *id)
           .collect();
       for uid in to_heal {
           if let Some(&hex) = state.positions.get(&uid) {
               let healing = state.board
                   .terrain_at(hex)
                   .map(|t| state.board.healing_for(t))
                   .unwrap_or(0);
               if healing > 0 {
                   let unit = state.units.get_mut(&uid).unwrap();
                   unit.hp = (unit.hp + healing).min(unit.max_hp);
               }
           }
       }
       ```

    **In `norrust_core/src/gdext_node.rs`:**

    6. In `set_terrain_at()`, after the existing `state.board.set_terrain(...)` call, add:
       ```rust
       if let Some(terrain) = self.terrain.as_ref()
           .and_then(|r| r.get(&terrain_id.to_string())) {
           state.board.set_healing(terrain_id.to_string(), terrain.healing);
       }
       ```

    7. In `get_unit_data()`, extend the per-unit push block from 5 fields to 7:
