<objective>
## Goal
Implement 6 weapon specials (drain, poison, charge, backstab, slow, first strike) and status effect tracking (poisoned, slowed) in the Rust combat system. Specials read from AttackDef.specials and affect combat resolution and per-turn state.

## Purpose
Adds tactical depth to combat — currently all attacks are functionally identical except damage/strikes/type. Specials make unit composition and positioning matter.

## Output
- Unit struct gains `poisoned: bool` and `slowed: bool` status fields
- AttackDef.specials drives combat behavior changes
- UnitSnapshot exposes status effects for UI
- EndTurn resolves poison damage and clears slowed; village cures poison
- Tests for each special
</objective>

<acceptance_criteria>

## AC-1: Drain heals attacker for damage dealt
```gherkin
Given an attacker with a drain weapon special
When the attack hits for N damage
Then the attacker heals N HP (capped at max_hp)
```

## AC-2: Poison applies DoT and is cured at villages
```gherkin
Given an attacker with a poison weapon special
When the attack hits (damage > 0)
Then the defender becomes poisoned (poisoned=true)
And on EndTurn, poisoned units take 8 damage
And poisoned units standing on a village (healing > 0) are cured instead
```

## AC-3: Charge doubles damage both ways
```gherkin
Given an attacker with a charge weapon special on melee
When combat resolves
Then attacker damage is doubled AND defender retaliation damage to this attacker is doubled
```

## AC-4: Backstab doubles damage when enemy flanked
```gherkin
Given an attacker with backstab weapon special
When the defender has an enemy on the opposite hex (flanked)
Then attacker damage is doubled
```

## AC-5: Slow halves movement and damage next turn
```gherkin
Given an attacker with a slow weapon special
When the attack hits (damage > 0)
Then the defender becomes slowed (slowed=true)
And slowed units have movement halved on their next turn
And slowed units deal halved damage on their next turn
And slowed is cleared at the start of the slowed unit's faction turn (EndTurn)
```

## AC-6: First strike — attacker goes first, defender dies before retaliating
```gherkin
Given an attacker with a first_strike weapon special
When the attacker kills the defender
Then no retaliation occurs (defender is dead before retaliating)
Note: In normal combat, attacker strikes then defender retaliates. First strike doesn't change this — it just means the attacker's strikes are resolved first. This is already the default behavior in our engine, so first_strike is a no-op marker for now but ensures future compatibility if we add simultaneous strike resolution.
```

## AC-7: Status effects in snapshot and cleared properly
```gherkin
Given units with poisoned/slowed status
When get_state_json is called
Then UnitSnapshot includes poisoned and slowed boolean fields
And status effects persist across save/load (via snapshot)
```

<task type="auto">
  <name>Task 2: Drain, charge, backstab, slow, poison in combat</name>
  <files>norrust_core/src/game_state.rs, norrust_core/src/combat.rs</files>
  <action>
    In the Action::Attack handler in game_state.rs, after selecting the attack and before/after resolve_attack:

    **Charge:** If attacker's attack has "charge" special AND range is "melee":
    - Double effective_damage before resolve_attack
    - Later, double ret_effective_damage for defender's retaliation

    **Backstab:** If attacker's attack has "backstab" special:
    - Check if defender_pos has an enemy (same faction as attacker, not the attacker itself) on the opposite hex
    - Opposite hex = defender_pos + (defender_pos - attacker_pos) in cube coords
    - If flanked, double effective_damage before resolve_attack

    **Drain:** After resolve_attack for attacker:
    - If attacker's attack has "drain" special AND damage > 0:
    - Heal attacker by damage dealt (capped at max_hp)

    **Poison:** After applying attacker damage:
    - If attacker's attack has "poison" special AND damage > 0 AND defender survived:
    - Set defender.poisoned = true

    **Slow:** After applying attacker damage:
    - If attacker's attack has "slow" special AND damage > 0 AND defender survived:
    - Set defender.slowed = true

    **Slow damage effect:** When computing effective_damage for any attacker:
    - If attacker.slowed, halve effective_damage (integer division)

    Also apply poison/slow/drain for defender's retaliation attack (same logic, reversed roles).

    **In EndTurn handler:**
    - After village healing, before faction flip:
    - For each unit of the ENDING faction:
      - If poisoned AND on a village (healing > 0): cure (poisoned = false)
      - Else if poisoned: take 8 damage; if hp reaches 0, remove unit
    - For each unit of the NEWLY ACTIVE faction (after flip):
      - If slowed: clear slowed flag (slowed wears off at start of your turn)

    **Slow movement effect:** In pathfinding or reachable_hexes:
    - If unit.slowed, halve movement budget

    **First strike:** No-op for now (our engine already resolves attacker first). Add a comment noting this.

    **In simulate_combat (combat.rs):**
    - Add charge/backstab/drain/slow damage modifiers to the Monte Carlo simulation
    - For charge: double attacker damage and double defender retaliation damage
    - For backstab: pass a `flanked: bool` parameter, double attacker damage if true
    - For slow: halve damage if attacker is slowed
    - For drain: track net damage (attacker heals) in kill probability calculation
    - Poison/slow don't affect immediate combat damage, just apply status
  </action>
  <verify>cargo test in norrust_core — all existing tests pass + new tests</verify>
  <done>AC-1 through AC-6 satisfied</done>
</task>
