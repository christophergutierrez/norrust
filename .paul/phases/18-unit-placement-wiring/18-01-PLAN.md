## Goal
Define a TOML unit placement file format, add `scenario::load_units()` and the `load_units()` GDExtension bridge, then wire both `load_board()` and `load_units()` into game.gd — replacing all hardcoded startup calls with two file-driven calls.

Completes the scenario system. After this phase, starting a game means loading two files: a board file and a units file. The hardcoded `create_game()`, `generate_map()`, and 10× `place_unit_at()` calls in game.gd are gone. The first real hand-authored scenario (`contested.toml` + `contested_units.toml`) is the default game.

    ```rust
    use crate::schema::{BoardDef, UnitsDef, UnitPlacement};

    /// Load unit placements from a TOML units file.
    ///
    /// The file must contain one or more `[[units]]` entries, each with
    /// `id`, `unit_type`, `faction`, `col`, `row` fields.
    /// Returns Err if the file cannot be read or parsed.
    pub fn load_units(path: &Path) -> Result<Vec<UnitPlacement>, String> {
        let text = std::fs::read_to_string(path)
            .map_err(|e| format!("load_units: cannot read {:?}: {}", path, e))?;
        let def: UnitsDef = toml::from_str(&text)
            .map_err(|e| format!("load_units: parse error in {:?}: {}", path, e))?;
        Ok(def.units)
    }
    ```
    Update the `use crate::schema::BoardDef;` import at the top of scenario.rs to also import `UnitsDef` and `UnitPlacement`.

    **1. Add `load_units()` to gdext_node.rs** (after `load_board()`):
    ```rust
    /// Load unit placements from a TOML units file and place all units on the board.
    ///
    /// The file must contain `[[units]]` entries with `id`, `unit_type`, `faction`,
    /// `col`, `row`. Calls `place_unit_at()` for each entry — stats (hp, movement,
    /// attacks etc.) are copied from the UnitDef registry exactly as the typed bridge
    /// method does.
    /// Returns true on success, false if no game exists or the file cannot be read.
    #[func]
    fn load_units(&mut self, units_path: GString) -> bool {
        if self.game.is_none() {
            godot_error!("load_units: no game exists — call load_board() first");
            return false;
        }
        let path = std::path::PathBuf::from(units_path.to_string());
        let placements = match crate::scenario::load_units(&path) {
            Ok(p) => p,
            Err(e) => {
                godot_error!("{}", e);
                return false;
            }
        };
        for p in placements {
            self.place_unit_at(
                p.id as i32,
                p.unit_type.as_str().into(),
                0,       // hp ignored — place_unit_at always sets max_hp from registry
                p.faction as i32,
                p.col,
                p.row,
            );
        }
        true
    }
    ```
    Note: `self.place_unit_at(...)` is a direct Rust method call — no GDScript overhead. `hp = 0` is passed because `place_unit_at()` always overrides it with `max_hp` from the UnitDef registry.
