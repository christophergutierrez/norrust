<objective>
## Goal
Remove the `get_unit_data()` and `get_unit_positions()` flat-array bridge methods from the
Rust/GDScript boundary and replace GDScript's unit rendering and input handling with JSON
parsing of `get_state_json()`. Add named stride constants for `get_reachable_hexes()`.

## Purpose
Eliminates two architectural defects identified post-v0.1:
- **Magic indices**: GDScript accessed unit fields by hardcoded integer offsets (`data[i+4]`
  for HP) — a silent failure mode on any future field addition.
- **Dual extraction**: `PackedInt32Array` (7-tuple) and `StateSnapshot` JSON described the
  same units in different field sets and different order. `StateSnapshot` is now the sole
  source of truth.

## Output
- `gdext_node.rs`: `get_unit_positions()` and `get_unit_data()` removed.
- `game.gd`:
  - 3 named constants: `RH_STRIDE`, `RH_COL`, `RH_ROW` for `get_reachable_hexes()` pairs
  - `_parse_state() -> Dictionary` helper (calls `get_state_json()`, parses JSON once)
  - `_draw_units(state)` takes a Dictionary instead of calling `get_unit_data()`
  - `_build_unit_pos_map(state)` takes a Dictionary instead of calling `get_unit_data()`
  - `_draw()` calls `_parse_state()` once, passes result to helpers
  - `_input()` calls `_parse_state()` once, passes result to `_build_unit_pos_map()`
  - Reachable hexes loop uses `RH_COL`, `RH_ROW`, `RH_STRIDE` constants
</objective>

    ```gdscript
            var faction  = unit["faction"]

            draw_circle(center, HEX_RADIUS * 0.45, color)
            draw_string(
                ThemeDB.fallback_font,
                center + Vector2(-8, 5),
                str(hp),
                HORIZONTAL_ALIGNMENT_LEFT,
                -1, 13, Color.WHITE
            )
    ```

    ```gdscript
    func _build_unit_pos_map(state: Dictionary) -> Dictionary:
        var result: Dictionary = {}
        for unit in state.get("units", []):
            result[Vector2i(unit["col"], unit["row"])] = [unit["id"], unit["faction"]]
        return result
    ```
