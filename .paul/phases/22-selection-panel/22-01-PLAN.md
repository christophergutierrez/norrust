<objective>
Goal: Clicking any unit on the board opens a stat panel in the right sidebar showing
      all relevant unit information — type name, faction, HP, XP, movement, attacks,
      and abilities.

Purpose: Players currently have no way to identify units or understand their stats
         during play without memorizing every unit type. The panel makes the game
         usable without prior knowledge.

Output:
  - UnitSnapshot JSON extended with movement, attacks, abilities
  - _inspect_unit_id state in game.gd; any unit click sets it
  - _draw_unit_panel() renders a full stat panel in the right sidebar
  - Panel visible for both friendly and enemy units; hidden during recruit mode
</objective>

<context>
@norrust_core/src/snapshot.rs    — UnitSnapshot struct to extend
@norrust_client/scripts/game.gd  — click handling + draw pipeline
@norrust_core/src/unit.rs        — Unit struct (has movement, attacks, abilities)
</context>

<task id="1" name="Extend UnitSnapshot with movement, attacks, abilities">
  Files: norrust_core/src/snapshot.rs
  Action:
    1. Add AttackSnapshot struct above UnitSnapshot:
         #[derive(Debug, Serialize)]
         pub struct AttackSnapshot {
             pub id:      String,
             pub name:    String,
             pub damage:  u32,
             pub strikes: u32,
             pub range:   String,
         }
    2. Add to UnitSnapshot: movement: u32, attacks: Vec<AttackSnapshot>, abilities: Vec<String>
    3. In from_game_state(), populate new fields from unit.movement, unit.attacks, unit.abilities
       attacks: unit.attacks.iter().map(|a| AttackSnapshot {
           id: a.id.clone(), name: a.name.clone(),
           damage: a.damage, strikes: a.strikes, range: a.range.clone(),
       }).collect()
    4. Add/update test: assert snapshot includes movement, attacks, abilities fields in JSON
    5. touch src/snapshot.rs && cargo test (in norrust_core/)
    6. cargo build && cp target/debug/libnorrust_core.so ../norrust_client/bin/
  Verify: cargo test passes; JSON output contains "movement", "attacks", "abilities"
  Done: AC-1, AC-7
</task>

<task id="2" name="Add inspection state and panel to game.gd">
  Files: norrust_client/scripts/game.gd
  Action:
    1. Add var _inspect_unit_id: int = -1 at top with other vars

    2. Update click handler (left-mouse-button section of _input()):
       - When clicking a hex with ANY unit (friendly or enemy):
           _inspect_unit_id = that_unit_id
       - When clicking an empty hex:
           _inspect_unit_id = -1
       - When clicking a friendly unit (existing select behavior): unchanged —
           _selected_unit_id set, reachable hexes computed; ALSO set _inspect_unit_id
       - When attack occurs: ALSO set _inspect_unit_id = enemy_id after attack

    3. In _clear_selection(): add _inspect_unit_id = -1

    4. Add _draw_unit_panel(unit: Dictionary) function:
       Draws right-sidebar panel (same 200px strip as other panels) showing:
         - Header: unit["def_id"] in faction color (Blue/Red)
         - "HP: {hp} / {max_hp}"
         - "XP: {xp} / {xp_needed}" (only if xp_needed > 0)
         - "Move: {movement}" + "(exhausted)" if moved and attacked,
           "(moved)" if moved only, "(attacked)" if attacked only
         - "── Attacks ──"
         - For each attack: "{name}: {damage}×{strikes} {range}"
         - "── Abilities ──" + each ability (if abilities non-empty)

    5. In _draw(), after existing HUD block and before/after _draw_recruit_panel:
       if not _recruit_mode and _inspect_unit_id != -1:
           find the unit dict in state["units"] by id
           if found: _draw_unit_panel(unit)

  Verify: clicking friendly unit shows panel with correct stats;
          clicking enemy shows panel; clicking empty hex hides panel;
          recruit mode hides panel (recruit panel appears instead)
  Done: AC-2, AC-3, AC-4, AC-5, AC-6
</task>
