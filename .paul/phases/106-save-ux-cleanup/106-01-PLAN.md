<objective>
## Goal
Wire the new `norrust_save_json()` and `norrust_load_json()` FFI calls into Lua, replacing the manual multi-call save/load flow. Delete dead TOML serialization code. Keep backward compatibility for reading legacy TOML saves.

## Purpose
Phase 105 built the Rust save/load foundation. This phase connects it to the game UI. After this, save.write_save uses a single FFI call instead of manually building a data table, and save.load_save uses a single FFI call instead of ~14 separate restore calls. Dead code (TOML serializer, custom TOML parser for saves) is removed.

## Output
- norrust.lua: FFI declarations + wrappers for save_json, load_json, set_display_name
- save.lua: simplified write_save (single FFI call) + simplified load_save (single FFI call)
- input.lua: updated restore_from_save to work with new JSON-native save format
- Dead TOML code removed from save.lua
- F5 save / F9 load / save list all work with new format
</objective>

    Rewrite save.write_save to use the new single-call FFI:

    1. Before saving, call `norrust.set_display_name(engine, display_name)` to set the display name on the engine (Lua knows the scenario display name from the board metadata).

    2. Replace the entire data-building block with:
       ```lua
       local json_str = norrust.save_json(engine)
       if not json_str or json_str == "" then return nil end
       ```

    3. Keep the filename generation logic (date-first naming) as-is.

    4. Write json_str directly to disk (already JSON, no need for json_encode).

    5. The function signature changes to:
       `function save.write_save(engine, norrust, scenario_board, display_name)`
       - Remove `scenarios_path` param (engine already knows board_path)
       - Remove `campaign_ctx` param (engine already knows campaign state)
       - Add `display_name` param (for scenario name in save metadata)

    6. Delete dead TOML serialization code: toml_value(), toml_array(), save.serialize_toml().

    7. Update all callers (input.lua: F5 handler, auto-save, exit-save):
       - Remove build_save_campaign_ctx() — no longer needed
       - Change write_save calls from:
         `save.write_save(engine, norrust, scn.board, scn.path, build_save_campaign_ctx())`
         to:
         `save.write_save(engine, norrust, scn.board, scn.display_name)`
       - Also update main.lua auto-save call similarly

    Avoid: Breaking the save filename format (date-first naming must stay the same).
    Avoid: Removing parse_save_toml — still needed for legacy TOML load.
  </action>

  <action>
    Rewrite save.load_save for JSON saves to use the single FFI call:

    1. For .json saves:
       - Read file contents
       - Call `norrust.load_json(engine, json_str)` — this restores everything in the engine

       - Return the parsed data table for restore_from_save()

    2. For .toml saves (legacy):
       - Keep the existing multi-call restore path exactly as-is
       - parse_save_toml + manual FFI calls

    3. The load_save signature stays the same:
       `function save.load_save(engine, norrust, filepath, center_camera_fn)`
       - center_camera_fn is unused by the engine restore but may be used by callers

    4. Update restore_from_save() in input.lua:
       - For JSON saves loaded via norrust_load_json(), the engine is already fully restored

       - The JSON save format from Rust uses different field names than the old Lua format:
         * board_path (top-level, not game.board_path)
         * turn (top-level, not game.turn)
         * campaign (full CampaignState, not campaign.campaign_file)
       - Update field references accordingly
       - Campaign restore: the Rust save includes campaign state with campaign_def, scenario_index, etc.
         Read these to update Lua campaign tables

    5. Remove build_save_campaign_ctx() from input.lua (no longer needed after Task 2).

    Avoid: Breaking legacy TOML load path.
    Avoid: Changing list_saves metadata extraction (it already handles both formats).
