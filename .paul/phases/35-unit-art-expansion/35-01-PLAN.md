  <name>Task 1: Expand generate_sprites.lua with generic drawing system + 14 unit definitions</name>
  <files>norrust_love/generate_sprites.lua</files>
  <action>
    **1a. Refactor to generic humanoid drawing system:**

    Keep the existing drawing primitives (draw_ellipse, draw_rect, draw_line) unchanged.

    Create a `draw_humanoid(imageData, gx, gy, config, pose)` function that replaces `draw_spearman`:
    - `config` table contains: colors (skin, tunic, tunic_lt, hair, boots), weapon drawing function, body_scale (for height variation: 1.0 = Spearman size)
    - `pose` table (same as current): body_lean, arm_dx, arm_dy, weapon_dx, weapon_dy, leg_spread, crouch, alpha
    - Core body drawing: legs, boots, torso, head, hair, arms — parameterized by config colors and body_scale
    - Weapon drawn via `config.draw_weapon(imageData, hand_x, hand_y, pose, config.weapon_colors)` callback

    **Weapon drawing functions** (one per weapon type):
    - `weapon_spear(imageData, hx, hy, pose, colors)` — shaft + pointed tip (existing Spearman weapon)
    - `weapon_sword(imageData, hx, hy, pose, colors)` — blade + crossguard
    - `weapon_bow(imageData, hx, hy, pose, colors)` — curved bow + string
    - `weapon_staff(imageData, hx, hy, pose, colors)` — long shaft + glowing orb tip
    - `weapon_mace(imageData, hx, hy, pose, colors)` — shaft + heavy head
    - `weapon_dagger(imageData, hx, hy, pose, colors)` — short blade
    - `weapon_crossbow(imageData, hx, hy, pose, colors)` — stock + limbs

    Each weapon function uses the pose's weapon_dx/weapon_dy to position the weapon relative to the hand.

    **1b. Create draw_portrait_generic(imageData, config):**
    - Parameterized version of generate_portrait using config colors
    - Config includes optional weapon_in_portrait callback (shows weapon peeking from side)

    **1c. Create animation frame factories per weapon style:**
    - Melee swing frames (sword/mace/dagger): wind up → slash → recover (6 frames)
    - Melee thrust frames (spear): existing spear thrust pattern (6 frames)
    - Ranged draw frames (bow): nock → draw → release (4 frames)
    - Ranged throw frames (javelin/knife): raise → throw (4 frames)
    - Ranged cast frames (staff/missile): charge → release (4 frames)
    - Ranged crossbow frames: aim → fire (4 frames)
    - Idle: same subtle sway (4 frames) — reuse for all
    - Defend: same brace pattern (3 frames) — reuse for all
    - Death: same stagger → fall (4 frames) — reuse for all

    **1d. Define all 15 units in UNITS table:**

    Each entry: `{ def_id, config, anims }` where config sets colors/weapon/scale.

    **Faction color palettes:**
    - Loyalists: warm browns/reds/blues (steel armor, colored tunics)
    - Elves: greens/silvers/golds (lighter build, nature tones)
    - Orcs: dark greens/browns/grays (bulky, crude weapons)

    **Unit definitions** (each with unique tunic color + weapon):
    | def_id | Weapon | Melee Anim | Ranged Anim | Scale | Palette |
    |--------|--------|------------|-------------|-------|---------|
    | Spearman | spear | thrust | throw | 1.0 | brown tunic, tan skin |
    | Lieutenant | sword | swing | crossbow | 1.05 | blue tunic, cape effect |
    | Bowman | bow (melee: sword) | swing | draw | 0.95 | green tunic |
    | Cavalryman | sword | swing | none | 1.1 | red tunic, taller |
    | Mage | staff | thrust | cast | 0.95 | purple robe |
    | Heavy Infantryman | mace | swing | none | 1.15 | gray armor |
    | Sergeant | sword | swing | crossbow | 1.0 | dark blue tunic |
    | Elvish Captain | sword | swing | draw | 1.0 | silver/green |
    | Elvish Fighter | sword | swing | draw | 0.95 | light green |
    | Elvish Archer | bow (melee: sword) | swing | draw | 0.9 | forest green |
    | Elvish Scout | sword | swing | draw | 0.9 | tan/green |
    | Elvish Shaman | staff | thrust | cast | 0.9 | white/gold |
    | Orcish Warrior | greatsword (large) | swing | none | 1.2 | dark brown |
    | Orcish Grunt | sword | swing | none | 1.1 | olive/brown |
    | Orcish Archer | bow (melee: dagger) | swing | draw | 1.0 | dark green |
    | Orcish Assassin | dagger | swing | throw | 0.95 | black/dark gray |

    For units with both melee and ranged:
    - `attack-melee` uses the melee weapon animation
    - `attack-ranged` uses the ranged weapon animation
    - The config stores both weapon draw functions; ranged_weapon used for attack-ranged frames

    For units with only melee:
    - No `attack-ranged` entry in anims table
    - No [attacks.ranged] in sprite.toml

    **1e. Update generate_spritesheet to use draw_humanoid:**
    - Instead of `draw_spearman(imageData, ox, anchor_y, pose)`, call `draw_humanoid(imageData, ox, anchor_y, unit.config, pose)`
    - Portrait generation uses `draw_portrait_generic(imageData, unit.config)`

    **1f. Keep Spearman drawing backward-compatible:**
    - The refactored Spearman config produces identical drawing output to the current draw_spearman
    - Same colors, same weapon (spear), same pose parameters

    Avoid:
    - Do NOT modify animation.lua, assets.lua, toml_parser.lua, viewer.lua, or main.lua
    - Do NOT add external dependencies
    - Do NOT change the sprite.toml schema
    - Keep file size reasonable — weapon functions should be concise (10-20 lines each)
