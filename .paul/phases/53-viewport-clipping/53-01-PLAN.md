<objective>
## Goal
Clip board rendering at the right panel edge so hexes and units don't draw under the sidebar. Also prevent clicks in the panel area from hitting board hexes underneath.

## Purpose
Fixes bug where enemies under the right panel can't be selected for attack. The board area and panel area become visually and functionally separate regions.

## Output
- draw.lua: scissor clip on board-space drawing to vp_w - 200
- main.lua: click-to-hex ignores clicks in the panel region
</objective>

<task type="auto">
  <name>Task 1: Add scissor clip around board-space drawing</name>
  <files>norrust_love/draw.lua</files>
  <action>
    Before the board-space push (line ~531), set a scissor rectangle that covers
    the screen area left of the panel. The scissor must be in actual pixel coordinates
    (not UI_SCALE-divided), since setScissor works in window pixels:

    ```lua
    local panel_w = 200
    local sw, sh = love.graphics.getDimensions()
    local clip_w = sw - panel_w * ctx.UI_SCALE
    love.graphics.setScissor(0, 0, clip_w, sh)
    ```

    After the board-space pop (line ~682), clear the scissor:
    ```lua
    love.graphics.setScissor()
    ```

    This ensures all board-space content (terrain, units, highlights, paths, animations)
    is clipped at the panel edge, while screen-space UI (HUD, panels, overlays) draws
    unrestricted after the scissor is cleared.
  </action>
  <verify>Launch game, pan board so hexes would be under panel — they clip at the edge</verify>
  <done>AC-1 satisfied: board content clipped at panel edge</done>
</task>

<task type="auto">
  <name>Task 2: Guard click-to-hex against panel region</name>
  <files>norrust_love/main.lua</files>
  <action>
    In the click handler (love.mousepressed and love.mousereleased), before converting
    screen coords to hex, check if the click is in the panel region:

    At the top of the PLAYING section in love.mousepressed (before the click-to-hex
    conversion at ~line 1028), add:
    ```lua
    local vp_w = select(1, get_viewport())
    if x >= vp_w - 200 then return end
    ```

    This early-returns for clicks in the rightmost 200px (panel area), preventing
    them from being interpreted as board clicks. The panel's own click handling
    (inspect unit, recruit, etc.) happens in screen-space UI code which runs separately.

    Also add the same guard to the right-click terrain inspection (~line 957) and
    setup click (~line 995).
  </action>
  <verify>Click in panel area where a unit is behind it — unit is NOT selected</verify>
  <done>AC-2 satisfied: panel clicks don't hit board</done>
</task>
