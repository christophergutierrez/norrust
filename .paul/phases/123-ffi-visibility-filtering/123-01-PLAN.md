<objective>
## Goal
Add a fog-of-war-aware state query: `norrust_get_state_json_fow(engine, faction)` that returns a StateSnapshot with enemy units filtered to visible-only, plus a `visible_hexes` array. Keep the existing unfiltered `norrust_get_state_json` for AI and debug.

## Purpose
FFI boundary for fog of war — Lua gets a filtered view of game state where invisible enemies are hidden. This is the bridge between Rust visibility calculation (Phase 122) and Lua fog rendering (Phase 124).

## Output
- `StateSnapshot::from_game_state_fow()` — filtered snapshot builder
- `norrust_get_state_json_fow` FFI function
- Lua wrapper `norrust.get_state_fow(engine, faction)`
- `visible_hexes` field on StateSnapshot (as col/row pairs)
</objective>

<task type="auto">
  <name>Task 2: Add FFI function and Lua wrapper</name>
  <files>norrust_core/src/ffi.rs, norrust_love/norrust.lua</files>
  <action>
    In ffi.rs, add near norrust_get_state_json:
    ```rust
    #[no_mangle]
    pub unsafe extern "C" fn norrust_get_state_json_fow(
        engine: *mut NorRustEngine,
        faction: i32,
    ) -> *mut c_char
    ```
    - Get game state from engine
    - Call StateSnapshot::from_game_state_fow(state, faction as u8)
    - Serialize to JSON and return
    - No caching (fow queries are per-faction, caching is complex)

    In norrust.lua:
    - Add C declaration: `char* norrust_get_state_json_fow(NorRustEngine* engine, int32_t faction);`
    - Add Lua wrapper:
      ```lua
      function M.get_state_fow(engine, faction)
          local raw = get_string(lib.norrust_get_state_json_fow(engine, faction))
          if raw == "" then return {} end
          return json_decode(raw) or {}
      end
      ```
  </action>
