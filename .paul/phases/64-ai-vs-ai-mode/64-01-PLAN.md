<task type="auto">
  <name>Task 2: Love2D --ai-vs-ai mode</name>
  <files>norrust_love/main.lua</files>
  <action>
    Add --ai-vs-ai CLI flag handling in love.load:
    - Sets a local `ai_vs_ai` flag and optionally parses --ai-delay N (default 0.5)
    - Also starts agent server (same as --agent-server)

    In love.update, when ai_vs_ai is true and game_mode == PLAYING and not game_over:
    - Use a timer (ai_turn_timer) counting down with dt
    - When timer reaches 0: run ai_recruit for active faction + ai_take_turn + end_turn + check_game_over
    - Reset timer to ai_delay after each turn
    - Skip if move animation is playing (pending_anims.move ~= nil)
    - Skip if combat animation is playing (#pending_anims > 0)

    Store ai_vs_ai state in the shared table to avoid upvalue pressure:
    - shared.ai_vs_ai = false
    - shared.ai_delay = 0.5
    - shared.ai_timer = 0

    IMPORTANT: The existing E key handler does end_turn then AI for faction 1 only.
    For AI vs AI, both factions need ai_take_turn. The loop is:
    1. ai_recruit for active faction
    2. ai_take_turn for active faction
    3. check_game_over
    4. If not over, timer resets for next faction's turn

    Do NOT change the existing E key behavior or any other keybindings.
  </action>
