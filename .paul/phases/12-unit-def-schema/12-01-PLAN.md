<objective>
## Goal
Extend `UnitDef` and `AttackDef` with the fields needed for the full Wesnoth unit roster —
`race`, `cost`, `usage`, `abilities`, `alignment` on UnitDef; `specials` on AttackDef — all with
`#[serde(default)]` so the 4 existing TOMLs load without changes. Wire `alignment` from the
UnitDef registry into the runtime `Unit` struct at spawn and on advancement.

## Purpose
Phase 13 will generate 218 unit TOMLs. The schema must exist first. The `alignment` field is the
only new field Rust computes with (ToD modifier); all others are stored-for-future-use. Moving
alignment from a hardcoded default (Liminal) to data-driven means the AI and player units will
correctly benefit/suffer from Day/Night even before Phase 13.

## Output
- `norrust_core/src/schema.rs` — expanded UnitDef + AttackDef structs
- `norrust_core/src/unit.rs` — `parse_alignment()` helper; `advance_unit()` copies alignment
- `norrust_core/src/gdext_node.rs` — `place_unit_at()` copies alignment from registry
- `norrust_core/tests/simulation.rs` — test UnitDef/AttackDef constructions updated
- `data/units/{fighter,archer,hero,ranger}.toml` — alignment field added
</objective>

    /// Parse a UnitDef alignment string into the runtime Alignment enum.
    /// "lawful" → Lawful, "chaotic" → Chaotic, anything else → Liminal.
    /// "neutral" and "" both map to Liminal (same ToD modifier — no bonus/penalty).
