<objective>
## Goal
Implement trigger zones (enemies spawn when the player enters specific hexes) and create a second standalone scenario — "The Ambush" — a forest-heavy 12×8 board with hidden enemy reinforcements.

## Purpose
Adds genuine gameplay variety to v1.3 Campaign Mode. The crossing scenario is an open push across a large map; the ambush scenario is a tactical forest crawl with surprise enemies. Trigger zones introduce a new game mechanic that makes scenarios feel distinct.

## Output
- Rust core: `TriggerZone` system — schema, game state integration, trigger check in apply_action::Move, FFI functions
- New scenario: `scenarios/ambush.toml` (12×8 board) + `scenarios/ambush_units.toml` (preset units + trigger definitions)
- Love2D client: ambush added to scenario list, `get_next_unit_id()` for recruitment after trigger-spawned units
- Integration tests proving trigger zones work headlessly
</objective>

<acceptance_criteria>

## AC-1: Trigger zone spawns enemies on player move
```gherkin
Given a GameState with a trigger zone at hex (4,3) containing 2 pending spawns
When a faction 0 unit moves to hex (4,3)
Then the trigger fires: 2 enemy units appear at their designated hexes, and the trigger is marked as triggered
```

## AC-2: Trigger fires only once
```gherkin
Given a trigger zone that has already been triggered
When another unit moves to the same trigger hex
Then no additional units spawn (triggered flag prevents re-fire)
```

## AC-3: Trigger skips occupied spawn hexes
```gherkin
Given a trigger zone with a spawn at hex (5,3) which is already occupied
When the trigger fires
Then the spawn at (5,3) is silently skipped, and other valid spawns still place
```

## AC-4: Trigger only fires for designated faction
```gherkin
Given a trigger zone with trigger_faction=0
When a faction 1 (AI) unit moves to the trigger hex
Then the trigger does NOT fire
```

## AC-5: Ambush scenario loads and plays
```gherkin
Given scenarios/ambush.toml (12×8) and ambush_units.toml exist
When the scenario is loaded via the engine
Then the board renders correctly, trigger zones fire when player enters, AI defends objective
```

## AC-6: Love2D uses get_next_unit_id for recruitment
```gherkin
Given trigger spawn units have pre-assigned IDs at load time
When the player recruits a unit after scenario load
Then the recruited unit's ID comes from norrust_get_next_unit_id(), avoiding ID conflicts
```

  <action>
    **1a. Schema — trigger definitions (schema.rs):**
    - Add `TriggerSpawnDef` struct: `{ unit_type: String, faction: u8, col: i32, row: i32 }` — Deserialize
    - Add `TriggerDef` struct: `{ trigger_col: i32, trigger_row: i32, #[serde(default)] trigger_faction: u8, spawns: Vec<TriggerSpawnDef> }` — trigger_faction defaults to 0 (player)
    - Update `UnitsDef` to include `#[serde(default)] pub triggers: Vec<TriggerDef>` — backward compatible with existing units files that have no `[[triggers]]`

    **1b. Game state — trigger zone runtime (game_state.rs):**
    - Add `PendingSpawn` struct: `{ pub unit: Unit, pub destination: Hex }` — pre-built Unit with assigned ID
    - Add `TriggerZone` struct: `{ pub trigger_hex: Hex, pub trigger_faction: u8, pub spawns: Vec<PendingSpawn>, pub triggered: bool }`
    - Add `pub trigger_zones: Vec<TriggerZone>` to GameState — initialized as empty vec in `new()`
    - Add `pub next_unit_id: u32` to GameState — initialized to 1 in `new()`
    - In `apply_action::Move` branch, after position update + moved flag: add trigger zone check:
      1. Collect all spawns from untriggered zones where `trigger_hex == destination` AND `trigger_faction == state.active_faction` (the faction of the moving unit)
      2. Mark those zones as triggered
      3. For each spawn: if board contains destination AND hex is unoccupied, call `place_unit()`
      4. Use a two-phase approach to avoid borrow issues: first drain spawns into a local Vec, then place units

    **1c. Scenario — load_triggers function (scenario.rs):**
    - Add `pub fn load_triggers(path: &Path) -> Result<Vec<TriggerDef>, String>` that parses the same units TOML file and extracts the `triggers` field (via UnitsDef)
    - This is separate from `load_units()` to avoid changing its return type

    **1d. FFI — trigger loading + next_unit_id (ffi.rs):**
    - Update `norrust_load_units()`: after placing all units, call `load_triggers()` on the same path. For each TriggerDef, create PendingSpawn entries using `unit_from_registry()` with auto-assigned IDs from `state.next_unit_id` (incrementing). Store the TriggerZones in `state.trigger_zones`.
    - Before processing triggers: set `state.next_unit_id` to `max(all placed unit IDs) + 1`
    - Add `norrust_get_next_unit_id(engine) -> i32` FFI function — returns `state.next_unit_id as i32`

    **1e. New scenario files:**
    - `scenarios/ambush.toml`: 12×8 board
      - Dense forest theme with winding paths through clearings
      - Blue keep at (1,4), castle cluster cols 0-2
      - Red keep at (10,4), castle cluster cols 9-11
      - Objective: (10,4), max_turns = 25
      - Heavy forest/hills interior with villages along the path for healing
    - `scenarios/ambush_units.toml`:
      - Blue: Lieutenant at (1,4) — player leader
      - Red: Lieutenant at (10,4) + 2 visible defenders near objective
      - 3 trigger zones in forest hexes along the path:
        - Trigger at ~(4,4): spawns 2 enemies (Spearman + Bowman) nearby
        - Trigger at ~(6,2): spawns 1 enemy (Heavy Infantryman) nearby
        - Trigger at ~(7,5): spawns 2 enemies (Spearman + Bowman) nearby

    **1f. Integration tests (simulation.rs):**
    - `test_trigger_zone_spawns_units`: Set up GameState with one trigger zone at (3,0). Move faction 0 unit to (3,0). Assert: trigger marked triggered, spawned unit exists at spawn hex.
    - `test_trigger_fires_only_once`: Same setup, move a second unit to the trigger hex. Assert: no additional spawns (triggered = true).
    - `test_trigger_skips_occupied_hex`: Place a unit on the spawn hex before triggering. Assert: spawn silently skipped, no panic.
    - `test_trigger_faction_filter`: Set trigger_faction=0. Move a faction 1 unit to trigger hex. Assert: trigger NOT fired.
