<objective>
## Goal
Add --debug launch flag that switches Love2D to use debug/data/ instead of data/, and add cheat keys (X/G/T) that only work in debug mode.

## Purpose
Enables rapid gameplay testing without playing full campaigns. Combined with Phase 112's generator, this completes the debug sandbox workflow: edit config → generate → launch debug → test with cheat keys.

## Output
- --debug flag in Love2D arg parsing switches data_path and asset path
- Cheat keys: X = max XP on selected unit, G = add 1000 gold, T = cycle ToD
- Cheat keys gated behind debug flag — zero effect in production
</objective>

<task type="auto">
  <name>Task 2: Add cheat keys X/G/T in debug mode</name>
  <files>norrust_love/input.lua</files>
  <action>
    **In handle_global_keys function (after existing global keys):**

    Add debug-only cheat keys gated behind `shared.debug_mode`:
    ```lua
    -- Debug cheat keys (only active with --debug flag)
    if shared.debug_mode and vars.game_mode == MODES.PLAYING then
        if key == "x" and sel.unit_id >= 0 then
            -- Max XP on selected unit → triggers advancement_pending
            mods.norrust.cheat_max_xp(vars.engine, sel.unit_id)
            return true
        elseif key == "g" then
            -- Add 1000 gold to active faction
            mods.norrust.cheat_add_gold(vars.engine, 1000)
            return true
        elseif key == "t" then
            -- Advance turn by 1 (cycles time of day)
            mods.norrust.cheat_advance_turn(vars.engine)
            return true
        end
    end
    ```

    **FFI functions needed (norrust.lua wrapper + ffi.rs):**

    These are simple state mutations. Check if existing FFI can do them:
    - XP: Need a new FFI `norrust_cheat_max_xp(engine, unit_id)` that sets unit.xp = unit.xp_needed
    - Gold: Need `norrust_cheat_add_gold(engine, amount)` that adds to active faction gold
    - Turn: Need `norrust_cheat_advance_turn(engine)` that increments state.turn

    **Wait — check norrust_core constraints.** The plan says zero norrust_core changes.

    Alternative: Use existing FFI where possible:
    - Gold: No existing FFI for adding gold arbitrarily. BUT apply_action with EndTurn would advance turn + give income.
    - XP: No existing FFI to set XP directly.
    - Turn: No existing FFI to increment turn.

    **Decision needed: We need minimal FFI additions for cheat functionality.**
    The constraint was "zero production code changes" but cheat FFI functions are additions (not modifications). They don't affect existing behavior.

    Add 3 small FFI functions to ffi.rs + norrust.lua:
    - `norrust_cheat_set_xp(engine, unit_id, xp)` — set unit XP
    - `norrust_cheat_add_gold(engine, faction, amount)` — add gold
    - `norrust_cheat_set_turn(engine, turn)` — set turn number

    These are pure additions — no existing code modified.
  </action>
