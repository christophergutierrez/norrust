<objective>
## Goal
Create `data/sounds/` directory for global SFX files. Update `sound.lua` to prefer loading audio files from `data/sounds/` with procedural fallback when files are missing. Support per-scenario music via an optional `music.ogg` in scenario directories.

## Purpose
Contributors can replace placeholder procedural effects with real audio files by dropping `.ogg`/`.wav` files into `data/sounds/`. Per-scenario music is as simple as placing `music.ogg` in a scenario directory. No code changes needed to add or replace audio.

## Output
- `data/sounds/` directory with a README explaining the naming convention
- `sound.lua` loads files from `data/sounds/<name>.ogg` (or `.wav`) with procedural fallback
- Per-scenario music path: `scenarios/<name>/music.ogg` (loaded if present)
</objective>

<task type="auto">
  <name>Task 2: Update sound.lua to load files with procedural fallback</name>
  <files>norrust_love/sound.lua</files>
  <action>
    Modify sound.load() to:
    1. For each effect name, check if `data/sounds/<name>.ogg` or `data/sounds/<name>.wav` exists
    2. If a file exists, load it via love.audio.newSource(path, "static")
    3. If no file exists, fall back to the existing procedural generation

    Keep all procedural generators (noise_burst, sine_tone, sweep) as fallbacks.

    The load function should try .ogg first, then .wav, then procedural.

    Add a helper: `local function load_or_generate(name, gen_fn)` that encapsulates this pattern.
  </action>
  <verify>luajit -bl norrust_love/sound.lua /dev/null</verify>
  <done>AC-1, AC-2, AC-4 satisfied</done>
</task>

<task type="auto">
  <name>Task 3: Wire per-scenario music loading</name>
  <files>norrust_love/main.lua</files>
  <action>
    In the scenario_loaded event handler (where dialogue is loaded), add music loading:
    1. Derive music path: same directory as board.toml, named "music.ogg"
       - e.g., board = "crossing/board.toml" → music = "scenarios/crossing/music.ogg"
    2. If music file doesn't exist at OS path, try via Love2D VFS path
    3. Call shared.sound.play_music(path) if file exists
    4. Call shared.sound.stop_music() if no music file (previous scenario's music shouldn't persist)

    Need a symlink for scenarios/ too: `norrust_love/scenarios -> ../scenarios`

    The music path needs to work with sound.play_music() which uses love.filesystem.getInfo + love.audio.newSource.
  </action>
