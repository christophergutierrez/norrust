## Goal
Define a TOML board file format and wire it into the simulation core and GDExtension bridge, replacing `create_game()` + `generate_map()` as the board setup path.

## Purpose
The scenario system starts here. A hand-authored (or LLM-authored) `scenarios/contested.toml` file defines the terrain layout. Loading it produces a fully populated `Board` with all tile properties resolved from the TerrainDef registry — the same data chain as the existing terrain system.

## Output
- `BoardDef` serde struct in `schema.rs`
- `scenario::load_board(path) -> Result<Board, String>` pure Rust function (no registry, no bridge)
- `scenarios/contested.toml` — first hand-authored 8×5 scenario board
- `test_load_board_from_file` integration test
- `load_board(path, seed)` GDExtension bridge method
- 53+ tests passing

    ```rust
    use std::path::Path;
    use crate::board::Board;
    use crate::hex::Hex;
    use crate::schema::BoardDef;

    /// Load a Board from a TOML board file.
    ///
    /// The file must have `width`, `height`, and `tiles` fields.
    /// `tiles` is a flat row-major array of terrain ID strings.
    /// Returns Err if the file cannot be read, parsed, or if
    /// `tiles.len() != width * height`.
    pub fn load_board(path: &Path) -> Result<Board, String> {
        let text = std::fs::read_to_string(path)
            .map_err(|e| format!("load_board: cannot read {:?}: {}", path, e))?;
        let def: BoardDef = toml::from_str(&text)
            .map_err(|e| format!("load_board: parse error in {:?}: {}", path, e))?;
        let expected = (def.width * def.height) as usize;
        if def.tiles.len() != expected {
            return Err(format!(
                "load_board: tiles.len()={} but width×height={}",
                def.tiles.len(), expected
            ));
        }
        let mut board = Board::new(def.width, def.height);
        for row in 0..def.height as i32 {
            for col in 0..def.width as i32 {
                let idx = (row as usize) * (def.width as usize) + (col as usize);
                let terrain_id = def.tiles[idx].clone();
                board.set_terrain(Hex::from_offset(col, row), terrain_id);
            }
        }
        Ok(board)
    }
    ```
    Use `toml` crate (already a dependency). `load_board` is pure — no registry, no bridge.

<task type="auto">
  <name>Task 2: load_board() GDExtension bridge method</name>
  <files>norrust_core/src/gdext_node.rs</files>
  <action>
    Add `load_board(board_path: GString, seed: i64) -> bool` bridge method after `generate_map()`.

    This method:
    1. Parses the file path and calls `scenario::load_board()`
    2. Creates `GameState::new_seeded(board, seed)` — replaces `create_game()` + `generate_map()` in one call
    3. Upgrades all tiles from the TerrainDef registry using the same collect-then-apply pattern as `generate_map()`
    4. Returns `true` on success, `false` on any error (missing game, bad seed, file error)

    ```rust
    /// Load board terrain from a TOML scenario file and create a new game.
    ///
    /// The board file must have `width`, `height`, and `tiles` (row-major flat array).
    /// This replaces `create_game()` + `generate_map()` for scenario-based play.
    /// seed must be > 0 (used for combat RNG).
    /// Returns true on success, false on any error.
    #[func]
    fn load_board(&mut self, board_path: GString, seed: i64) -> bool {
        if seed <= 0 {
            godot_error!("load_board: seed must be > 0");
            return false;
        }
        let path = std::path::PathBuf::from(board_path.to_string());
        let board = match crate::scenario::load_board(&path) {
            Ok(b) => b,
            Err(e) => {
                godot_error!("{}", e);
                return false;
            }
        };
        let width = board.width as i32;
        let height = board.height as i32;
        self.game = Some(GameState::new_seeded(board, seed as u64));

        // Upgrade tiles from TerrainDef registry if available
        let Some(registry) = self.terrain.as_ref() else { return true };
        let assignments: Vec<(Hex, crate::board::Tile)> = (0..width)
            .flat_map(|col| (0..height).map(move |row| (col, row)))
            .filter_map(|(col, row)| {
                let hex = Hex::from_offset(col, row);
                let state = self.game.as_ref()?;
                let terrain_id = state.board.terrain_at(hex)?.to_string();
                let def = registry.get(&terrain_id)?;
                Some((hex, crate::board::Tile::from_def(def)))
            })
            .collect();
        let Some(state) = self.game.as_mut() else { return true };
        for (hex, tile) in assignments {
            state.board.set_tile(hex, tile);
        }
        true
    }
    ```

    Note: the collect-then-apply pattern is already established in `generate_map()` — use the same approach to avoid borrow conflicts. The `self.game.as_ref()` inside the closure is valid because `registry` was extracted before the closure runs.

    Actually the borrow here is trickier — `self.terrain` and `self.game` are both on `&self`/`&mut self`. Look at how `generate_map()` resolves this: it extracts `width`/`height` from state first, then builds assignments using `state.board.terrain_at()` (which only borrows state immutably), then applies with `state.board.set_tile()`. The registry is already a separate reference. In the bridge closure, reference `self.game.as_ref()` will conflict with the outer `let Some(registry) = self.terrain.as_ref()` because both are borrowing `self` — but `generate_map()` avoids this by extracting width/height first and then using a local `state` borrow. Follow the same local binding pattern exactly as `generate_map()` does: extract `state` as a local ref within the closure-free path.

    Simplest correct approach — mirror `generate_map()` exactly, extracting state before building assignments:
    ```rust
    // After creating self.game:
    let Some(registry) = self.terrain.as_ref() else { return true };
    let Some(state) = self.game.as_ref() else { return true };
    let width = state.board.width as i32;
    let height = state.board.height as i32;
    let assignments: Vec<(Hex, crate::board::Tile)> = (0..width)
        .flat_map(|col| (0..height).map(move |row| (col, row)))
        .filter_map(|(col, row)| {
            let hex = Hex::from_offset(col, row);
            let terrain_id = state.board.terrain_at(hex)?.to_string();
            let def = registry.get(&terrain_id)?;
            Some((hex, crate::board::Tile::from_def(def)))
        })
        .collect();
    let Some(state) = self.game.as_mut() else { return true };
    for (hex, tile) in assignments {
        state.board.set_tile(hex, tile);
    }
    ```
    This drops the immutable borrow of `self.game` (via `state`) before the `self.game.as_mut()` call, matching the exact pattern in `generate_map()`.
