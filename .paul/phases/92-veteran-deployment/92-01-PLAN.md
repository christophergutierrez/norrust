<objective>
## Goal
When campaign veterans exceed available keep+castle slots, show a bench/deploy selection screen before the scenario starts. Players choose which veterans to field. Currently, overflow veterans are silently dropped.

## Purpose
In longer campaigns, players accumulate more veterans than fit on the keep+castles. They need to choose their strongest squad rather than getting arbitrary placement.

## Output
- DEPLOY_VETERANS game mode with roster display
- Toggle deploy/bench per veteran with visual feedback
- Auto-limit deployed count to available slots
- Scenario starts after player confirms deployment
</objective>

<task type="auto">
  <name>Task 1: Deploy screen state and slot counting</name>
  <files>norrust_love/state.lua, norrust_love/main.lua, norrust_love/campaign_client.lua</files>
  <action>
    **state.lua:**
    - Add `deploy` table to campaign context: `deploy = {active = false, veterans = {}, slots = 0, selected = 1}`
    - Each veteran entry in deploy.veterans: `{def_id, hp, xp, xp_needed, advancement_pending, deployed = true/false, uuid}`

    **main.lua:**
    - Add `DEPLOY_VETERANS = 6` to MODES table
    - Pass deploy state to draw ctx: `ctx.DEPLOY_VETERANS = MODES.DEPLOY_VETERANS; ctx.deploy = campaign.deploy`
    - Add early return guard in draw for DEPLOY_VETERANS (like LOAD_SAVE)

    **campaign_client.lua:**
    - In `load_campaign_scenario()`, before calling `place_veterans()`:
      - Count available slots via `find_keep_and_castles()` (keep + unoccupied castles)
      - If `#campaign_veterans > available_slots`:
        - Populate `campaign.deploy.veterans` from living roster entries (with uuid)
        - Set first N entries as `deployed = true` (N = available slots)
        - Set `campaign.deploy.slots = available_slots`
        - Set `campaign.deploy.active = true`
        - Set `ctx.game_mode = ctx.DEPLOY_VETERANS` (new constant passed via ctx)
        - Return early (don't place veterans or start playing yet)
      - If veterans fit: proceed as before (auto-place all)
    - New function `campaign_client.commit_deployment(ctx)`:
      - Filter `campaign.deploy.veterans` to only `deployed == true` entries
      - Build a filtered campaign_veterans list
      - Call existing `place_veterans()` with filtered list
      - Set `campaign.deploy.active = false`
      - Set `ctx.game_mode = ctx.PLAYING`
  </action>
  <verify>luajit -bl state.lua /dev/null && luajit -bl main.lua /dev/null && luajit -bl campaign_client.lua /dev/null</verify>
  <done>AC-1, AC-2 satisfied: auto-skip when fit, deploy screen triggered on overflow</done>
</task>
