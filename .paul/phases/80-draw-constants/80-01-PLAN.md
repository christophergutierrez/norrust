## Goal
Consolidate draw.lua's magic numbers and repeated patterns into named constants and helper functions, reducing duplication and improving readability.

## Purpose
draw.lua has 120 setColor calls with hardcoded RGB tuples, the sidebar width (200) hardcoded 71 times, and repeated patterns like faction color selection (x4) and sidebar background drawing (x6). Consolidating these makes the rendering code more maintainable and easier to tune.

## Output
- `norrust_love/draw.lua` — magic numbers replaced with named constants, helper functions for repeated patterns
- `norrust_love/main.lua` — tile color cache built at scenario load instead of per-frame
</objective>

  <action>
    **1. Add constants block at top of draw.lua (after `local draw = {}`):**

    ```lua
    -- Layout constants
    local SIDEBAR_W = 200
    local SIDEBAR_PAD = 10
    local SIDEBAR_X_OFF = SIDEBAR_W - SIDEBAR_PAD  -- 190, content offset from right edge

    -- Frequently used colors (name = {r, g, b, a})
    local C_SIDEBAR_BG     = {0, 0, 0, 0.6}
    local C_SIDEBAR_BG_DK  = {0, 0, 0, 0.75}
    local C_SIDEBAR_BG_XDK = {0, 0, 0, 0.85}
    local C_GRAY           = {0.83, 0.83, 0.83}
    local C_GOLD           = {1, 0.85, 0}
    local C_WARM_TITLE     = {0.9, 0.85, 0.6}
    local C_WHITE          = {1, 1, 1}
    local C_YELLOW         = {1, 1, 0}
    ```

    Only extract colors that appear 3+ times. Leave one-off colors inline.

    **2. Add helper functions after constants:**

    ```lua
    -- Draw sidebar background panel
    local function draw_sidebar_bg(vp_w, vp_h, alpha)
        local c = alpha == 0.85 and C_SIDEBAR_BG_XDK or
                  alpha == 0.75 and C_SIDEBAR_BG_DK or C_SIDEBAR_BG
        love.graphics.setColor(c[1], c[2], c[3], c[4])
        love.graphics.rectangle("fill", vp_w - SIDEBAR_W, 0, SIDEBAR_W, vp_h)
    end

    -- Get faction color table from ctx
    local function faction_color(ctx, faction)
        return faction == 0 and ctx.BLUE or ctx.RED
    end
    ```

    **3. Replace occurrences throughout draw.lua:**
    - `vp_w - 200` → `vp_w - SIDEBAR_W`
    - `vp_w - 190` → `vp_w - SIDEBAR_X_OFF` (content area left edge)
    - `200, vp_h` in sidebar rectangles → `SIDEBAR_W, vp_h`
    - Sidebar background blocks → `draw_sidebar_bg(vp_w, vp_h, alpha)`
    - `faction == 0 and ctx.BLUE or ctx.RED` → `faction_color(ctx, faction)`
    - Repeated color tuples → named constants where used 3+ times

    **4. Cache tile colors in main.lua:**

    In main.lua, add a `tile_color_cache` table. Build it when scenario loads (after load_board/load_campaign_scenario) and pass to ctx:
    ```lua
    local tile_color_cache = {}

    local function build_tile_color_cache()
        tile_color_cache = {}
        local state = norrust.get_state(vars.engine)
        for _, tile in ipairs(state.terrain or {}) do
            local key = int(tile.col) .. "," .. int(tile.row)
            tile_color_cache[key] = parse_html_color(tile.color) or COLOR_FLAT
        end
    end
    ```

    Call `build_tile_color_cache()` at end of `call_load_scenario()` and `call_load_campaign_scenario()`.
    Add `tile_color_cache` to the draw ctx in `build_draw_ctx_state()`.

    In draw.lua's `draw_frame()`, replace the per-frame tile_colors loop (lines 682-689) with:
    ```lua
    local tile_colors = ctx.tile_color_cache or {}
    ```

    Still build tile_ids per-frame (needed for terrain asset lookup, not worth caching since it's lightweight string refs).

    **Important:**
    - Do NOT rename any ctx fields
    - Do NOT change visual output (colors, positions, sizes)
    - Do NOT extract constants for values that appear only 1-2 times
    - Do NOT change game logic
    - Keep the constants local to draw.lua (not exported)
