<task type="auto">
  <name>Task 1: Create stat verification script</name>
  <files>tools/verify_stats.py</files>
  <action>
    Create tools/verify_stats.py that:

    1. Reuses scrape_wesnoth.py's WML parsing logic (parse_movetypes, parse_units_from_file, resolve_unit)
       - Import from scrape_wesnoth or duplicate the parsing functions
    2. Builds a dict of WML-sourced units keyed by unit ID
    3. Walks data/units/ tree to find all existing TOMLs
    4. For each TOML, parses it and compares against WML source:
       - Numeric fields: max_hp, movement, level, experience, cost
       - String fields: alignment, race, usage
       - Attacks: count, damage, strikes, attack_type, range for each attack
       - Resistances: all 6 damage types
       - Movement costs: all terrain types
       - Defense: all terrain types
    5. Reports discrepancies as: file, field, expected (WML), actual (TOML)
    6. Skips comparison for fields we intentionally override:
       - abilities (we add "leader" to faction leaders)
       - advances_to (we prune to our 4-faction tree)
       - name (display name may differ)
    7. Summary: total units checked, units with discrepancies, total field mismatches

    Use Python stdlib only (toml parsing via simple regex or tomllib if Python 3.11+).
    Reuse scrape_wesnoth.py's parse functions via import (add __name__ guard if needed).

    Output format:
    ```
    DISCREPANCY: data/units/foo/bar.toml
      max_hp: expected 45 (WML), got 40 (TOML)
      attacks[0].damage: expected 8, got 9

    Summary: 132 checked, 3 with discrepancies, 5 field mismatches
    ```
  </action>
  <verify>
    python3 tools/verify_stats.py runs without errors and produces a report
  </verify>
  <done>AC-1 satisfied: verification script exists and compares all units</done>
</task>
