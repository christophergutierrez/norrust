<objective>
## Goal
Create a `SaveState` struct in Rust that captures all engine state needed for save/load, with `norrust_save_json()` and `norrust_load_json()` FFI functions enabling single-call save and restore. Lua does NOT change in this phase — we're building the Rust serialization layer only.

## Purpose
Currently save/load requires ~14 separate FFI calls from Lua (restore_unit_at, set_gold, set_turn, etc.). Moving serialization to Rust makes saves atomic, eliminates Lua-side serialization bugs, and produces clean JSON that's forward-compatible. This is the serialization layer for v3.7 — Phase 106 will wire it into Lua.

## Output
- `SaveState` and `SaveUnit` structs with Serialize/Deserialize
- `SaveState::from_engine()` captures full engine state as serializable struct
- `norrust_save_json()` FFI returns complete save data as JSON
- `norrust_load_json()` FFI restores full engine state from JSON in a single call
- Integration test proving save → modify → load → verify round-trip
</objective>

  <name>Task 1: SaveState and SaveUnit types with from_engine()</name>
  <files>norrust_core/src/save.rs, norrust_core/src/lib.rs</files>
  <action>
    Create norrust_core/src/save.rs and register it in lib.rs:

    1. `SaveUnit` struct (Serialize, Deserialize, Debug, Clone):
       - id: u32
       - def_id: String
       - faction: u8
       - col: i32, row: i32
       - hp: u32, max_hp: u32
       - xp: u32, xp_needed: u32
       - advancement_pending: bool
       - moved: bool, attacked: bool
       - poisoned: bool, slowed: bool
       - abilities: Vec<String>
       - level: u8

       Note: attacks/defense/resistances/movement_costs are NOT saved — they come from the registry on restore via unit_from_registry(). This matches the current save.lua pattern.

    2. `SaveState` struct (Serialize, Deserialize, Debug, Clone):
       - board_path: String (path to board.toml for reload)
       - rng_seed: u64 (GameState.rng state for combat reproducibility)
       - turn: u32
       - active_faction: u8
       - gold: [u32; 2]
       - max_turns: Option<u32>
       - objective_hex: Option<(i32, i32)> (col, row)
       - units: Vec<SaveUnit>
       - next_unit_id: u32
       - trigger_zones_fired: Vec<bool> (fired state per zone, indexed by position)
       - village_owners: Vec<(i32, i32, i8)> (col, row, owner — only villages with owner != -1)
       - dialogue_path: Option<String> (path to dialogue.toml for reload)
       - dialogue_fired: Vec<String> (fired dialogue entry IDs)
       - campaign: Option<CampaignState> (from Phase 104)
       - display_name: Option<String> (human-readable scenario name for save list UI)

    3. `SaveUnit::from_unit(unit: &Unit, hex: &Hex) -> Self` — captures runtime unit state.

    4. `SaveState::from_engine()` — this will be a free function or associated function that takes references to the engine's components (game_state, dialogue_state, campaign, board_path, dialogue_path, display_name). It cannot take &NorRustEngine directly since NorRustEngine is in ffi.rs.

       Signature: `SaveState::build(state: &GameState, board_path: &str, dialogue_state: Option<&DialogueState>, dialogue_path: Option<&str>, campaign: Option<&CampaignState>, display_name: Option<&str>) -> Self`

       Implementation:
       - Iterates state.positions to build SaveUnit vec from each unit
       - Captures state.rng internal state (needs pub(crate) accessor on Rng)
       - Captures trigger_zones_fired as vec of bools
       - Captures village_owners where owner >= 0
       - Captures dialogue fired IDs via fired_ids()
       - Clones campaign state if present

    5. Add `pub fn state(&self) -> u64` method to combat::Rng to expose RNG state for serialization. (Rng.state is currently private.)

    6. Unit tests:
       - test_save_unit_from_unit: create Unit, convert to SaveUnit, verify fields
       - test_save_state_serde_round_trip: create SaveState with sample data, serialize to JSON, deserialize, verify equality
       - test_save_state_build: build SaveState from a GameState with units, verify all fields captured

    Avoid: Modifying ffi.rs in this task. Keep it pure types + methods.
    Avoid: Adding Serialize/Deserialize to GameState or Unit directly — SaveState is the serialization boundary.
  </action>

  <done>AC-1 and AC-2 satisfied: SaveState captures all state and round-trips through JSON.</done>

<task type="auto">
  <name>Task 2: FFI save/load functions</name>
  <files>norrust_core/src/ffi.rs</files>
  <action>
    Add save/load state to NorRustEngine and expose via FFI:

    1. Add metadata fields to NorRustEngine for save context:
       - board_path: Option<String> (captured when load_board is called)
       - dialogue_path: Option<String> (captured when load_dialogue is called)
       - display_name: Option<String> (set via new FFI function or captured from LoadedBoard)

       Update norrust_load_board() to store board_path on engine.
       Update norrust_load_dialogue() to store dialogue_path on engine.

    2. `norrust_set_display_name(engine, name: *const c_char)` — stores display_name on engine (Lua sets this from scenario metadata).

    3. `norrust_save_json(engine) -> *mut c_char`:
       - Calls SaveState::build() with engine's game state, paths, dialogue, campaign
       - Serializes to JSON via serde_json::to_string()
       - Returns as C string (caller frees with norrust_free_string)
       - Returns null if no game state loaded

    4. `norrust_load_json(engine, json: *const c_char) -> i32`:
       - Deserializes JSON to SaveState
       - Calls load_board(board_path) to rebuild terrain + trigger zones from TOML
       - For each SaveUnit: calls unit_from_registry() to get full stats, then overrides hp/xp/moved/attacked/poisoned/slowed/advancement_pending from SaveUnit
       - Sets next_unit_id from save
       - Sets gold, turn, active_faction
       - Sets objective_hex and max_turns
       - Restores trigger_zones_fired state
       - Restores village_owners
       - If dialogue_path present: loads dialogue, marks fired IDs
       - If campaign present: restores engine.campaign
       - Stores board_path, dialogue_path, display_name on engine
       - Clears state_cache
       - Sets RNG state from rng_seed (needs Rng::with_state or similar — add to combat.rs)
       - Returns 0 on success, -1 on error

    5. Add `pub fn with_state(state: u64) -> Self` to combat::Rng (like new() but without the assert, or just use new() if state is guaranteed non-zero — save always captures non-zero state).

    6. Keep ALL existing FFI functions working. Do NOT remove or modify signatures of existing restore functions.

    Avoid: Changing any Lua code. Avoid: Breaking existing FFI function signatures.
  </action>
