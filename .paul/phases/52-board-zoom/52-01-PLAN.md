<objective>
## Goal
Add scroll wheel zoom in/out on the hex board. Zoom scales the board-space rendering around the mouse cursor position. Pan, click-to-hex, drag, and camera lerp all account for zoom level.

## Purpose
Allows the player to zoom out for strategic overview on large maps or zoom in for detail on individual units.

## Output
- main.lua: camera_zoom variable, love.wheelmoved handler, zoom-aware input transforms
- draw.lua: zoom-aware board-space scale transform
</objective>

<task type="auto">
  <name>Task 1: Add zoom state and love.wheelmoved handler</name>
  <files>norrust_love/main.lua</files>
  <action>
    1. Add `local camera_zoom = 1.0` in the Camera section (after line 107).
    2. Add zoom constants: `local ZOOM_MIN = 0.5` and `local ZOOM_MAX = 3.0` and `local ZOOM_STEP = 0.1`.
    3. Add `love.wheelmoved(x, y)` function:
       - If y > 0: zoom in (camera_zoom = math.min(camera_zoom + ZOOM_STEP, ZOOM_MAX))
       - If y < 0: zoom out (camera_zoom = math.max(camera_zoom - ZOOM_STEP, ZOOM_MIN))
       - After zoom change, call center_camera() to recompute pan bounds for new zoom level
    4. Update `center_camera()`:
       - Divide vp_w and vp_h by camera_zoom to get the effective viewport at current zoom
       - This makes pan range grow when zoomed in (more board to scroll through)
    5. Update all 3 click-to-hex conversions (right-click terrain ~line 950, setup click ~line 988, playing click ~line 1021):
       - Change `local local_x = x - (board_origin_x + camera_offset_x)` to
         `local local_x = (x - board_origin_x) / camera_zoom - camera_offset_x`
       - Same for local_y
       - This accounts for zoom in the screen-to-board coordinate conversion
    6. Update drag panning in love.mousemoved (~line 1191):
       - Divide the drag delta by camera_zoom so dragging moves the same screen distance regardless of zoom
    7. Pass camera_zoom in the ctx table for draw.lua to use.
  </action>
  <verify>Launch game, scroll wheel zooms board in/out; clicking hexes at various zoom levels selects correct hex</verify>
  <done>AC-1, AC-2, AC-3, AC-4 satisfied</done>
</task>

    This centers the board, then scales from that center, then applies pan offset in board-space.
