<objective>
## Goal
Make the board scrollable with larger hexes so the game is visually comfortable at any board size. Increase HEX_RADIUS from 64→96px, add drag-to-pan and arrow/WASD camera panning with board-edge clamping, and soft-pan the camera to follow unit selection.

## Purpose
At 64px hex radius, units are cramped and labels are tiny. Larger hexes improve readability but exceed viewport width, requiring scrolling. This unlocks future larger boards (v1.2 campaign scenario at 16×6).

## Output
Modified `norrust_client/game.gd` with larger hexes and full camera panning system. Zero Rust changes.
</objective>

<task type="auto">
  <name>Task 2: Add camera panning (drag + keyboard) with board clamping</name>
  <files>norrust_client/game.gd</files>
  <action>
    Add camera state variables:
    - var _camera_offset: Vector2 = Vector2.ZERO — accumulated pan offset
    - var _drag_active: bool = false
    - var _drag_start: Vector2 = Vector2.ZERO — mouse position at drag start
    - var _drag_camera_start: Vector2 = Vector2.ZERO — camera offset at drag start
    - const CAMERA_PAN_SPEED = 400.0 — pixels/sec for keyboard pan

    Modify _center_camera():
    - Rename to _recalc_board_origin() or keep name but store the "neutral" board center offset
    - Store _board_origin (the centered position) and _board_bounds (min/max world coords)
    - Final tilemap position = _board_origin + _camera_offset

    Add _process(delta) for continuous keyboard panning:
    - Check Input.is_action_pressed for ui_left/ui_right/ui_up/ui_down (arrow keys)
    - Also check KEY_W/A/S/D as secondary bindings via Input.is_key_pressed
    - Accumulate _camera_offset by direction × CAMERA_PAN_SPEED × delta
    - Clamp _camera_offset to board bounds
    - Update _tile_map.position = _board_origin + _camera_offset
    - Call queue_redraw() if offset changed

    Add drag-to-pan in _input():
    - On MOUSE_BUTTON_LEFT pressed: if no unit at clicked hex AND not in recruit mode, set _drag_active = true, record _drag_start and _drag_camera_start
    - On mouse motion while _drag_active: _camera_offset = _drag_camera_start + (current_mouse - _drag_start), clamp, update position, queue_redraw()
    - On MOUSE_BUTTON_LEFT released: _drag_active = false
    - Guard: do NOT start drag if click lands on a unit or sidebar area

    Board clamping:
    - Compute board pixel extent from _tile_map.map_to_local() for corners (0,0) and (BOARD_COLS-1, BOARD_ROWS-1)
    - Allow panning such that any board edge can reach viewport center (half-viewport margin)
    - Clamp _camera_offset within computed range

    Mouse-to-hex conversion:
    - Existing _tile_map.to_local(get_global_mouse_position()) should still work since _tile_map.position changes. Verify and adjust if needed.

    Do NOT change:
    - HUD drawing positions (already screen-space)
    - Sidebar drawing positions (already screen-space)
    - Any game logic or Rust bridges
  </action>
  <verify>Arrow keys and WASD pan the board; drag on empty space pans; HUD and sidebar stay fixed; board stops at edges</verify>
  <done>AC-2, AC-3, AC-4 satisfied: drag pan + keyboard pan + board clamping all work</done>
</task>

<task type="auto">
  <name>Task 3: Camera follows unit selection</name>
  <files>norrust_client/game.gd</files>
  <action>
    Add smooth-pan state:
    - var _camera_target: Vector2 = Vector2.ZERO — target offset to lerp toward
    - var _camera_lerping: bool = false
    - const CAMERA_LERP_SPEED = 8.0 — lerp factor (reaches target in ~0.3s)

    When a unit is selected (_selected_unit_id changes to a valid ID):
    - Get unit's world position from _tile_map.map_to_local(unit_cell)
    - Calculate _camera_target offset that would center that world position in viewport
    - Set _camera_lerping = true

    In _process(delta), when _camera_lerping:
    - Lerp _camera_offset toward _camera_target: _camera_offset = _camera_offset.lerp(_camera_target, CAMERA_LERP_SPEED * delta)
    - Clamp to board bounds after lerp
    - Update _tile_map.position
    - queue_redraw()
    - When distance < 1px, snap to target and set _camera_lerping = false

    Do NOT:
    - Interrupt ongoing keyboard/drag pan with lerp (keyboard/drag should cancel lerp)
    - Pan camera on inspect (only on selection for movement)
  </action>
  <verify>Selecting a unit off-screen smoothly pans to center it; keyboard pan cancels lerp</verify>
  <done>AC-5 satisfied: camera smoothly follows unit selection</done>
</task>
