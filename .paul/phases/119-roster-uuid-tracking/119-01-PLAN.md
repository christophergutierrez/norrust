<objective>
## Goal
Replace all Lua roster.lua usage with Rust engine FFI calls. After this plan, roster.lua is no longer required — the engine's CampaignState is the sole owner of unit identity tracking.

## Purpose
Unit roster tracking (UUID assignment, alive/dead status, sync from game state) is gameplay logic that belongs in the engine. Currently Lua maintains a parallel roster via roster.lua. The Rust CampaignState already has equivalent functionality — this plan wires it up via FFI and removes the Lua duplicate.

## Output
- Lua wrapper functions in norrust.lua for existing campaign FFI functions
- input_play.lua roster operations replaced with FFI calls
- input_setup.lua no longer creates Lua roster
- campaign.roster removed from Lua state
- roster.lua no longer required (kept for backward compat but unused)
</objective>
