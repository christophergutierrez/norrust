<objective>
## Goal
Move sprite assets from `norrust_love/assets/units/` into `data/units/<name>/` alongside each unit's TOML, making units with sprites self-contained in one directory. Update the Rust registry loader to find TOML files inside subdirectories. Update `assets.lua` to load sprites from the new `data/units/` location.

## Purpose
Non-programmer contributors should find everything about a unit in one place: its stats TOML, sprite images, and sprite.toml metadata. This is the first phase of content organization for contributor accessibility.

## Output
- 16 unit directories in `data/units/<name>/` each containing the unit TOML + sprite files
- 306 flat unit TOMLs remain as `data/units/<name>.toml` (no sprites)
- Rust `Registry::load_from_dir` handles both flat TOMLs and `<subdir>/<name>.toml`
- `assets.lua` loads sprites from `../data/units/` instead of `assets/units/`
- `norrust_love/assets/units/` deleted
</objective>

<task type="auto">
  <name>Task 1: Update Registry loader to scan subdirectories</name>
  <files>norrust_core/src/loader.rs</files>
  <action>
    Modify `Registry::load_from_dir` to also look inside immediate subdirectories for a single .toml file:
    - Continue scanning for `*.toml` files in the directory root (existing behavior)
    - Additionally, for each subdirectory entry, look for a `.toml` file inside it (the unit's definition)
    - A subdirectory like `data/units/bowman/` should find `data/units/bowman/bowman.toml` (name matches dir)
    - If no matching-name TOML exists, try any single `.toml` file in the subdir
    - Non-TOML files in subdirs (PNGs, sprite.toml) should be ignored by the registry loader — sprite.toml won't parse as UnitDef

    Strategy: after the existing loop over dir entries, add a second pass for directory entries. For each directory, look for `<dirname>.toml` inside it. If found, load it the same way as a flat TOML.

    Avoid: changing the Registry API or existing test assertions. The loader should be transparent — callers don't know if a unit came from a flat file or subdirectory.
  </action>
  <verify>cargo test -p norrust_core -- test_unit_registry_loads</verify>
  <done>AC-1, AC-4 satisfied: all 322 units load from mixed flat/subdir layout</done>
</task>
