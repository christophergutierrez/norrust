<objective>
## Goal
Add portrait generation pipeline to generate_sprites_v2.py. Portraits use a separate prompt (painterly style, black background, close-up face/bust), are processed to 128x128, and validated for size (<100KB). CLI gets --portrait flag for generating just the portrait, and full unit generation includes portrait automatically.

## Purpose
Units need portrait images for the UI panel. Portraits have different requirements than sprites (painterly vs pixel art, black bg vs transparent, no consistency requirement with poses), so they need their own prompt, processing, and validation.

## Output
- `generate_sprites_v2.py` updated with: `build_portrait_prompt()`, `process_portrait()`, `generate_portrait()` functions
- Portrait generated as part of `generate_unit()` flow (after all poses)
- CLI --portrait flag for standalone portrait generation
</objective>

  <action>
    Add these functions:

    1. build_portrait_prompt(unit_name):
       - Extract unit description from UNITS dict
       - Return prompt for: painterly style, close-up face/upper body portrait,
         solid black (#000000) background, no environment, dramatic lighting OK
       - Do NOT use STYLE_PROMPT (that's for sprites)
       - Include unit description for character appearance

    2. process_portrait(img_data, out_path):
       - Load image, scale to 128x128 (fit, not stretch)
       - Sample corners for bg color, replace with solid black
       - Save as PNG
       - Return (ok, file_bytes)

    3. generate_portrait(api_key, unit_name, ref_path=None, max_attempts=3):
       - Build portrait prompt
       - Loop up to max_attempts:
         - Generate via API (ref_path optional, not required for portraits)
         - Process with process_portrait
         - Check size <= 102400 bytes (100KB)
         - If valid: return True
         - If oversized: retry
       - Save to data/units/{unit_name}/portrait.png
       - Save raw to sprites_raw/{unit_name}_v2_portrait.png

    Update generate_unit() to call generate_portrait() after all poses complete.

    Add --portrait flag to CLI argparse:
    - --portrait alone with --unit: generate only portrait
    - Without --portrait: full generation includes portrait

    Avoid: modifying existing validation functions or generate_pose.
    Avoid: using STYLE_PROMPT for portraits.
