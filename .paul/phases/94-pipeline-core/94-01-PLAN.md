<objective>
## Goal
Add validation functions and retry loop to generate_sprites_v2.py. After each generated pose, validate direction (auto-flip if facing left), multi-blob detection (reject if multiple disconnected figures), size enforcement (<30KB hard, <20KB target with downscale), and edge quality (reject if sprite touches frame edge). Retry up to 3 times on failure, then flag for manual review.

## Purpose
The current pipeline generates and processes images but has no quality gate. Bad images (wrong direction, multiple figures, oversized, clipped edges) pass through silently, requiring manual inspection of every output. Automated validation with retry makes the pipeline reliable enough for batch use.

## Output
- `generate_sprites_v2.py` updated with: `validate_sprite()`, `check_direction()`, `check_multi_blob()`, `check_size()`, `check_edges()` functions
- `generate_pose()` updated with retry loop calling validate after process
- CLI unchanged (existing --unit, --redo, --base flags work with validation integrated)
</objective>

## AC-1: Direction Validation with Auto-Flip
```gherkin
Given a generated sprite processed to 256x256 RGBA
When the center-of-mass X is left of center (character facing left)
Then the image is flipped horizontally and saved (auto-corrected, not rejected)
```

## AC-2: Multi-Blob Detection
```gherkin
Given a generated sprite with transparent background
When connected component analysis finds more than 1 significant blob (>5% of total opaque pixels)
Then validation returns failure with reason "multi-blob"
```

## AC-3: Size Enforcement
```gherkin
Given a processed sprite PNG saved to disk
When file size exceeds 30KB
Then validation returns failure with reason "oversized"
And if file size is between 20KB and 30KB, a warning is logged but validation passes
```

## AC-4: Edge Quality
```gherkin
Given a processed sprite on 256x256 canvas
When any opaque pixel exists in the outermost 2-pixel border
Then validation returns failure with reason "edge-clipped"
```

## AC-5: Retry Loop
```gherkin
Given a pose that fails validation
When fewer than 3 attempts have been made
Then the pose is regenerated with the same prompt and reference
And after 3 failures, the pose is flagged "NEEDS REVIEW" and generation continues to next pose
```

<task type="auto">
  <name>Task 1: Add validation functions</name>
  <files>norrust_love/tools/generate_sprites_v2.py</files>
  <action>
    Add these functions after process_single_image():

    1. check_direction(img_path) -> (ok, flipped):
       - Load RGBA image, find center-of-mass of opaque pixels (alpha > 0)
       - If center-of-mass X < FRAME_SIZE/2, flip horizontally, save, return (True, True)
       - Otherwise return (True, False)
       - Uses PIL only (no external deps)

    2. check_multi_blob(img_path) -> (ok, blob_count):
       - Load RGBA, create binary mask from alpha > 0
       - BFS/flood-fill to find connected components (4-connectivity)
       - Count blobs with pixel count > 5% of total opaque pixels
       - Return (blob_count <= 1, blob_count)

    3. check_size(img_path, hard_limit=30720, warn_limit=20480) -> (ok, file_bytes):
       - Check os.path.getsize()
       - If > hard_limit: return (False, size)
       - If > warn_limit: print warning, return (True, size)
       - Else return (True, size)

    4. check_edges(img_path, border=2) -> (ok, edge_pixel_count):
       - Load RGBA, check outermost `border` pixels on all 4 sides
       - Count opaque pixels (alpha > 0) in border zone
       - Return (count == 0, count)

    5. validate_sprite(img_path) -> (ok, issues):
       - Run check_direction (auto-fix, always passes)
       - Run check_multi_blob, check_size, check_edges
       - Collect failures as list of strings
       - Return (len(issues) == 0, issues)
       - Print each check result as it runs

    Avoid: importing numpy or scipy — use only PIL and stdlib.
    Avoid: modifying existing functions (generate_image, process_single_image, build_prompt).
