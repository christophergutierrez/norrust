<objective>
## Goal
Split the `shared` catch-all table (8 unrelated concerns) into focused groupings with clear ownership, reducing shared to a minimal cross-module bridge.

## Purpose
The `shared` table was created as a quick way to pass state between main.lua, input.lua, and draw.lua without exceeding the LuaJIT 60-upvalue limit. Now that context tables exist (Phase 78), the unrelated concerns in `shared` can be separated for clarity. AI state only used in main.lua shouldn't be in a shared table. Recruit palette belongs with selection state.

## Output
- `norrust_love/main.lua` — shared table split into focused locals, ctx passing updated
- `norrust_love/input.lua` — references updated for new table locations
</objective>
