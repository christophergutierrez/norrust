Create a CONTRIBUTING.md at the project root with how-to guides for non-programmer contributors: adding units, creating scenarios, defining factions, and adding sounds/music.

<task type="auto">
  <name>Task 1: Create CONTRIBUTING.md</name>
  <files>CONTRIBUTING.md</files>
  <action>
    Create CONTRIBUTING.md at project root with these sections:

    1. **Overview** — Brief intro: content lives in data/ and scenarios/, no code changes needed
    2. **Directory Layout** — Tree showing data/units/, data/terrain/, data/factions/, data/sounds/, scenarios/
    3. **Adding a Unit** — Step-by-step:
       - Create data/units/<name>/<name>.toml
       - Required fields: id, name, race, level, max_hp, movement, attacks, defense, movement_costs, resistances
       - Optional: experience, advances_to, alignment, cost, abilities
       - Optional sprites: idle.png, attack-melee.png, etc. + sprite.toml
       - Reference spearman as example
    4. **Creating a Scenario** — Step-by-step:
       - Create scenarios/<name>/
       - board.toml: width, height, tiles array (row-major), optional objective_hex/max_turns
       - units.toml: [[units]] entries with id, unit_type, faction, col, row
       - Optional dialogue.toml: [[dialogue]] entries with trigger, turn, faction, text
       - Optional music.ogg for background music
       - Wiring into game: add entry to SCENARIOS table in main.lua (only code touch needed)
    5. **Adding Sounds** — Drop .ogg/.wav into data/sounds/ with one of: hit, miss, death, move, recruit, turn_end, select
    6. **Factions** — Brief: data/factions/<name>.toml with id, name, leader_def, recruits
    7. **Terrain** — Brief: data/terrain/<id>.toml with id, name, color, movement_cost, defense, healing
    8. **Testing** — How to verify: run `cd norrust_love && love .`, check game loads new content

    Keep it practical and concise. Use real examples from existing files. No fluff.
  </action>
