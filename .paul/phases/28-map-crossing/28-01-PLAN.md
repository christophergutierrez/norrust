<objective>
## Goal
Implement the "map crossing" scenario — a large board where the player must reach the enemy keep to win, with a turn limit enforcing forward pressure.

## Purpose
First step toward campaign mode (v1.3). Extends the win condition system beyond "eliminate all enemies" to support objective-based gameplay. Proves the engine can handle larger boards and richer victory conditions.

## Output
- Rust core: `objective_hex` + `max_turns` fields on GameState, extended `norrust_get_winner()` with "reach hex" + "turn limit" checks, new FFI query functions
- New scenario: `scenarios/crossing.toml` (16×10 board) + `scenarios/crossing_units.toml`
- Love2D client: dynamic board dimensions from state JSON, scenario selection, HUD turn/limit display, objective hex highlight, win/loss message variants
- Integration tests proving new win conditions headlessly
</objective>

## AC-1: Objective hex win condition
```gherkin
Given a GameState with objective_hex set to a target hex
When a unit of the active faction moves to that hex
Then norrust_get_winner() returns that faction as winner
```

## AC-2: Turn limit loss condition
```gherkin
Given a GameState with max_turns set to N
When the turn counter exceeds N without reaching the objective
Then norrust_get_winner() returns the defending faction (1) as winner
```

```gherkin
Given a GameState with no objective_hex or max_turns set
```

  <action>
    **1a. GameState fields:**
    - Add `pub objective_hex: Option<Hex>` to GameState — the hex a faction must reach to win
    - Add `pub max_turns: Option<u32>` to GameState — turn limit (None = unlimited)
    - Initialize both as None in `GameState::new()`

    **1b. Win condition logic:**
    - Refactor `norrust_get_winner()` in ffi.rs to call a new `pub fn check_winner(state: &GameState) -> Option<u8>` method on GameState (or free function in game_state.rs) so it can be tested headlessly
    - Logic order:
      1. If `objective_hex` is set: check if any unit occupies it — if faction 0 unit there, faction 0 wins; if faction 1 unit there, faction 1 wins
      2. If `max_turns` is set and `state.turn > max_turns`: defender (faction 1) wins by timeout
      3. Otherwise: existing elimination check (has_0 / has_1)
    - `norrust_get_winner()` calls `check_winner()` and returns 0/1/-1

    **1c. StateSnapshot extension:**
    - Add `pub max_turns: Option<u32>` to StateSnapshot
    - Add `pub objective_col: Option<i32>` and `pub objective_row: Option<i32>` to StateSnapshot
    - Populate from GameState in `from_game_state()` — convert objective_hex via `hex.to_offset()`

    **1d. FFI functions for setting objective/turns:**
    - `norrust_set_objective_hex(engine, col, row)` → sets `state.objective_hex = Some(Hex::from_offset(col, row))`
    - `norrust_set_max_turns(engine, max_turns: i32)` → sets `state.max_turns = Some(max_turns as u32)` (or None if <= 0)
    - These are called by Love2D after loading a scenario

    **1e. BoardDef extension for scenario metadata:**
    - Add optional `objective_col`, `objective_row`, `max_turns` fields to BoardDef (all `#[serde(default)]`)
    - In `scenario::load_board()`, after creating GameState: if objective fields present, set `state.objective_hex`; if max_turns present, set `state.max_turns`
    - This way scenarios self-describe their win conditions

    **1f. New scenario files:**
    - `scenarios/crossing.toml`: 16×10 board
      - Blue keep at (1,5), castle cluster around it (cols 0-2)
      - Red keep at (14,5), castle cluster around it (cols 13-15)
      - Mixed terrain corridor in between: forest, hills, mountains, villages
      - `objective_col = 14`, `objective_row = 5` (red keep — player must reach it)
      - `max_turns = 30`
    - `scenarios/crossing_units.toml`: Blue faction starts with leader only on keep (1,5); Red faction starts with leader on keep (14,5) + 3-4 defenders in castle hexes

    **1g. Integration tests:**
    - `test_objective_hex_win`: Set up small board with objective_hex. Move unit to objective → check_winner() returns that faction
    - `test_turn_limit_loss`: Set max_turns=2 on a state. Advance past turn 2 → check_winner() returns faction 1
    - `test_elimination_still_works`: Existing elimination path still works when objective_hex is None
  </action>
