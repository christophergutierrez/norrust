<objective>
## Goal
Add a help overlay that shows all keybindings, toggled with the ? key.

## Purpose
Players shouldn't need to memorize keybindings. A toggleable help overlay provides discoverability without cluttering the screen.

## Output
- Help overlay toggled with ? key
- Shows all keybindings organized by context (global, gameplay, menu)
- Semi-transparent panel over the game board
</objective>

<context>
## Source Files
@norrust_love/main.lua — love.keypressed handler with all keybindings
@norrust_love/draw.lua — rendering module using ctx table pattern
</context>

<acceptance_criteria>

## AC-1: Help overlay toggles with ? key
```gherkin
Given the game is running in any mode
When the player presses ?
Then a semi-transparent help overlay appears showing all keybindings
And pressing ? again hides it
```

## AC-2: Help overlay shows correct keybindings
```gherkin
Given the help overlay is visible
Then it displays keybindings grouped by context:
  - Global: M (mute), -/= (volume), F5 (save), F9 (load), ? (help)
  - Gameplay: E (end turn), R (recruit), A (advance), H (history), P (path), Escape (cancel), Enter (confirm)
  - Menu: 1-9 (select scenario), C (campaign)
```

## AC-3: Help overlay doesn't block input
```gherkin
Given the help overlay is visible
When the player presses any game key (e.g., E, R, Escape)
Then the key functions normally AND the help overlay closes
```

</acceptance_criteria>

<tasks>

<task type="auto">
  <name>Task 1: Add help overlay state and rendering</name>
  <files>norrust_love/main.lua, norrust_love/draw.lua</files>
  <action>
    1. In main.lua, add `local show_help = false` near other state variables.
    2. Pass `show_help` to ctx in love.draw(): `ctx.show_help = show_help`
    3. In love.keypressed, add ? handler in the global section (after M/-/= handlers):
       - If key == "/" (? is shift+/ but Love2D reports "/" for ?) then toggle show_help
       - If show_help is true and any other key is pressed, set show_help = false (let the key pass through)
    4. In draw.lua, add a `draw.draw_help_overlay(ctx)` function:
       - Semi-transparent black background covering the board area (not the right panel)
       - Title: "Controls" in large font
       - Three columns or sections:
         - Global: ?, M, -/=, F5, F9, scroll wheel (zoom), drag (pan)
         - Gameplay: E, R, A, H, P, Escape, Enter, click (select), right-click (inspect terrain)
         - Menu: 1-9, C
       - Use ctx.fonts for consistent styling
    5. Call draw.draw_help_overlay(ctx) at the end of draw.draw_frame() when ctx.show_help is true
  </action>
  <verify>luajit -bl norrust_love/main.lua /dev/null && luajit -bl norrust_love/draw.lua /dev/null</verify>
