<objective>
## Goal
Create a standalone asset viewer app within the Love2D project, launchable via `--viewer` CLI flag. Enables browsing terrain tiles and unit sprites, cycling through animation states, zooming, flipping, and validating sprite.toml metadata visually.

## Purpose
Provides a tool for validating art assets before and during mass production (Phase 35). Artists and developers can preview sprites, check animation timing, verify frame alignment, and validate metadata without running the full game.

## Output
- `norrust_love/viewer.lua` — standalone asset viewer module (~250-350 lines)
- Updated `norrust_love/main.lua` — `--viewer` CLI flag wiring (3 lines)
</objective>

  <name>Task 1: Create viewer.lua + wire --viewer CLI flag</name>
  <files>norrust_love/viewer.lua, norrust_love/main.lua</files>
  <action>
    **1a. viewer.lua** — Standalone asset viewer module:

    **Layout:**
    - Left sidebar (220px): scrollable list of assets
      - Section "TERRAIN" header, then terrain_ids alphabetically
      - Section "UNITS" header, then def_ids alphabetically
      - Selected item highlighted (yellow text on dark background)
    - Main area: large preview of selected asset
    - Bottom bar: controls help text + metadata

    **State variables:**
    - `asset_list`: flat array of {type="terrain"|"unit", id=string}
    - `selected_idx`: 1-indexed into asset_list
    - `zoom`: scale factor (default 1.0, range 0.25-4.0)
    - `flipped`: boolean
    - `anim_state`: animation.new_state() for current unit
    - `anim_names`: array of available animation state names for current unit
    - `anim_name_idx`: index into anim_names for cycling

    **Viewer lifecycle (love callbacks):**
    - `viewer.load()` — called from love.load when --viewer:
      - Load terrain tiles via assets.load_terrain_tiles("assets")
      - Load unit sprites via assets.load_unit_sprites("assets")
      - Build asset_list from terrain keys + unit sprite keys
      - Select first item
      - Create fonts
    - `viewer.update(dt)` — update animation state if viewing a unit
    - `viewer.draw()` — render sidebar + preview + metadata
    - `viewer.keypressed(key)` — handle input
    - `viewer.wheelmoved(x, y)` — zoom via scroll

    **Input handling:**
    - Up/Down arrows: navigate asset list
    - Left/Right arrows: cycle animation state (units only)
    - +/= and -: zoom in/out
    - F: toggle flip
    - R: reset zoom and flip to defaults
    - 1-5 number keys: jump to animation state by index (idle=1, attack-melee=2, etc.)
    - Escape: quit

    **Terrain preview:**
    - Draw raw image centered and scaled by zoom factor
    - Draw a hex-clipped version below/beside it (reuse stencil pattern from assets.lua)
    - Show: terrain_id, image dimensions, file size (if available)

    **Unit preview:**
    - Draw current animation frame large and centered, scaled by zoom
    - Apply flip (negative scale_x) if flipped flag set
    - Draw spritesheet strip below preview (all frames visible, current frame highlighted)
    - Draw portrait in top-right corner if available
    - Show: def_id, current anim state, frame N/total, FPS, frame dimensions
    - Cycle through anim states with Left/Right or number keys

    **1b. Wire --viewer in main.lua:**
    - Add `elseif arg == "--viewer"` block in the CLI flag loop in love.load()
    - Set a module-level `viewer_mode = true` flag
    - Override love.update, love.draw, love.keypressed, love.wheelmoved to call viewer functions
    - Pattern: same as --generate-tiles but without quitting (viewer stays open)

    Avoid:
    - Do NOT modify assets.lua, animation.lua, or toml_parser.lua (reuse as-is)
    - Do NOT require the Rust engine (viewer is pure Lua + Love2D)
    - Do NOT add external Lua dependencies
  </action>
