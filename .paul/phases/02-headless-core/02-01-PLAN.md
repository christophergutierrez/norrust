## AC-1: Hex Distance Calculation

```gherkin
Given Hex at cubic coordinates (0, 0, 0)
And another Hex at cubic coordinates (2, -1, -1)
When distance() is called
Then the result is exactly 2
```

## AC-2: Hex Neighbor Retrieval

```gherkin
Given the origin Hex at (0, 0, 0)
When neighbors() is called
Then exactly 6 neighboring Hex values are returned
And each neighbor differs from origin by exactly 1 step in one axis direction
And each neighbor satisfies the cubic invariant (x + y + z == 0)
```

## AC-3: Offset Coordinate Round-Trip

```gherkin
Given any valid Hex with cubic coordinates (x, y, z)
When converted to odd-r offset coordinates (col, row)
And then converted back to cubic coordinates
Then the resulting Hex is identical to the original
```

  <action>
    Create norrust_core/src/hex.rs.

    Use cubic coordinates where x + y + z == 0 always holds.

    Define the 6 direction vectors as a const array:
      const DIRECTIONS: [Hex; 6] = [
          Hex { x: 1, y: -1, z: 0 },
          Hex { x: 1, y: 0, z: -1 },
          Hex { x: 0, y: 1, z: -1 },
          Hex { x: -1, y: 1, z: 0 },
          Hex { x: -1, y: 0, z: 1 },
          Hex { x: 0, y: -1, z: 1 },
      ];

    Define the Hex struct:
      #[derive(Debug, Clone, Copy, PartialEq, Eq, Hash)]
      pub struct Hex {
          pub x: i32,
          pub y: i32,
          pub z: i32,
      }

    Implement Hex methods:

    pub fn new(x: i32, y: i32, z: i32) -> Self
      - Use debug_assert!(x + y + z == 0) to catch bugs in debug builds
      - In release, trust the caller (no panic overhead)
      - Return Self { x, y, z }

    pub const ORIGIN: Hex = Hex { x: 0, y: 0, z: 0 };

    pub fn add(self, other: Hex) -> Hex
      - Return Hex { x: self.x + other.x, y: self.y + other.y, z: self.z + other.z }
      - No validation needed: sum of two valid Hexes is always valid

    pub fn subtract(self, other: Hex) -> Hex
      - Return Hex { x: self.x - other.x, y: self.y - other.y, z: self.z - other.z }

    pub fn distance(self, other: Hex) -> u32
      - Cube distance formula: (|Δx| + |Δy| + |Δz|) / 2
      - Return ((self.x - other.x).abs() + (self.y - other.y).abs() + (self.z - other.z).abs()) as u32 / 2

    pub fn neighbors(self) -> [Hex; 6]
      - Return DIRECTIONS.map(|d| self.add(d))

    Offset coordinate conversion (odd-r offset, pointy-top hex grid):
    This matches Wesnoth's coordinate convention.

    pub fn to_offset(self) -> (i32, i32)
      - col = self.x + (self.z - (self.z & 1)) / 2
      - row = self.z
      - Return (col, row)

    pub fn from_offset(col: i32, row: i32) -> Hex
      - x = col - (row - (row & 1)) / 2
      - z = row
      - y = -x - z
      - Return Hex::new(x, y, z)

    Add #[cfg(test)] module with these tests:

    test_distance_two():
      let a = Hex::new(0, 0, 0);
      let b = Hex::new(2, -1, -1);
      assert_eq!(a.distance(b), 2);

    test_distance_zero():
      let a = Hex::new(1, -1, 0);
      assert_eq!(a.distance(a), 0);

    test_neighbors_count_and_invariant():
      let center = Hex::ORIGIN;
      let neighbors = center.neighbors();
      assert_eq!(neighbors.len(), 6);
      for n in &neighbors {
          assert_eq!(n.x + n.y + n.z, 0, "neighbor violates cubic invariant");
          assert_eq!(center.distance(*n), 1, "each neighbor is exactly 1 step away");
      }

    test_offset_roundtrip():
      for x in -5..=5_i32 {
          for z in -5..=5_i32 {
              let y = -x - z;
              let hex = Hex::new(x, y, z);
              let (col, row) = hex.to_offset();
              let recovered = Hex::from_offset(col, row);
              assert_eq!(hex, recovered, "roundtrip failed for {:?}", hex);
          }
      }

    test_add_neighbors_back_to_origin():
      // Adding all 6 directions and their inverses returns to origin
      let start = Hex::new(1, -1, 0);
      for dir in Hex::ORIGIN.neighbors() {
          let moved = start.add(dir);
          let back = moved.subtract(dir);
          assert_eq!(back, start);
      }

    Edit norrust_core/src/lib.rs — add:
      pub mod hex;
    after the existing pub mod declarations.

    Avoid:
    - Using f32/f64 anywhere in this module — all hex math is integer arithmetic
    - Panicking in release builds (use debug_assert!, not assert!)
    - Implementing Display or any trait beyond Debug/Clone/Copy/PartialEq/Eq/Hash

<task type="auto">
  <name>Task 2: Implement Board struct with hex boundary validation</name>
  <files>norrust_core/src/board.rs, norrust_core/src/lib.rs</files>
  <action>
    Create norrust_core/src/board.rs.

    The Board defines the playable area in offset coordinate space (width × height).
    It uses offset coordinates because map dimensions are specified in grid cells,
    not in cubic coordinates.

      #[derive(Debug, Clone)]
      pub struct Board {
          pub width: u32,
          pub height: u32,
      }

      impl Board {
          pub fn new(width: u32, height: u32) -> Self {
              assert!(width > 0 && height > 0, "Board dimensions must be positive");
              Self { width, height }
          }

          /// Returns true if `hex` is within the board's bounds.
          pub fn contains(&self, hex: crate::hex::Hex) -> bool {
              let (col, row) = hex.to_offset();
              col >= 0 && row >= 0
                  && col < self.width as i32
                  && row < self.height as i32
          }
      }

    Add #[cfg(test)] module:

    test_board_contains_origin():
      let board = Board::new(10, 10);
      assert!(board.contains(Hex::ORIGIN));

    test_board_excludes_negative():
      let board = Board::new(10, 10);
      // Any hex with negative offset coords is out of bounds
      assert!(!board.contains(Hex::from_offset(-1, 0)));
      assert!(!board.contains(Hex::from_offset(0, -1)));

    test_board_excludes_beyond_size():
      let board = Board::new(5, 5);
      assert!(!board.contains(Hex::from_offset(5, 0)));
      assert!(!board.contains(Hex::from_offset(0, 5)));

      pub mod board;
