    3. Add `M.get_dialogue(engine, trigger, turn, faction)` — returns decoded Lua table (array of {id, text})

    **main.lua:**
    1. Add local `active_dialogue = {}` state variable (near other state vars)
    2. In scenario loading (when game_mode becomes PLAYING):
       - Derive dialogue path: `scenarios_path .. "/" .. scenario_board:gsub("%.toml$", "_dialogue.toml")`
       - Call `norrust.load_dialogue(engine, dialogue_path)` (ignore false return — file may not exist)
       - Fire scenario_start: `active_dialogue = norrust.get_dialogue(engine, "scenario_start", 1, 0)`
    3. In the "e" key handler (end turn), before `norrust.end_turn()`:
       - Fire turn_end: merge results into active_dialogue
       - After end_turn + AI: fire turn_start for new turn/faction
       - Set `active_dialogue` to the combined results
    4. Pass `active_dialogue` into the draw context (ctx.active_dialogue)

    Wire dialogue loading into both paths that enter PLAYING:
    - Preset scenario path (faction selection completes)
    - Campaign path (call_load_campaign_scenario)
    - Non-preset path (SETUP_RED → PLAYING)

    Create a helper `fire_dialogue_triggers(trigger, turn, faction)` to avoid duplication.
