<objective>
## Goal
Move campaign scenario orchestration from Lua (campaign_client.lua) into Rust so the engine owns all campaign state transitions. After this plan, Lua calls a single FFI function to advance to the next campaign scenario — no multi-step query-mutate sequences in Lua.

## Purpose
The Rust engine must own all gameplay logic so any frontend (Love2D, JS, etc.) can drive a campaign without reimplementing orchestration. Currently campaign_client.lua performs keep/castle finding, veteran placement, deploy overflow detection, and gold application — all gameplay logic that belongs in the engine.

## Output
- New Rust functions on `CampaignState`: `find_keep_and_castles()`, `place_veterans()`, `load_next_scenario()` (or equivalent)
- New FFI function: `norrust_campaign_load_next_scenario` — single call that loads board, places veterans (or signals deploy overflow), applies carry gold
- campaign_client.lua reduced to thin FFI wrapper
- Existing tests pass + new unit tests for the moved logic
</objective>

<task type="auto">
  <name>Task 1: Add keep/castle finding and veteran placement to CampaignState</name>
  <files>norrust_core/src/campaign.rs</files>
  <action>
    Add methods to CampaignState (or free functions operating on GameState):

    1. `find_keep_and_castles(state: &GameState, faction: u8) -> (Option<Hex>, Vec<Hex>)`:
       - Scan terrain for keep hexes. For faction 0 pick leftmost, for faction 1 pick rightmost.
       - Collect adjacent castle hexes (hex distance == 1 from selected keep).
       - Return (keep_hex, castle_hexes).

    2. `place_veterans(&mut self, state: &mut GameState, registry: &Registry) -> PlaceResult`:
       - Build slot list: keep first, then castles (from find_keep_and_castles).
       - Skip occupied hexes.
       - If more veterans than slots, return `PlaceResult::DeployNeeded { slots, veterans }`.
       - Otherwise place each veteran via registry lookup (like norrust_place_veteran_unit logic),
         heal to max HP, set XP/advancement.
       - Update self.id_map with new engine unit IDs → roster UUIDs.
       - Return `PlaceResult::Placed`.

    3. `load_next_scenario(&mut self, state: &mut GameState, registry: &Registry, scenarios_path: &str) -> ScenarioLoadResult`:
       - Read current_scenario() to get board/units paths.
       - Load board (reuse existing board loading). Load preset units if flagged.
       - Apply starting gold from faction definitions.
       - If scenario_index > 0 and veterans exist: call place_veterans.
       - Apply carry_gold to faction 0.
       - If first scenario (index 0): populate roster from all faction-0 units.
       - Return enum: Playing or DeployNeeded(deploy_info).

    Add unit tests for find_keep_and_castles and place_veterans.
    Do NOT touch balance tests.
  </action>
