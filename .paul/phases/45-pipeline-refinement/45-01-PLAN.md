<task type="auto">
  <name>Task 1: Create Python pipeline and unit prompt definitions</name>
  <files>
    norrust_love/tools/generate_sprites.py,
    norrust_love/tools/unit_prompts.toml
  </files>
  <action>
    Create `generate_sprites.py` — a self-contained Python 3 script (stdlib only) that:

    1. **Reads unit_prompts.toml** for the target unit's appearance description
    2. **Reads sprite.toml** from the unit's asset directory for frame counts
    3. **Generates frames** via Gemini API (same pattern as Phase 44 gen_mage.py):
       - One API call per frame
       - Prompt = base_desc + animation-specific suffix
       - Animation suffixes are generic (same for all units) — e.g., "raising weapon overhead"
       - The unit_prompts.toml provides the character-specific base description
    4. **Post-processes** each frame with ImageMagick flood-fill:
       - Fuzz 20% general, 8% for portrait
       - Trim, resize 256×256, center-pad with transparent background
    5. **Assembles spritesheets** via montage
    6. **Verifies dimensions** via identify

    CLI interface:
    - `python3 generate_sprites.py --unit mage` — generate one unit
    - `python3 generate_sprites.py --all` — generate all 16 units
    - `python3 generate_sprites.py --unit mage --dry-run` — print prompts only
    - `python3 generate_sprites.py --unit mage --portrait-fuzz 8` — override portrait fuzz

    Create `unit_prompts.toml` with entries for all 16 units:
    ```toml
    [mage]
    base_desc = "An elderly wizard mage facing to the right..."
    faction = "loyalist"
    weapon = "tall wooden staff topped with glowing purple crystal orb"

    [spearman]
    base_desc = "A young human soldier facing to the right..."
    faction = "loyalist"
    weapon = "long iron-tipped spear and round wooden shield"
    ```

    Each entry needs:
    - `base_desc`: Full character appearance (race, build, clothing, colors, distinguishing features)
    - `faction`: loyalist/elf/orc (for thematic consistency)
    - `weapon`: Primary weapon description (used in animation prompts)
    - `portrait_fuzz` (optional): Override fuzz % for portrait bg removal

    Unit roster (from assets/units/):
    Loyalists (7): bowman, cavalryman, heavy_infantryman, lieutenant, mage, sergeant, spearman
    Elves (5): elvish_archer, elvish_captain, elvish_fighter, elvish_scout, elvish_shaman
    Orcs (4): orcish_archer, orcish_assassin, orcish_grunt, orcish_warrior

    Animation suffix templates (generic, unit-agnostic):
    - idle: neutral standing, weapon glow, robes sway, contemplative
    - attack-melee: raise weapon, wind up, mid-swing, full strike, follow-through, recover
    - attack-ranged: prepare, channel, release, aftermath (skip for melee-only units)
    - defend: brace, full defense/barrier, recover
    - death: stagger, fall, collapse angle, ground
    - portrait: close-up head and upper body

    Avoid: hardcoding mage-specific prompts in the script. All character specifics come from unit_prompts.toml.
    Avoid: external dependencies beyond Python stdlib + ImageMagick.
  </action>
  <verify>
    python3 norrust_love/tools/generate_sprites.py --unit mage --dry-run
    (should print prompts and commands without API calls)
  </verify>
