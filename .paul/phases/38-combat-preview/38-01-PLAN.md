<objective>
## Goal
Add Monte Carlo combat simulation that previews attack outcomes without mutating game state, displayed in a panel when clicking an enemy from ghost position.

## Purpose
Players need to see expected combat outcomes before committing to an attack. This is the core tactical information layer — without it, players must memorize damage tables and mentally calculate hit probabilities. The preview transforms the game from guesswork to informed decision-making.

## Output
- `simulate_combat()` pure Rust function running N attack simulations
- `norrust_simulate_combat()` FFI function returning JSON distribution
- Combat preview panel in Love2D showing damage/kill probabilities
- Click-enemy-from-ghost shows preview instead of immediate attack
</objective>

  <name>Task 1: Rust simulate_combat() + FFI + Lua binding</name>
  <files>norrust_core/src/combat.rs, norrust_core/src/ffi.rs, norrust_love/norrust.lua</files>
  <action>
    **combat.rs — Add `simulate_combat()` function:**

    Add a new public struct `CombatPreview` with fields:
    - `attacker_damage_min`, `attacker_damage_max`, `attacker_damage_mean` (u32/u32/f64)
    - `defender_damage_min`, `defender_damage_max`, `defender_damage_mean` (u32/u32/f64)
    - `attacker_kill_pct` (f64) — probability attacker kills defender
    - `defender_kill_pct` (f64) — probability defender kills attacker (via retaliation)
    - `attacker_hit_pct` (u32) — hit chance for attacker (100 - defender terrain defense)
    - `defender_hit_pct` (u32) — hit chance for defender retaliation (100 - attacker terrain defense)
    - `attacker_attack_name` (String) — name of attacker's chosen attack
    - `defender_attack_name` (Option&lt;String&gt;) — name of defender's retaliation attack (None if no matching range)
    - `attacker_damage_per_hit` (u32) — effective damage per hit (after resistance + ToD)
    - `attacker_strikes` (u32) — number of strikes
    - `defender_damage_per_hit` (u32) — effective damage per hit for retaliation
    - `defender_strikes` (u32) — number of retaliation strikes

    Add `pub fn simulate_combat(...)` that takes:
    - Immutable references to: attacker Unit, defender Unit, attacker terrain defense (u32), defender terrain defense (u32), turn (u32)
    - `num_simulations: u32` (e.g. 100)

    Logic:
    1. Determine engagement range: use distance=1 (melee) since ghost adjacent enemies are always distance 1
    2. Find attacker's "melee" attack — if none, return empty/zero preview
    3. Calculate attacker's effective damage: apply defender's resistance to attack_type, then ToD modifier from attacker alignment
    4. Calculate attacker hit_pct = 100 - defender_terrain_defense
    5. Find defender's "melee" retaliation attack — may be None
    6. If retaliation exists: calculate effective damage (apply attacker's resistance to def_attack_type, ToD from defender alignment), hit_pct = 100 - attacker_terrain_defense
    7. Run `num_simulations` iterations with Rng seeded from `i + 1` each round:
       - resolve_attack() for attacker → damage to defender
       - If defender survives AND has retaliation: resolve_attack() for defender → damage to attacker
       - Track: attacker_damage, defender_damage, attacker_kills, defender_kills
    8. Compute min/max/mean for both damage arrays
    9. Kill percentages = kills / num_simulations * 100

    **ffi.rs — Add `norrust_simulate_combat()` FFI function:**

    ```
    norrust_simulate_combat(engine, attacker_id, defender_id, attacker_col, attacker_row, num_sims) -> *mut c_char
    ```

    Logic:
    - Look up attacker and defender from state.units
    - Determine attacker terrain defense at (attacker_col, attacker_row) — this is the ghost position, not current position. Use same fallback chain as combat: unit.defense[terrain_id] → tile.defense → unit.default_defense
    - Determine defender terrain defense at defender's current position (from state.positions)
    - Call simulate_combat() with extracted parameters
    - Return JSON string using format!() (same pattern as norrust_get_unit_terrain_info)

    JSON format:
    ```json
    {
      "attacker_hit_pct": 60,
      "defender_hit_pct": 50,
      "attacker_damage_per_hit": 7,
      "attacker_strikes": 3,
      "defender_damage_per_hit": 5,
      "defender_strikes": 2,
      "attacker_damage_min": 0,
      "attacker_damage_max": 21,
      "attacker_damage_mean": 12.6,
      "defender_damage_min": 0,
      "defender_damage_max": 10,
      "defender_damage_mean": 5.0,
      "attacker_kill_pct": 15.0,
      "defender_kill_pct": 3.0,
      "attacker_attack_name": "sword",
      "defender_attack_name": "spear",
      "defender_hp": 36,
      "attacker_hp": 38
    }
    ```
    Include defender_hp and attacker_hp so the UI can show context.
    If defender has no retaliation attack: defender_* damage fields are 0, defender_attack_name is "none", defender_kill_pct is 0.

    **norrust.lua — Add FFI declaration + Lua wrapper:**

    Add to ffi.cdef: `char* norrust_simulate_combat(NorRustEngine* engine, int32_t attacker_id, int32_t defender_id, int32_t attacker_col, int32_t attacker_row, int32_t num_sims);`

    Add wrapper:
    ```lua
    function M.simulate_combat(engine, attacker_id, defender_id, attacker_col, attacker_row, num_sims)
        local raw = get_string(lib.norrust_simulate_combat(engine, attacker_id, defender_id, attacker_col, attacker_row, num_sims or 100))
        if raw == "" then return nil end
        return json_decode(raw)
    end
    ```
