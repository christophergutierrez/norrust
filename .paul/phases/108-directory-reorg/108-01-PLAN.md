<task type="auto">
  <name>Task 1: Make load_from_dir recursive</name>
  <files>norrust_core/src/loader.rs</files>
  <action>
    Modify `load_from_dir` to recurse into all subdirectories, not just immediate children.

    Current behavior:
    - Loads flat `.toml` files at root
    - For immediate subdirs, looks for `<dirname>.toml` inside

    New behavior:
    - Loads flat `.toml` files at root (unchanged)
    - For subdirectories, looks for `<dirname>.toml` inside
    - THEN recurses into each subdirectory to find deeper nested directories
    - Recursion continues to arbitrary depth

    Implementation approach:
    - Extract the directory scanning into a recursive helper function
    - The helper takes a path and the items HashMap
    - For each entry in the directory:
      - If it's a `.toml` file, parse and insert (only at the top-level call — flat TOMLs at root)
      - If it's a subdirectory, look for `<dirname>.toml` and recurse into it
    - Flat `.toml` files in subdirectories should be IGNORED (only `<dirname>.toml` convention)

    This is backward-compatible: existing 1-level deep directories still load identically. New deeper directories also load.
  </action>
  <verify>cargo test -p norrust_core test_unit_registry_loads — existing test passes with recursive loader</verify>
  <done>AC-1 satisfied: loader recurses to arbitrary depth while maintaining backward compatibility</done>
</task>
