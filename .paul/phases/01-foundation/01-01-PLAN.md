<objective>
## Goal

Initialize the Rust library project and implement the data schema layer: TOML data
files for units and terrain, corresponding Serde-compatible Rust structs, and a
`Registry` that loads game data from disk into memory.

## Purpose

This establishes the "language of the game" — static data that all future simulation
systems will read. A working data loader with passing tests proves the Rust core can
ingest configuration files before any game logic is written. It also satisfies the
CI/linting requirement and stubs the `GameState` public API boundary that Redot will
eventually interact with.

## Output

- `norrust_core/` — initialized `cargo new --lib` Rust library
- `data/units/` — 2 unit TOML files (Fighter, Archer)
- `data/terrain/` — 2 terrain TOML files (Grassland, Forest)
- `norrust_core/src/schema.rs` — Serde structs: `UnitDef`, `TerrainDef`, `AttackDef`
- `norrust_core/src/loader.rs` — `Registry` struct with `load_from_dir()`
- Unit tests embedded in `loader.rs` — all pass under `cargo test`
- `cargo clippy -- -D warnings` is clean
</objective>

    Create norrust_core/src/loader.rs.

    Define a RegistryError using thiserror:
      #[derive(Debug, thiserror::Error)]
      pub enum RegistryError {
          #[error("IO error reading {path}: {source}")]
          Io { path: String, #[source] source: std::io::Error },
          #[error("TOML parse error in {path}: {source}")]
          Toml { path: String, #[source] source: toml::de::Error },
      }

    Define Registry:
      pub struct Registry<T> {
          items: std::collections::HashMap<String, T>,
      }

      impl<T: serde::de::DeserializeOwned + IdField> Registry<T> {
          /// Load all .toml files from `dir` into a Registry keyed by each item's id.
          pub fn load_from_dir(dir: &std::path::Path) -> Result<Self, RegistryError>
          pub fn get(&self, id: &str) -> Option<&T>
          pub fn all(&self) -> impl Iterator<Item = &T>
          pub fn len(&self) -> usize
          pub fn is_empty(&self) -> bool
      }

    Define a helper trait IdField:
      pub trait IdField { fn id(&self) -> &str; }

    Implement IdField for UnitDef and TerrainDef (each returns &self.id).

    The load_from_dir implementation should:
    - Read the directory listing
    - Filter for files with `.toml` extension
    - Read each file to a String, parse with `toml::from_str::<T>()`
    - Insert into the HashMap keyed by item.id().to_owned()
    - Propagate errors with RegistryError variants

    Add a #[cfg(test)] module at the bottom with two tests:

    Test 1 — unit registry loads from real data dir:
      fn test_unit_registry_loads() {
          // Path relative to norrust_core/ during cargo test
          let dir = std::path::Path::new("../data/units");
          let registry: Registry<UnitDef> = Registry::load_from_dir(dir).unwrap();
          assert_eq!(registry.len(), 2);
          let fighter = registry.get("fighter").expect("fighter not found");
          assert_eq!(fighter.max_hp, 30);
          assert_eq!(fighter.movement, 5);
          assert_eq!(fighter.attacks[0].damage, 7);
          let archer = registry.get("archer").expect("archer not found");
          assert_eq!(archer.max_hp, 25);
      }

    Test 2 — terrain registry loads correctly:
      fn test_terrain_registry_loads() {
          let dir = std::path::Path::new("../data/terrain");
          let registry: Registry<TerrainDef> = Registry::load_from_dir(dir).unwrap();
          assert_eq!(registry.len(), 2);
          let grass = registry.get("grassland").expect("grassland not found");
          assert_eq!(grass.default_defense, 40);
          assert_eq!(grass.default_movement_cost, 1);
      }

    Test 3 — missing key returns None:
      fn test_get_nonexistent_returns_none() {
          let dir = std::path::Path::new("../data/units");
          let registry: Registry<UnitDef> = Registry::load_from_dir(dir).unwrap();
          assert!(registry.get("dragon").is_none());
      }

    Avoid: using unwrap() outside of tests, using global/static mutable state,
    using std::sync::Mutex on Registry (it should be constructed once and treated
    as immutable after loading).
