<objective>
## Goal

Configure `norrust_core` to compile as a GDExtension dynamic library and prove that
Redot can call Rust code. By the end: running the Redot project prints "0.1.0" to the
output console — a string originating from Rust.

## Purpose

This validates the entire cross-language architecture before any game logic is written.
Once a Rust function executes inside Redot, all future simulation features have a proven
integration path.

## Output

- `norrust_core` compiles as both `cdylib` (for Redot) and `rlib` (for tests)
- `norrust_core/src/gdext_node.rs` — `NorRustCore` GodotClass with `get_core_version()`
- `norrust_client/` — Redot project scaffold (project.godot, dirs, scenes, scripts)
- `norrust_client/norrust_core.gdextension` — extension config pointing to compiled .so
- `norrust_client/scripts/test_bridge.gd` — GDScript calling `get_core_version()`
- Redot output console shows: `Core version: 0.1.0`
</objective>

  <name>Task 1: Configure norrust_core as cdylib + implement NorRustCore GodotClass</name>
  <files>
    norrust_core/Cargo.toml,
    norrust_core/src/lib.rs,
    norrust_core/src/gdext_node.rs
  </files>
  <action>
    Edit norrust_core/Cargo.toml:

    1. Add a [lib] section above [dependencies]:
         [lib]
         crate-type = ["cdylib", "rlib"]

       Both types are required: "cdylib" for Redot to load as a shared library,
       "rlib" for "cargo test" to continue working.

    2. Add to [dependencies]:
         godot = "0.2"

       The "godot" crate is the official gdext binding (https://crates.io/crates/godot).
       It ships pre-generated bindings for Godot 4.x — no Redot binary needed to compile.
       Note: if Redot's API version differs from the bundled API, add a feature flag like
       features = ["api-4-3"] to pin to the matching Godot 4.3 API.

    Create norrust_core/src/gdext_node.rs:

      use godot::prelude::*;

      /// GDExtension entry point — registers all Rust classes with Redot.
      struct NorRustExtension;

      #[gdextension]
      unsafe impl ExtensionLibrary for NorRustExtension {}

      /// The primary Rust node exposed to Redot.
      /// Instantiate via NorRustCore.new() in GDScript.
      #[derive(GodotClass)]
      #[class(base=Node)]
      pub struct NorRustCore {
          base: Base<Node>,
      }

      #[godot_api]
      impl INode for NorRustCore {
          fn init(base: Base<Node>) -> Self {
              Self { base }
          }
      }

      #[godot_api]
      impl NorRustCore {
          /// Returns the norrust_core library version string.
          #[func]
          fn get_core_version(&self) -> GString {
              "0.1.0".into()
          }
      }

    Edit norrust_core/src/lib.rs — add the gdext_node module declaration:
      pub mod gdext_node;

    Add it after the existing pub mod loader; and pub mod schema; lines.

    Avoid:
    - Importing godot types in schema.rs or loader.rs — keep those modules headless
    - Adding game logic to gdext_node.rs — it's a bridge only
    - Removing the existing GameState stub
  </action>
  <verify>
    cd norrust_core && cargo build 2>&1 | tail -5
    # Expect: "Finished" with libnorrust_core.so in target/debug/
    # Then verify existing tests still pass:
    cd norrust_core && cargo test 2>&1 | tail -5
    # Expect: "3 passed; 0 failed"
  </verify>
  <done>AC-1 satisfied: cargo build produces .so; cargo test 3/3 still pass</done>
</task>
