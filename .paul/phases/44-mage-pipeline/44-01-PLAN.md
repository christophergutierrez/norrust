<task type="auto">
  <name>Task 1: Generate Mage sprites via AI and assemble spritesheets</name>
  <files>
    norrust_love/assets/units/mage/idle.png,
    norrust_love/assets/units/mage/attack-melee.png,
    norrust_love/assets/units/mage/attack-ranged.png,
    norrust_love/assets/units/mage/defend.png,
    norrust_love/assets/units/mage/death.png,
    norrust_love/assets/units/mage/portrait.png
  </files>
  <action>
    Use the mcp__nana-banana__generate_image MCP tool to generate individual frames for each animation state. The approach:

    1. **Generate individual frames** for each animation state using carefully crafted prompts:
       - Each prompt must describe: fantasy mage character, purple robes, long white beard, glowing purple crystal staff, transparent/plain background, pixel art or painterly game sprite style, specific pose for the animation frame
       - Request 256x256 size for each frame
       - Idle (4 frames): subtle breathing/staff glow variations
       - Attack-melee (6 frames): staff swing sequence
       - Attack-ranged (4 frames): casting spell sequence with magic effects
       - Defend (3 frames): defensive crouch/shield pose
       - Death (4 frames): falling/collapse sequence
       - Portrait (1 frame): face/upper body close-up

    2. **Post-process each frame** with ImageMagick:
       - Resize/pad to exactly 256×256 if needed: `magick input.png -resize 256x256 -gravity center -extent 256x256 output.png`
       - Ensure transparent background (PNG alpha channel)

    3. **Assemble spritesheets** with ImageMagick montage:
       - `montage frame1.png frame2.png ... -tile Nx1 -geometry 256x256+0+0 -background none output.png`
       - N = frame count from sprite.toml

    4. **Verify dimensions** with `identify` on each output file

    Style reference: Match the examples in mage/examples/ — detailed painterly fantasy character with transparent background. The character should be a classic fantasy mage: elderly, long white beard, purple/violet flowing robes, tall pointed hat or hood, wooden staff topped with a glowing purple crystal orb.

    Avoid: changing sprite.toml frame counts, modifying any Lua code, generating JPEG (must be PNG with alpha)
  </action>
  <verify>
    Run `identify` on all 6 output PNGs and confirm dimensions match sprite.toml:
    - idle.png: 1024×256
    - attack-melee.png: 1536×256
    - attack-ranged.png: 1024×256
    - defend.png: 768×256
    - death.png: 1024×256
    - portrait.png: 256×256
  </verify>
  <done>AC-1 partially satisfied (generation done inline rather than script), AC-2 satisfied</done>
</task>
