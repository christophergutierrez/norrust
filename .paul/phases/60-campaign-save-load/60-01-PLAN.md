<objective>
## Goal
Extend save/load to support campaign mode: save campaign position, veterans, faction IDs, trigger zone state, and dialogue fired state. Add auto-save at turn start.

## Purpose
Campaigns span multiple scenarios. Without campaign save/load, progress is lost on quit. This enables resuming mid-campaign with all context intact.

## Output
- save.lua extended with campaign fields in TOML
- 2 new Rust FFI functions for trigger zone and dialogue fired state
- norrust.lua wrappers for new FFI
- Auto-save on turn start (end-turn key handler)
- F9 restores campaign context on load
</objective>

## AC-3: Trigger Zone State Preserved
```gherkin
Given a trigger zone has already fired (spawned units)

Then the trigger zone remains marked as triggered and does not fire again
```

## AC-4: Dialogue Fired State Preserved
```gherkin
Given dialogue entries have already been shown (fired)
When the game is saved and loaded
Then the same dialogue entries do not replay
```

## AC-5: Auto-Save at Turn Start
```gherkin
Given a game is in progress (scenario or campaign)
When the player ends their turn (presses 'e')
Then a save file is automatically written before the AI takes its turn
```

<task type="auto">
  <name>Task 1: FFI for trigger zone and dialogue fired state</name>
  <files>norrust_core/src/ffi.rs, norrust_love/norrust.lua</files>
  <action>
    Add 4 new FFI functions:

    1. `norrust_get_trigger_zones_fired(engine) -> *mut c_char`
       Returns JSON array of booleans matching trigger_zones order: [true, false, true, ...]
       If no game state, return "[]".

    2. `norrust_set_trigger_zone_fired(engine, index: i32, fired: i32)`
       Sets trigger_zones[index].triggered = (fired != 0).

    3. `norrust_get_dialogue_fired(engine) -> *mut c_char`
       Returns JSON array of strings: the fired HashSet entries.
       If no dialogue state, return "[]".

    4. `norrust_set_dialogue_fired(engine, ids_json: *const c_char)`
       Parses JSON array of strings and inserts each into dialogue.fired HashSet.

    Add Lua FFI declarations and wrapper functions in norrust.lua:
    - M.get_trigger_zones_fired(engine) -> table of booleans
    - M.set_trigger_zone_fired(engine, index, fired)
    - M.get_dialogue_fired(engine) -> table of strings
    - M.set_dialogue_fired(engine, ids_json_str)
  </action>
  <verify>cargo test passes; luajit -bl norrust_love/norrust.lua /dev/null succeeds</verify>
  <done>AC-3, AC-4 partial: FFI exists to query/restore trigger and dialogue state</done>
</task>

    - def_id, hp, xp, xp_needed, advancement_pending

    Add [state] section for trigger/dialogue fired:
    - trigger_zones_fired = [true, false, ...] (TOML array of booleans)
    - dialogue_fired = ["id1", "id2", ...] (TOML array of strings)

    For TOML arrays: add toml_array(arr) helper that serializes a Lua table as TOML inline array.

    Extend parse_save_toml to handle TOML inline arrays (values starting with "[").

    Extend save.load_save():
    - If data.campaign exists, return it so main.lua can restore campaign state
    - After placing units, restore trigger zones: loop data.state.trigger_zones_fired, call norrust.set_trigger_zone_fired for each true entry
    - After loading dialogue, restore fired state: build JSON array from data.state.dialogue_fired, call norrust.set_dialogue_fired
  </action>
