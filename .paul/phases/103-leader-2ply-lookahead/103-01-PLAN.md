<objective>
## Goal
Add 2-ply lookahead to `plan_unit_action`: for each candidate (move, attack), simulate the opponent's best response, then evaluate. This catches oscillation (leader leaves keep → opponent punishes → leader returns next turn) because the planner sees the consequences of leaving keep one turn ahead.

## Purpose
The current 1-ply lookahead only evaluates the board state after one unit's action. It can't see that leaving keep will force a wasted turn returning. 2-ply lookahead lets units (especially the leader) anticipate opponent responses, making movement decisions much more intelligent.

## Output
Modified `plan_unit_action` in `norrust_core/src/ai.rs` with optional 2-ply depth. Performance-gated: 2-ply for leader only if unit count > threshold, otherwise 2-ply for all units.
</objective>

- 2-ply means: for each candidate action, simulate ONE opponent response (opponent's best per-unit action on the resulting state), then evaluate
- Opponent response simulation: for each enemy unit, find best 1-ply action, apply all, then evaluate

    Create a helper function `evaluate_with_opponent_response`:

    ```rust
    fn evaluate_with_opponent_response(state: &GameState, faction: u8) -> f32 {
        let enemy = 1 - faction;
        let mut sim = state.clone();

        // Simulate opponent's best response: each enemy unit takes its best 1-ply action
        let enemy_ids: Vec<u32> = sim.units.iter()
            .filter(|(_, u)| u.faction == enemy && !u.attacked)
            .map(|(&id, _)| id)
            .collect();

        for &eid in &enemy_ids {
            if !sim.units.contains_key(&eid) { continue; } // May have died
            if let Some((dest, target)) = plan_unit_action(&sim, eid, enemy) {
                let start = sim.positions[&eid];
                if dest != start {
                    let _ = apply_action(&mut sim, Action::Move { unit_id: eid, destination: dest });
                }
                if let Some(tid) = target {
                    if sim.units.contains_key(&eid) && sim.units.contains_key(&tid) {
                        let _ = apply_action(&mut sim, Action::Attack { attacker_id: eid, defender_id: tid });
                    }
                }
            }
        }

        evaluate_state(&sim, faction)
    }
    ```

    This function takes a post-action state and simulates the opponent's best 1-ply response before evaluating. The opponent uses the same `plan_unit_action` logic, so their response is realistic.
