<objective>
## Goal
Reduce love.load upvalues from 55 to ~30 by bundling mode constants, game data, and module references into tables. Gives permanent headroom for future features.

## Purpose
LuaJIT enforces a hard 60-upvalue limit per function. love.load is at 55 — adding any significant feature risks hitting the ceiling. Grouping related locals into tables is the established pattern (Phase 78 did this for state variables).

## Output
- `norrust_love/main.lua` — mode constants, game data, and modules grouped into tables
- `norrust_love/input.lua` — references updated for new table locations
</objective>
