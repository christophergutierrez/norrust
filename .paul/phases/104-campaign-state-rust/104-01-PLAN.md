<objective>
## Goal
Create a `CampaignState` struct in Rust that owns all campaign progression state: scenario index, carry-over gold, veteran list, and UUID-based roster. Wire it into `NorRustEngine` with FFI functions. Lua does NOT change in this phase — we're building the Rust foundation only.

## Purpose
Currently campaign state is scattered across Lua tables (campaign_ctx, roster.lua, save.lua). Moving ownership to Rust makes save/load trivial (Phase 105: just serialize the struct) and eliminates Lua-side state bugs. This is the architectural foundation for the entire v3.7 milestone.

## Output
- `CampaignState` struct on `NorRustEngine` with full campaign progression
- `RosterEntry` struct with UUID-based unit identity tracking
- FFI functions to start/query/advance campaigns and manage roster
- Headless integration tests proving the full campaign lifecycle works
</objective>

    Add to campaign.rs:

    1. `RosterEntry` struct:
       - uuid: String (8-char hex)
       - def_id: String
       - hp: u32, max_hp: u32
       - xp: u32, xp_needed: u32
       - advancement_pending: bool
       - abilities: Vec<String>

       - campaign_def: CampaignDef (loaded once)
       - scenario_index: usize (0-based, current scenario)
       - carry_gold: u32 (gold for next scenario)

       - roster: HashMap<String, RosterEntry> (uuid → entry)
       - id_map: HashMap<u32, String> (engine_id → uuid, per-scenario)

    3. `CampaignState` methods:
       - `new(def: CampaignDef) -> Self` — empty roster, index 0
       - `generate_uuid() -> String` — 8-char hex from SmallRng

       - `clear_id_map(&mut self)` — clears between scenarios

       - `record_victory(&mut self, state: &GameState, faction: u8)` — calls sync, extracts veterans via get_survivors, calculates carry_gold, advances scenario_index
       - `current_scenario(&self) -> Option<&CampaignScenarioDef>` — returns scenario at current index

       - test_roster_add_and_get_living: add 3 units, mark 1 dead, get_living returns 2
       - test_roster_sync_marks_dead: sync from state missing a unit → status=Dead
       - test_generate_uuid: verify 8-char hex format

    Avoid: Adding any FFI or Lua changes in this task. Keep it pure Rust types + methods.
    Avoid: Using external UUID crates — use SmallRng like existing code.

<task type="auto">
  <name>Task 2: Wire CampaignState into NorRustEngine with FFI</name>
  <files>norrust_core/src/ffi.rs</files>
  <action>
    Add campaign state to NorRustEngine and expose via FFI:

    1. Add `campaign: Option<CampaignState>` field to `NorRustEngine` struct.

    2. New FFI functions (all follow existing let-else error pattern):

       `norrust_start_campaign(engine, path) -> *mut c_char`
       - Loads CampaignDef via load_campaign()
       - Creates CampaignState::new(def)
       - Stores in engine.campaign = Some(state)
       - Returns JSON of campaign def (same as current load_campaign)
       - Replaces current norrust_load_campaign

       `norrust_campaign_add_unit(engine, def_id, engine_id, hp, max_hp, xp, xp_needed, advancement_pending) -> *mut c_char`
       - Calls campaign.add_unit(), returns UUID as C string
       - Returns null if no active campaign

       `norrust_campaign_map_id(engine, engine_id, uuid)`
       - Maps engine ID to existing UUID in roster

       `norrust_campaign_sync_roster(engine, faction) -> i32`
       - Calls campaign.sync_from_state(game_state, faction)
       - Returns 0 on success, -1 if no campaign/state

       `norrust_campaign_record_victory(engine, faction) -> *mut c_char`
       - Calls campaign.record_victory(state, faction)
       - Returns JSON with {veterans: [...], carry_gold: N, scenario_index: N, roster: [...]}

       `norrust_get_campaign_state_json(engine) -> *mut c_char`
       - Returns full CampaignState as JSON (for save system in Phase 105)
       - Includes: scenario_index, carry_gold, veterans, roster entries

       `norrust_campaign_get_living_json(engine) -> *mut c_char`
       - Returns JSON array of living RosterEntry for deployment UI

    3. Keep existing FFI functions (get_survivors_json, get_carry_gold, place_veteran_unit) working — they're used by Lua until Phase 106 migrates. Do NOT remove them.

    Avoid: Changing any Lua code. Avoid: Breaking existing FFI function signatures.
  </action>
