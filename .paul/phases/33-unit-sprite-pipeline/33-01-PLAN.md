<objective>
## Goal
Build a complete unit sprite pipeline for the Spearman — from programmatic sprite generation through animation playback in-game — validating every stage before mass-producing art for other units.

## Purpose
Replaces colored circles with animated unit sprites, making the game visually recognizable as a strategy game. One unit end-to-end proves the pipeline before scaling to 300+ units.

## Output
- Programmatic Spearman spritesheets (idle, attack-melee, attack-ranged, defend, death) + portrait
- sprite.toml metadata file
- Minimal TOML parser for Lua (sprite.toml subset only)
- Animation module (frame cycling, state machine, per-unit tracking)
- Updated draw_units with animation playback, facing/flip, team underlay
- Portrait rendering in unit panel sidebar
</objective>

    **1a. generate_sprites.lua** — Programmatic sprite generator (like generate_tiles.lua pattern):
    - Create `norrust_love/generate_sprites.lua` with Love2D canvas drawing

      - `idle.png`: 4-frame horizontal strip (256×256 per frame = 1024×256 total). Draw a stylized spearman figure with slight idle sway between frames (shift body/spear position slightly). Use earth tones (brown/tan) for body, grey for spear tip, dark outline.
      - `attack-melee.png`: 6 frames (1536×256). Spear thrust animation — arm extends forward across frames.
      - `attack-ranged.png`: 4 frames (1024×256). Javelin throw — arm raises then releases.
      - `defend.png`: 3 frames (768×256). Shield-raise/brace pose.
      - `death.png`: 4 frames (1024×256). Figure falls/collapses.
      - `portrait.png`: Single 256×256 image. Close-up bust of spearman (head + shoulders + spear tip).
    - Each frame should have a transparent background (alpha=0)
    - Use similar canvas → ImageData → encode("png") pattern as generate_tiles.lua
    - Add `--generate-sprites` CLI flag in main.lua's love.load() (alongside existing --generate-tiles)
    - Generated files go to Love2D save directory, then must be copied to source tree

    **1b. sprite.toml** — Create `norrust_love/assets/units/Spearman/sprite.toml`:
    ```toml
    id = "Spearman"

    [idle]
    file = "idle.png"
    frame_width = 256
    frame_height = 256
    frames = 4
    fps = 4

    [attacks.melee]
    file = "attack-melee.png"
    frame_width = 256
    frame_height = 256
    frames = 6
    fps = 8

    [attacks.ranged]
    file = "attack-ranged.png"
    frame_width = 256
    frame_height = 256
    frames = 4
    fps = 6

    [defend]
    file = "defend.png"
    frame_width = 256
    frame_height = 256
    frames = 3
    fps = 6

    [death]
    file = "death.png"
    frame_width = 256
    frame_height = 256
    frames = 4
    fps = 6

    [portrait]
    file = "portrait.png"
    ```

    **1c. toml_parser.lua** — Minimal TOML parser for sprite.toml subset:
    - Create `norrust_love/toml_parser.lua`
    - Parse: key = "string", key = number, [section], [section.subsection]
    - No need: arrays, arrays-of-tables, multiline strings, dates
    - Returns nested Lua table: `{id="Spearman", idle={file="idle.png", frames=4, ...}, attacks={melee={...}}, ...}`
    - ~50-80 lines, pure Lua (no external deps)

    **1d. animation.lua** — Animation state machine module:
    - Create `norrust_love/animation.lua`
    - `animation.load_unit_anims(base_path, def_id, toml_data)` — loads spritesheet Images + creates Quad arrays for each animation state. Returns anim_data table: `{idle={img, quads, fps, loop=true}, attacks={melee={img, quads, fps, loop=false}}, ...}`
    - `animation.new_state()` — returns per-unit animation state: `{current="idle", frame=1, timer=0, facing="right"}`
    - `animation.update(anim_state, anim_data, dt)` — advances frame timer, wraps for looping anims, stops on last frame for one-shot
    - `animation.play(anim_state, anim_name)` — transitions to new state (reset frame/timer)
    - `animation.get_quad(anim_state, anim_data)` — returns current Image + Quad for drawing
    - `animation.is_finished(anim_state, anim_data)` — true if one-shot anim completed
    - Use Love2D Quads for spritesheet frame selection (no external anim8 library)

    Avoid:
    - Do NOT modify Rust code (sprites are purely client-side)
    - Do NOT add external Lua dependencies
    - Do NOT parse full TOML spec — only the subset used by sprite.toml
  </action>
