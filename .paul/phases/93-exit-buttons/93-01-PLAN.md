<objective>
## Goal
Add exit buttons: (1) "Exit" sidebar button during gameplay that returns to main menu with a save prompt, (2) "Quit" option on the main menu (PICK_SCENARIO) screen to close the application.

## Purpose
Players currently have no visible way to leave a game in progress or quit the application. The only exits are closing the window or knowing hidden key combos.

## Output
- Exit button in sidebar during PLAYING mode
- Save prompt before leaving mid-game
- Quit hint on main menu screen
</objective>

## AC-2: Quit on Main Menu

  <action>
    **draw.lua — draw_sidebar_buttons():**
    - Add an "Exit" button above the Help button (bottom of sidebar stack)
    - Only show during PLAYING mode (similar to End Turn / Recruit)
    - Use a muted red/dark tone (0.4, 0.15, 0.15 fill)
    - If `ctx.exit_confirm` is true, change button text to "Save+Exit? Y/N" or similar confirmation prompt
    - Register in ctx.buttons.exit for click detection

    **draw.lua — PICK_SCENARIO screen:**
    - Add "[Q] Quit" hint below the "[L] Load Game" line, same gray style

    **input.lua — handle_sidebar_button():**
    - Add hit check for btns.exit during PLAYING mode
    - On first click: set a flag (e.g., `shared.exit_confirm = true`) to show confirmation
    - Don't immediately exit

    **input.lua — keypressed():**
    - In PLAYING mode: when `shared.exit_confirm` is true:
      - 'y' key: auto-save via `mods.save_mod.save_game(...)`, then reset to PICK_SCENARIO (reuse existing return-to-menu pattern from line ~527-539: campaign.active=false, campaign.roster=nil, game_over=false, winner_faction=-1, game_mode=PICK_SCENARIO, play menu music)
      - 'n' key: skip save, same reset to PICK_SCENARIO
      - 'escape' key: cancel exit (set shared.exit_confirm = false)
    - In PICK_SCENARIO mode: 'q' or 'escape' → love.event.quit()
    - Clear shared.exit_confirm on any mode transition (keypressed guard at top, or on scenario load)
