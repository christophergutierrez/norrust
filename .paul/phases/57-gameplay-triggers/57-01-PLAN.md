<objective>
## Goal
Extend dialogue triggers beyond turn-based events: `leader_attacked` fires the first time a leader unit is attacked, and `hex_entered` fires when a unit moves to a specific hex. Both use the existing one-shot semantics.

## Purpose
Allows campaign authors to attach narrator dialogue to gameplay moments, not just turn boundaries. Completes the v2.0 Dialogue System milestone.

## Output
- DialogueEntry gains optional `col`/`row` fields for hex-entry triggers
- `leader_attacked` trigger fires from client on first attack on a leader
- `hex_entered` trigger fires from client when a unit moves to a matching hex
- Sample entries added to crossing_dialogue.toml demonstrating both triggers
- New unit tests for the extended schema
</objective>
