<task type="auto">
  <name>Task 2: Fix death animation visibility</name>
  <files>norrust_love/main.lua, norrust_love/draw.lua</files>
  <action>
    The problem: `apply_attack()` removes dead units from engine state. `draw_units()` iterates `state.units` (which no longer includes the dead unit), and then cleans up any `unit_anims` entry not in `alive_ids`. So the death animation state is deleted immediately.

    Fix approach — track "dying" units with their last known position:

    1. In `main.lua`, add a `dying_units` table (keyed by uid) alongside `unit_anims`:
       ```lua
       local dying_units = {}  -- uid -> {def_id, col, row, faction, timer}
       ```

    2. In `apply_attack_with_anims()`, when a unit dies (line 296-298):
       - Before calling `trigger_death_anim()`, capture the dead unit's last position from the pre-attack state
       - Store in `dying_units[defender_id] = {def_id=..., col=..., row=..., faction=..., timer=1.0}`
       - The timer is how long to keep showing the death animation (1.0 seconds)

    3. In `love.update()`, tick down dying_units timers:
       - For each dying unit, decrement timer by dt
       - When timer <= 0, remove from `dying_units` and `unit_anims`

    4. In `draw.lua` `draw_units()`:
       - After the main unit loop, draw dying units from `ctx.dying_units`
       - For each dying unit, render using the same sprite draw path (get anim_state from ctx.unit_anims, draw at stored col/row position)
       - Change the cleanup loop: don't delete anim states for uids in `dying_units`

    5. Pass `dying_units` through the ctx table (same pattern as `unit_anims`).

    6. Clear `dying_units` on scenario load/reset (wherever `unit_anims` is cleared).
  </action>
  <verify>Run game, attack a unit to death — death animation should play visibly at the unit's position, then disappear after ~1 second</verify>
  <done>AC-2, AC-3 satisfied: death animation visible, cleaned up after</done>
</task>
