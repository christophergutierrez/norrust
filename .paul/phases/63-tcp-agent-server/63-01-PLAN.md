<objective>
## Goal
Add a TCP server to the Love2D client that exposes the game engine's JSON API to external agents. Create a Python client library for programmatic play.

## Purpose
Enables external Python agents to observe game state and submit actions over TCP, fulfilling the project's core value of "human players and AI agents using the same clean engine." This is the infrastructure for AI vs AI testing in Phase 64.

## Output
- agent_server.lua: non-blocking TCP server using LuaSocket in Love2D update loop
- Python client: tools/agent_client.py with get_state(), send_action(), ai_turn() methods
- Server toggle via command-line flag (--agent-server) or in-game key
</objective>


<task type="auto">
  <name>Task 1: Create agent_server.lua TCP server module</name>
  <files>norrust_love/agent_server.lua</files>
  <action>
    Create agent_server.lua using LuaSocket (available in Love2D as `require("socket")`):

    Protocol (line-based, newline-delimited):
    - Client sends: "get_state\n" → Server responds: JSON state + "\n"
    - Client sends: ActionRequest JSON + "\n" → Server responds: result code + "\n"
    - Client sends: "ai_turn FACTION\n" → Server responds: "0\n"
    - Client sends: "get_faction\n" → Server responds: active faction number + "\n"
    - Client sends: "check_winner\n" → Server responds: winner faction (-1 if none) + "\n"

    Implementation:
    - agent_server.new(port) — create server, bind to port, set non-blocking
    - agent_server.update(norrust, engine) — called from love.update, accept new clients, read commands, send responses
    - agent_server.stop() — close all connections and server socket
    - Non-blocking: settimeout(0) on server and client sockets
    - Handle multiple clients (store in array, remove on disconnect)
    - Buffer partial reads (TCP may split lines)
  </action>
  <verify>luajit -bl norrust_love/agent_server.lua /dev/null — syntax check</verify>
  <done>AC-1, AC-2, AC-3 satisfied (server side)</done>
</task>
