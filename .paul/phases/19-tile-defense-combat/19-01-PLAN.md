<objective>
## Goal
Wire `Tile.defense` into combat resolution as the middle tier of the terrain defense fallback
chain, replacing `Unit.default_defense` as the immediate fallback when a unit has no
terrain-specific defense entry.

## Purpose
`Tile.defense` is already populated from `TerrainDef` TOML at `set_terrain_at()` time but is
never consulted during combat. The current fallback — a fixed value on the unit — ignores
the actual terrain. Wiring `Tile.defense` makes terrain data authoritative for combat, the
last step needed before the battle mechanics contract is stable.

## Output
- `norrust_core/src/game_state.rs`: two defense lookups updated (attack + retaliation)
- New test `test_tile_defense_used_in_combat` in `game_state.rs`
</objective>

```gherkin
When the defender retaliates
Then the attacker (who has no tile set, default_defense = 0) takes full damage
  And the retaliator benefits from its tile defense
```

    Current:
    ```rust
    let terrain_defense = {
        let defender = state.units.get(&defender_id).ok_or(ActionError::UnitNotFound(defender_id))?;
        let terrain_id = state.board.terrain_at(defender_pos).unwrap_or("");
    ```

    ```rust
    let terrain_defense = {
        let defender = state.units.get(&defender_id).ok_or(ActionError::UnitNotFound(defender_id))?;
        let terrain_id = state.board.terrain_at(defender_pos).unwrap_or("");
        defender.defense.get(terrain_id).copied()
            .unwrap_or_else(|| {
                state.board.tile_at(defender_pos)

            })
    };
    ```

    ```rust
    let ret_defense = {
    ```
