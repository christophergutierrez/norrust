<objective>
## Goal
Create a config-driven debug data generator that produces a complete game data directory with patched values for rapid testing.

## Purpose
Eliminates the need to play long campaigns to test features like advancement. A single TOML config with defaults (e.g., `experience = 1`) generates a ready-to-play `debug/data/` directory. Foundational tool for all future testing.

## Output
- `debug/debug_config.toml` — declarative test config with sensible defaults
- `tools/generate_debug.py` — Python generator that reads config, copies real data, patches fields
- `debug/data/` — generated output (gitignored)
</objective>

<task type="auto">
  <name>Task 1: Create debug_config.toml schema and generate_debug.py</name>
  <files>debug/debug_config.toml, tools/generate_debug.py</files>
  <action>
    **debug/debug_config.toml:**
    ```toml
    [defaults]
    experience = 1          # XP needed to advance (normally 42-150)
    # max_hp = 999          # uncomment to make units unkillable
    # cost = 1              # uncomment for free recruitment
    # starting_gold = 99999 # uncomment for unlimited gold

    [defaults.faction]
    starting_gold = 99999   # override faction starting gold

    # Per-unit overrides (key = unit ID exactly as in TOML)
    # [overrides.units.Spearman]
    # experience = 50
    ```

    Fields not set in config → keep original value from real TOML.
    All fields are optional — an empty [defaults] section means "copy data as-is".

    **tools/generate_debug.py:**
    - stdlib-only Python (like generate_sprites.py pattern)
    - Reads debug/debug_config.toml (simple TOML parser — or use tomllib on Python 3.11+)
    - Walks data/units/ recursively, copying tree structure into debug/data/units/
    - For each unit TOML: parse, apply defaults (if field exists in config), apply per-unit overrides, write
    - Copy data/terrain/, data/factions/, data/recruit_groups/ into debug/data/
    - For faction TOMLs: if defaults.faction.starting_gold set, patch starting_gold field
    - Print summary: "Generated debug data: X units patched, Y factions, Z terrain types"

    **Key implementation details:**
    - Use simple line-by-line TOML patching (read lines, find `field = value`, replace value)
      NOT a full TOML parser — avoids mangling [[attacks]] arrays-of-tables
    - Line-by-line approach: for each patchable field, regex match `^field\s*=\s*` and replace the value
    - Non-matching lines pass through unchanged (preserves [[attacks]], [resistances], etc.)
    - Tree directory copying: use shutil.copytree for terrain/factions/recruit_groups (no patching needed)
    - Unit directory: walk with os.walk, create matching subdirs, patch each .toml file

    **Config parsing:**
    - Use tomllib (Python 3.11+) or fallback to simple key=value parser for [defaults] section
    - Config is simple flat key-value — no arrays-of-tables complexity
  </action>
