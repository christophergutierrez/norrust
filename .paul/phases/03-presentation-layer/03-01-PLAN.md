<objective>
## Goal

Extend the GDExtension bridge (`NorRustCore`) to own a live `GameState` and
expose all game actions — create, place, move, attack, end turn, query — as
GDScript-callable `#[func]` methods. Add `reachable_hexes()` flood-fill to
`pathfinding.rs` so GDScript can highlight valid move destinations.

## Purpose

Phase 3 is the visual layer on top of the headless core. Before any Redot
TileMap or sprite work can begin, GDScript must be able to drive the Rust
engine: create a game, place units, apply actions, and query resulting state.
This plan delivers that bridge, fully verified by `cargo test`, with no Redot
involvement yet.

## Output

- `norrust_core/src/pathfinding.rs` — `reachable_hexes()` flood-fill added
- `norrust_core/src/gdext_node.rs` — `NorRustCore` gains `game: Option<GameState>`
  and 10 new `#[func]` bridge methods
- All tests pass; `cargo clippy -- -D warnings` clean
</objective>

<task type="auto">
  <name>Task 1: Add reachable_hexes() flood fill to pathfinding.rs</name>
  <files>norrust_core/src/pathfinding.rs</files>
  <action>
    Add a new public function `reachable_hexes` that returns all hexes a unit
    can reach within its movement budget, using Dijkstra (not repeated A* calls):

    ```rust
    pub fn reachable_hexes(
        board: &Board,
        movement_costs: &HashMap<String, u32>,
        default_movement_cost: u32,
        start: Hex,
        movement_budget: u32,
        zoc_hexes: &HashSet<Hex>,
        is_skirmisher: bool,
    ) -> HashSet<Hex>
    ```

    Algorithm:
    - Min-heap BFS (same Dijkstra structure as find_path without a destination)
    - Expand all hexes whose accumulated cost ≤ movement_budget
    - Apply ZOC stop rule: if current ∈ zoc_hexes AND !is_skirmisher AND
      current != start — add current to reachable set, but do NOT expand neighbours
    - Always include `start` in the result (cost 0)
    - Use `#[allow(clippy::too_many_arguments)]` (same as find_path)

    Add a test in the existing `#[cfg(test)]` block:

    `test_reachable_hexes_respects_budget()`:
      - 5×5 board, all "grassland" (cost=1 per hex), no ZOC
      - start = Hex::from_offset(2, 2), budget = 2
      - Assert reachable.contains(Hex::from_offset(2, 2)) // own hex
      - Assert reachable.contains(Hex::from_offset(1, 2)) // 1 step
      - Assert !reachable.contains(Hex::from_offset(0, 2)) // 2+ steps (hex grid
        distance from (2,2) in cubic coords — verify the actual hex distance)
      - Assert reachable.len() >= 7 (center of 5×5 with budget 2 reaches many hexes)

    Note on hex distances: In the odd-r layout, adjacent hexes in the same row
    have cubic distance 1. Use Hex::distance() to verify expectations rather than
    hardcoding counts; assert on specific named hexes only.

    Avoid:
    - Calling find_path in a loop — use a single Dijkstra pass
    - Including hexes with impassable terrain (cost >= 99)
  </action>

    # Expect: all pathfinding tests pass (3 existing + 1 new = 4)
  </verify>

    ── ActionError → i32 mapping ───────────────────────────────────────────

    Define a private helper (free function, NOT a method):

      fn action_err_code(e: ActionError) -> i32 {
          match e {
              ActionError::UnitNotFound(_)         => -1,
              ActionError::NotYourTurn             => -2,
              ActionError::DestinationOutOfBounds  => -3,
              ActionError::DestinationOccupied     => -4,
              ActionError::UnitAlreadyMoved        => -5,
              ActionError::DestinationUnreachable  => -6,
          }
      }

    ── New #[func] methods ─────────────────────────────────────────────────

    All new methods go inside `#[godot_api] impl NorRustCore { ... }`.

    1. create_game(cols: i32, rows: i32, seed: i64) -> bool
       - Validate cols > 0 && rows > 0 && seed > 0; return false if invalid
       - Create Board::new(cols as usize, rows as usize)
       - Create GameState::new_seeded(board, seed as u64)
       - Store in self.game; return true

    2. set_terrain_at(col: i32, row: i32, terrain_id: GString)
       - If self.game is None, return
       - Call state.board.set_terrain(Hex::from_offset(col, row), terrain_id.to_string())

    3. place_unit_at(unit_id: i32, def_id: GString, hp: i32, faction: i32, col: i32, row: i32)
       - If self.game is None, return
       - Create Unit::new(unit_id as u32, def_id.to_string(), hp as u32, faction as u8)
       - Call state.place_unit(unit, Hex::from_offset(col, row))

    4. apply_move(unit_id: i32, col: i32, row: i32) -> i32
       - If self.game is None, return -1
       - Call apply_action(state, Action::Move { unit_id: unit_id as u32,
           destination: Hex::from_offset(col, row) })
       - Return 0 on Ok(()), action_err_code(e) on Err(e)

    5. apply_attack(attacker_id: i32, defender_id: i32) -> i32
       - Same pattern: apply_action with Action::Attack
       - Return 0 or error code

    6. end_turn() -> i32
       - apply_action(state, Action::EndTurn)
       - Return 0 or error code

    7. get_unit_positions() -> PackedInt32Array
       - If self.game is None, return empty array
       - For each (unit_id, hex) in state.positions:
           let (col, row) = hex.to_offset();
           Push [unit_id as i32, col, row] into a PackedInt32Array
       - Return the array

    8. get_active_faction() -> i32
       - Return self.game.as_ref().map(|s| s.active_faction as i32).unwrap_or(-1)

    9. get_turn() -> i32
       - Return self.game.as_ref().map(|s| s.turn as i32).unwrap_or(-1)

    10. get_reachable_hexes(unit_id: i32) -> PackedInt32Array
        - If self.game is None, return empty
        - Look up unit by unit_id as u32; if not found, return empty
        - Get unit's start position from state.positions
        - Get unit's movement budget and movement_costs
        - Call get_zoc_hexes(state, unit.faction)
        - Call reachable_hexes(board, costs, 1, start, budget, &zoc, false)
        - For each reachable hex, call hex.to_offset() and push [col, row]
        - Return the flat PackedInt32Array

    Hex::to_offset(): verify this method exists in hex.rs. If it does not, add it
    to hex.rs as the inverse of from_offset(), converting cubic back to odd-r
    offset. Odd-r formula:
      col = x + (z - (z & 1)) / 2
      row = z
    (This is the inverse of from_offset which computes x from col/row.)

    Avoid:
    - Panicking inside #[func] — use early returns on None/missing
    - Exposing Hex, GameState, or other Rust types to GDScript directly
    - Adding new GodotClass types — use only NorRustCore
  </action>
