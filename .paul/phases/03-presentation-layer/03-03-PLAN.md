<objective>
## Goal
Extend the bridge to populate unit stats from UnitDef at spawn, expose faction/HP query data,
then add unit rendering, click-to-select with reachable-hex highlighting, and click-to-move /
click-to-attack / end-turn dispatch to the GDScript game scene.

## Purpose
This is the interactive heart of Phase 3: the player can see their units, select them, and
issue orders. The Rust simulation validates every action; GDScript redraws the result. After
this plan the game is visually playable for one turn.

## Output
- `gdext_node.rs`: `place_unit_at()` copies stats from UnitDef; `get_unit_data()` added
- `game.gd`: units rendered as coloured circles with HP; selection + movement + attack +
  end-turn fully wired; reachable hexes highlighted in yellow
</objective>

## AC-1: Units Render with Faction Colour and HP
```gherkin
Given the game scene starts with 2 fighter units placed at (0,2) and (4,2)
When the scene is running
Then a blue filled circle with "30" HP text appears at (0,2)
  And a red filled circle with "30" HP text appears at (4,2)
```

## AC-2: Click-to-Select Shows Reachable Hexes
```gherkin
Given the game is running and it is faction 0's turn
When the player left-clicks the blue unit's hex at (0,2)
Then that hex gains a white outline ring
  And all hexes the fighter can reach (movement budget = 5, grassland cost = 1, forest cost = 2)
    are highlighted with a semi-transparent yellow overlay
```

## AC-3: Click Reachable Hex Moves Unit
```gherkin
Given unit at (0,2) is selected with reachable hexes highlighted
When the player clicks a highlighted hex at (e.g.) (1,2)
Then apply_move() is called and returns 0
  And the blue circle redraws at the new hex
  And the selection is cleared
  And console prints "Moved unit 1 to (1, 2)"
```

## AC-4: Click Enemy Unit Dispatches Attack
```gherkin
Given unit at (0,2) is selected (or a yellow hex near the red unit is reachable)
When the player clicks the red unit's hex at (4,2)
Then apply_attack() is called with attacker_id=1, defender_id=2
  And the defender's HP circle redraws with reduced HP (or disappears if dead)
  And console prints "Attack result: 0 (ok)" or the error code
```

## AC-5: 'E' Key Ends Turn
```gherkin
Given the game is running
When the player presses the 'E' key
Then end_turn() is called and returns 0
  And console prints "End turn: faction N, turn M"
  And the selection is cleared
```

<task type="auto">
  <name>Task 2: Selection, move/attack dispatch, end-turn keybind</name>
  <files>norrust_client/scripts/game.gd</files>
  <action>
    All changes are in `game.gd`. Do not touch Rust files.

    1. Add state variables at class level:
       ```gdscript
       var _selected_unit_id: int = -1
       var _reachable_cells: Array = []   # Array of Vector2i
       ```

    2. Add helper `_build_unit_pos_map() -> Dictionary`:
       - Call `_core.get_unit_data()`, iterate in steps of 5
       - Return dict mapping Vector2i(col, row) → [unit_id, faction]
         e.g. `{ Vector2i(0,2): [1, 0], Vector2i(4,2): [2, 1] }`

    3. Replace `_input()` with the following logic (keep the existing bounds check):
       ```
       on left mouse button press at (col, row) within board:
         pos_map = _build_unit_pos_map()
         active = _core.get_active_faction()

         if _selected_unit_id != -1 and Vector2i(col, row) in _reachable_cells:
           # Move selected unit to this hex
           result = _core.apply_move(_selected_unit_id, col, row)
           print("Moved unit %d to (%d, %d): code %d" % [_selected_unit_id, col, row, result])
           _clear_selection()
           queue_redraw()

         elif _selected_unit_id != -1 and Vector2i(col, row) in pos_map \
              and pos_map[Vector2i(col, row)][1] != active:
           # Attack enemy unit at this hex
           enemy_id = pos_map[Vector2i(col, row)][0]
           result = _core.apply_attack(_selected_unit_id, enemy_id)
           print("Attack result: %d" % result)
           _clear_selection()
           queue_redraw()

         elif Vector2i(col, row) in pos_map \
              and pos_map[Vector2i(col, row)][1] == active:
           # Select friendly unit
           uid = pos_map[Vector2i(col, row)][0]
           _selected_unit_id = uid
           raw = _core.get_reachable_hexes(uid)
           _reachable_cells = []
           for i in range(0, raw.size(), 2):
             _reachable_cells.append(Vector2i(raw[i], raw[i+1]))
           queue_redraw()

         else:
           _clear_selection()
           queue_redraw()

       on key press 'E' (InputEventKey, keycode == KEY_E, pressed):
         result = _core.end_turn()
         print("End turn: code %d, faction %d, turn %d" % [result, _core.get_active_faction(), _core.get_turn()])
         _clear_selection()
         queue_redraw()
       ```

    4. Add `_clear_selection()` helper:
       ```gdscript
       func _clear_selection() -> void:
         _selected_unit_id = -1
         _reachable_cells = []
       ```

    5. Add highlight drawing at the start of `_draw()`, AFTER terrain hexes, BEFORE _draw_units():
       ```gdscript
       # Yellow semi-transparent overlay on reachable hexes
       for cell in _reachable_cells:
         var center = _tile_map.map_to_local(cell) + _tile_map.position
         draw_polygon(_hex_polygon(center, HEX_RADIUS), [Color(1, 1, 0, 0.35)])

       # White outline on selected unit's hex
       if _selected_unit_id != -1:
         var pos_map = _build_unit_pos_map()
         for cell in pos_map:
           if pos_map[cell][0] == _selected_unit_id:
             var center = _tile_map.map_to_local(cell) + _tile_map.position
             var pts = _hex_polygon(center, HEX_RADIUS)
             pts.append(pts[0])   # close the polyline
             draw_polyline(pts, Color.WHITE, 2.5)
       ```

    Avoid: modifying _setup_rust_core(), _setup_tilemap(), _center_camera(), or
    the _hex_polygon() helper. Do not add Camera2D or UI nodes.
  </action>

  </verify>
