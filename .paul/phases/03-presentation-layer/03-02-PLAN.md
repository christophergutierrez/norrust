<objective>
## Goal

Create the main game scene with a hex TileMap driven by Rust state: a 5×5 board
of grassland and forest hexes drawn in Redot, with mouse clicks printing the
clicked hex's offset coordinates to the Output console.

## Purpose

This is the first visual proof that the Rust headless core and the Redot
presentation layer communicate correctly end-to-end. After this plan, a human
can see a coloured hex grid and click hexes to confirm the coordinate bridge
works before units and action dispatch are added in Plan 03-03.

## Output

- `norrust_client/scenes/game_scene.tscn` — new main scene (Node2D + script)
- `norrust_client/scripts/game.gd` — creates TileMap in code, draws map from
  Rust, handles mouse → hex coordinate logging
- `norrust_client/project.godot` — main scene updated to `game_scene.tscn`
</objective>

    ── game.gd ──────────────────────────────────────────────────────────────

    Implement the following structure:

    ```
    extends Node2D

    const TILE_SIZE  = 64          # hex cell bounding box (pixels)
    const BOARD_COLS = 5
    const BOARD_ROWS = 5

    var _core: NorRustCore
    var _tile_map: TileMap

    # TileSet source IDs assigned during _setup_tileset()
    var _src_grassland: int = -1
    var _src_forest:    int = -1

    func _ready():
        _setup_rust_core()
        _setup_tilemap()
        _draw_map()
        _center_camera()

    func _setup_rust_core():
        _core = NorRustCore.new()
        add_child(_core)

        var project_dir = ProjectSettings.globalize_path("res://")
        var data_path = project_dir + "/../data"
        _core.load_data(data_path)

        _core.create_game(BOARD_COLS, BOARD_ROWS, 42)

        # Mixed terrain: grassland everywhere, forest on odd col+row sum hexes
        for col in range(BOARD_COLS):
            for row in range(BOARD_ROWS):
                var terrain = "forest" if (col + row) % 2 == 1 else "grassland"
                _core.set_terrain_at(col, row, terrain)

    func _setup_tilemap():
        # Build TileSet programmatically
        var tile_set = TileSet.new()
        tile_set.tile_shape      = TileSet.TILE_SHAPE_HEXAGON
        tile_set.tile_layout     = TileSet.TILE_LAYOUT_STACKED
        tile_set.tile_offset_axis = TileSet.TILE_OFFSET_AXIS_VERTICAL
        tile_set.tile_size       = Vector2i(TILE_SIZE, TILE_SIZE)

        _src_grassland = _add_solid_tile(tile_set, Color(0.29, 0.49, 0.31))  # #4a7c4e
        _src_forest    = _add_solid_tile(tile_set, Color(0.18, 0.35, 0.15))  # #2d5927

        _tile_map = TileMap.new()
        _tile_map.tile_set = tile_set
        add_child(_tile_map)

    func _add_solid_tile(tile_set: TileSet, color: Color) -> int:
        var img = Image.create(TILE_SIZE, TILE_SIZE, false, Image.FORMAT_RGBA8)
        img.fill(color)
        var tex = ImageTexture.create_from_image(img)

        var source = TileSetAtlasSource.new()
        source.texture = tex
        source.texture_region_size = Vector2i(TILE_SIZE, TILE_SIZE)
        source.create_tile(Vector2i(0, 0))

        return tile_set.add_source(source)

    func _draw_map():
        for col in range(BOARD_COLS):
            for row in range(BOARD_ROWS):
                # terrain_id is the string we set in _setup_rust_core
                var is_forest = (col + row) % 2 == 1
                var src_id    = _src_forest if is_forest else _src_grassland
                _tile_map.set_cell(0, Vector2i(col, row), src_id, Vector2i(0, 0))

    func _center_camera():
        # Find the screen-space centre of the board and offset the TileMap
        var top_left   = _tile_map.map_to_local(Vector2i(0, 0))
        var bot_right  = _tile_map.map_to_local(Vector2i(BOARD_COLS - 1, BOARD_ROWS - 1))
        var board_centre = (top_left + bot_right) / 2.0
        var screen_centre = Vector2(
            ProjectSettings.get_setting("display/window/size/viewport_width"),
            ProjectSettings.get_setting("display/window/size/viewport_height")
        ) / 2.0
        _tile_map.position = screen_centre - board_centre
    ```

    Note: `_draw_map()` queries the Rust bridge indirectly — terrain was already
    set via `set_terrain_at()`, and the same (col+row)%2 logic determines the
    visual tile. In Plan 03-03, this will be replaced with a proper query to
    the Rust board state.

    Avoid:
    - Creating a TileMap via the scene file — do it in code to stay editor-agnostic
    - Using `get_node()` for NorRustCore — always `add_child()` after `new()`
    - Importing gdext types at the top of the script — Redot auto-registers them

    Add `_input(event)` to game.gd:

    ```gdscript
    func _input(event: InputEvent) -> void:
        if not event is InputEventMouseButton:
            return
        if not (event as InputEventMouseButton).pressed:
            return
        if (event as InputEventMouseButton).button_index != MOUSE_BUTTON_LEFT:
            return

        var local_pos  = _tile_map.to_local(get_global_mouse_position())
        var cell       = _tile_map.local_to_map(local_pos)
        var col        = cell.x
        var row        = cell.y

        if col >= 0 and col < BOARD_COLS and row >= 0 and row < BOARD_ROWS:
            print("Clicked hex: (%d, %d)" % [col, row])
        else:
            print("Clicked off-board: (%d, %d)" % [col, row])
    ```

    Avoid:
    - Using `_unhandled_input` — `_input` is sufficient and runs before UI nodes
    - Using Input.get_global_mouse_position() — `get_global_mouse_position()` on
      the Node2D is correct; convert to TileMap local before calling local_to_map()
