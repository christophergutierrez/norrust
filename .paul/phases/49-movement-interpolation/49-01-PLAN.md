<objective>
## Goal
Add smooth sliding animation when a unit move is committed, replacing the instant teleport with a visible hex-by-hex traversal along the A* path.

## Purpose
Movement interpolation makes the game feel alive — players see their units march across the board rather than teleporting. This uses the ghost_path data from Phase 48.

## Output
- Movement animation state machine in main.lua
- Position override rendering in draw.lua
- Input blocked during movement animation
- Follow-up actions (attacks) execute after animation completes
</objective>

<task type="auto">
  <name>Task 1: Movement animation state and interpolation</name>
  <files>norrust_love/main.lua, norrust_love/draw.lua</files>
  <action>
    **main.lua changes:**

    1. Add movement animation state variables:
       - `move_anim = nil` — table: {uid, path, seg, t, speed, on_complete}
         - uid: unit being animated
         - path: array of {col, row} waypoints
         - seg: current segment index (1-based, animating from path[seg] to path[seg+1])
         - t: 0.0 to 1.0 progress within current segment
         - speed: segments per second (tune for feel, ~8-12 segs/sec)
         - on_complete: callback function to run when animation finishes

    2. Create `start_move_anim(uid, path, on_complete)`:
       - Apply engine move immediately: `norrust.apply_move(engine, uid, dest_col, dest_row)`
       - Set `move_anim = {uid=uid, path=path, seg=1, t=0, speed=10, on_complete=on_complete}`
       - Keep unit selected (same as current commit_ghost_move)
       - Cancel ghost state

    3. Modify `commit_ghost_move()`:
       - If ghost_path has >= 2 waypoints, call `start_move_anim()` instead of direct apply
       - If ghost_path is empty/short (adjacent move), fall back to instant apply
       - The on_complete callback: restore normal state (clear move_anim, set selected/inspect)

    4. Update all 4 commit+attack sites (lines ~716, ~918, ~732, ~932):
       - For move+attack: on_complete callback runs `execute_attack()` then `clear_selection()`
       - For move-only (line 732, 932): on_complete is nil or just state cleanup

    5. In `love.update(dt)`, add movement animation tick (before camera logic):
       - If `move_anim` is nil, skip
       - Advance: `move_anim.t = move_anim.t + move_anim.speed * dt`
       - While `move_anim.t >= 1.0`: advance to next segment (`seg += 1, t -= 1.0`)
       - If `seg >= #path`: animation done → call on_complete, set `move_anim = nil`

    6. Block input during animation:
       - At top of `love.keypressed`: if `move_anim ~= nil then return end`
       - At top of mouse click handler: if `move_anim ~= nil then return end`

    **draw.lua changes:**

    7. In `draw_units()`, check if unit uid matches `move_anim.uid`:
       - If match: compute interpolated pixel position from move_anim path/seg/t
       - Override cx, cy with lerped position between path[seg] and path[seg+1]
       - Draw unit at interpolated position instead of state position

    8. Pass `move_anim` through ctx table (same pattern as ghost_path)

    **Tuning:**
    - Speed ~10 segments/sec means each hex takes ~0.1s
    - A 5-hex path takes ~0.5s — fast enough to not feel slow
    - Adjust if needed during human-verify
