<objective>
## Goal
Eliminate redundant Lua-side campaign state (`campaign.veterans`, `campaign.gold`, `campaign.index`) from the game-over flow. The Rust engine already owns all this state via CampaignState — Lua should call FFI and read results, not maintain shadow copies.

## Purpose
After Phase 119, `campaign_record_victory` is a single FFI call that syncs roster, extracts veterans, calculates gold, and advances the scenario index. But Lua still copies the result into `campaign.veterans`, `campaign.gold`, `campaign.index` and uses those copies to drive the next-scenario decision. This phase makes the game-over flow fully engine-driven: Lua calls the FFI, reads the status, and either loads the next scenario or returns to menu.

## Output
- `campaign_record_victory` returns a `status` field ("next_scenario" or "campaign_complete")
- `campaign_load_next_scenario` returns the `board` filename in its result
- `input_play.lua` game-over handler simplified: no redundant state copying, no fallback path
- `campaign_client.load_campaign_scenario` no longer depends on `campaign_index` from Lua
</objective>
