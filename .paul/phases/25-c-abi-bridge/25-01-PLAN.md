<objective>
## Goal
Create a C ABI bridge (`extern "C"` functions) that exposes all 32 existing GDExtension bridge methods via LuaJIT-callable functions. An opaque `NorRustEngine` struct holds game state, created/freed by the caller. All string returns use caller-frees pattern with `norrust_free_string()`.

## Purpose
Enables Love2D (LuaJIT FFI) to call into the Rust simulation core without GDExtension. This is the critical infrastructure for the Love2D migration — once this works, the client port is mechanical.

## Output
- `norrust_core/src/ffi.rs` — NorRustEngine struct + 32 `extern "C"` functions + memory helpers
- Updated `lib.rs` with `pub mod ffi;`
- Integration test verifying C ABI functions work end-to-end
</objective>

  <name>Task 1: Create ffi.rs with NorRustEngine and all C ABI functions</name>
  <files>norrust_core/src/ffi.rs, norrust_core/src/lib.rs</files>
  <action>
    Create `norrust_core/src/ffi.rs` with:

    **NorRustEngine struct** (mirrors NorRustCore fields without Godot deps):
    ```
    pub struct NorRustEngine {
        unit_registry: Option<Registry<UnitDef>>,
        terrain_registry: Option<Registry<TerrainDef>>,
        faction_registry: Option<Registry<FactionDef>>,
        recruit_group_registry: Option<Registry<RecruitGroup>>,
        faction_recruit_map: HashMap<String, Vec<String>>,
        state: Option<GameState>,
    }
    ```

    **Lifecycle functions:**
    - `norrust_new() -> *mut NorRustEngine`
    - `norrust_free(engine: *mut NorRustEngine)`
    - `norrust_free_string(s: *mut c_char)`
    - `norrust_free_int_array(arr: *mut i32)`

    **All 32 bridge functions as extern "C"**, following the pattern:
    - First param is always `engine: *mut NorRustEngine`
    - String params: `*const c_char` (converted via CStr)
    - String returns: `*mut c_char` (caller frees via norrust_free_string)
    - i32/i64/bool params: passed directly
    - PackedInt32Array equivalent: `norrust_get_reachable_hexes(engine, unit_id, out_len: *mut i32) -> *mut i32`

    **Function naming convention:** `norrust_` prefix + snake_case method name.
    Examples: `norrust_load_data`, `norrust_get_state_json`, `norrust_apply_move`, `norrust_ai_take_turn`

    **Implementation:** Each function dereferences the engine pointer, calls the same core logic as gdext_node.rs (game_state::apply_action, pathfinding::reachable_hexes, etc.). Use the same error code mapping.

    **Safety:** Each function checks for null engine pointer. Use `unsafe` blocks only at the FFI boundary. Interior logic is safe Rust.

    Add `pub mod ffi;` to lib.rs.

    Do NOT modify gdext_node.rs or any existing module.
    Do NOT add conditional compilation — both bridges coexist until Phase 27.
  </action>
  <verify>cargo build succeeds with ffi.rs included</verify>
  <done>AC-1, AC-4, AC-5 satisfied: engine lifecycle + string/array memory management</done>
</task>
