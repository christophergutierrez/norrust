<objective>
## Goal
Remove death pose from the sprite generation pipeline and derive death animation at render time. Instead of loading/generating a death.png sprite, dying units render their idle sprite with rotation (tilting backward) and fading alpha over the death timer. Remove death from viewer anim list. Clean up viewer to show portrait in all-anims view.

## Purpose
Death sprites are the lowest quality output from AI generation (unnatural poses, inconsistent style). Deriving death from idle (tilt + fade) is visually cleaner, eliminates one API call per unit, and removes existing death.png files that look bad.

## Output
- `draw_board.lua`: dying units drawn with idle sprite + rotation + fade
- `combat_mod.lua`: death animation trigger uses idle (no death anim state needed)
- `animation.lua`: death loading preserved (backward compat) but not required
- `viewer.lua`: death removed from anim_names list
- `generate_sprites_v2.py`: "death" removed from POSE_NAMES
</objective>
