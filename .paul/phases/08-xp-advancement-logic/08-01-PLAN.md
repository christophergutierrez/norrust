<objective>
## Goal
Award XP to both attacker and defender after every combat exchange, set
`advancement_pending` when the threshold is crossed, and verify with unit tests.

## Purpose
XP gain in the Attack action is the core of the advancement system. All other
advancement features (the Advance action, Phase 9 HUD) depend on `unit.xp`
accumulating correctly after each fight. This plan makes that happen in pure Rust
with no new dependencies and no registry access.

## Output
- `game_state.rs` `Action::Attack` branch: attacker and defender both gain XP
  after their respective strike phases; `advancement_pending` set on threshold crossing
- 4 new unit tests in `game_state.rs` covering XP gain, kill bonus, threshold trigger,
  and defender retaliation XP
</objective>

<acceptance_criteria>

## AC-1: Attacker Gains XP for Dealing Damage
```gherkin
Given an attacker with xp=0, xp_needed=40 and a defender with non-zero defense
When an Attack action resolves and the attacker deals damage > 0
Then attacker.xp increases by 1
```

## AC-2: Attacker Gains Kill Bonus XP
```gherkin
Given an attacker with xp=0, xp_needed=40 and a 1-HP defender with 0 terrain defense
When an Attack action kills the defender (damage >= defender hp)
Then attacker.xp increases by 8 (kill bonus only; no separate hit XP since defender dead)
```

## AC-3: advancement_pending Set at Threshold
```gherkin
Given an attacker with xp=39, xp_needed=40 and a killable defender (0 defense)
When an Attack action grants 1 XP for a hit
Then attacker.xp == 40 AND attacker.advancement_pending == true
```

## AC-4: Defender Gains XP from Retaliation
```gherkin
Given a defender with xp=0, xp_needed=40 that survives the attacker's strike
When the defender retaliates and deals damage > 0 to the attacker
Then defender.xp increases by 1
```

## AC-5: No XP for Zero Damage
```gherkin
Given a defender with 100% terrain defense (never hit)
When an Attack action resolves and the attacker deals 0 damage
Then attacker.xp remains 0
```

<task type="auto">
  <name>Task 1: Add XP grant logic to the Attack branch</name>
  <files>norrust_core/src/game_state.rs</files>
  <action>
    In the `Action::Attack` block of `apply_action`, after the existing attacker-strike
    damage application and defender kill-check, insert:

    ```rust
    // XP grant: attacker earns 1 for a hit, +8 bonus for a kill.
    // Award before checking defender survival so the kill bonus is clear.
    if state.units.contains_key(&attacker_id) {
        let xp_earned = if damage > 0 { 1 } else { 0 }
            + if defender_hp == 0 { 8 } else { 0 };
        if xp_earned > 0 {
            let a = state.units.get_mut(&attacker_id).unwrap();
            a.xp += xp_earned;
            if a.xp_needed > 0 && a.xp >= a.xp_needed {
                a.advancement_pending = true;
            }
        }
    }
    ```

    After the retaliation block (after defender strikes back and the attacker-kill
    check), insert the symmetric defender XP grant:

    ```rust
    // XP grant: defender earns 1 for a retaliation hit, +8 if they killed the attacker.
    if state.units.contains_key(&defender_id) {
        let def_xp = if ret_damage > 0 { 1 } else { 0 }
            + if !state.units.contains_key(&attacker_id) { 8 } else { 0 };
        if def_xp > 0 {
            let d = state.units.get_mut(&defender_id).unwrap();
            d.xp += def_xp;
            if d.xp_needed > 0 && d.xp >= d.xp_needed {
                d.advancement_pending = true;
            }
        }
    }
    ```

    Note: `damage` is the attacker-phase total, `ret_damage` is the defender-phase
    total — both are already in scope at the insertion points.

    Note: The attacker XP block must be placed AFTER `defender_hp` is bound and BEFORE
    the defender is removed (so `state.units.contains_key(&attacker_id)` is reliable
    for the defender XP block). Insert the attacker XP block immediately after the
    `if defender_hp == 0 { ... }` purge block, and the defender XP block immediately
    after the `if attacker_hp == 0 { ... }` purge block inside the retaliation `if let`
    closure.

    Avoid: Do not change the XP logic for the `ret_damage == 0` (miss) path — units
    that miss deal no XP. Do not call into the registry; use only the fields already
    on `Unit`.

    fn test_attacker_gains_xp_on_hit() {
        let board = Board::new(10, 10);
        let mut state = GameState::new_seeded(board, 42);
        // Attacker: guaranteed hit (100 damage, 1 strike), defender has 0 defense
        let attack = AttackDef {
            id: "sword".to_string(), name: "Sword".to_string(),
            damage: 1, strikes: 1,
            attack_type: "blade".to_string(), range: "melee".to_string(),
        };
        let mut attacker = Unit::new(1, "fighter", 30, 0);
        attacker.attacks = vec![attack];
        attacker.xp_needed = 40;
        let mut defender = Unit::new(2, "archer", 30, 1);
        defender.default_defense = 0; // always hit
        state.place_unit(attacker, Hex::ORIGIN);
        state.place_unit(defender, Hex::from_offset(1, 0));

        apply_action(&mut state, Action::Attack { attacker_id: 1, defender_id: 2 }).unwrap();

        assert!(state.units[&1].xp >= 1, "attacker should gain at least 1 XP for a hit");
    }

    #[test]
    fn test_attacker_gains_kill_bonus_xp() {
        let board = Board::new(10, 10);
        let mut state = GameState::new_seeded(board, 42);
        // 100-damage one-shot kill; defender has 0 defense
        let attack = AttackDef {
            id: "sword".to_string(), name: "Sword".to_string(),
            damage: 100, strikes: 1,
            attack_type: "blade".to_string(), range: "melee".to_string(),
        };
        let mut attacker = Unit::new(1, "fighter", 30, 0);
        attacker.attacks = vec![attack];
        attacker.xp_needed = 40;
        let mut defender = Unit::new(2, "archer", 5, 1);
        defender.default_defense = 0;
        state.place_unit(attacker, Hex::ORIGIN);
        state.place_unit(defender, Hex::from_offset(1, 0));

        apply_action(&mut state, Action::Attack { attacker_id: 1, defender_id: 2 }).unwrap();

        assert!(!state.units.contains_key(&2), "defender must be dead");
        assert_eq!(state.units[&1].xp, 9, "1 hit XP + 8 kill bonus = 9");
    }

    #[test]
