<objective>
## Goal
Wire per-terrain colors from TOML data through the `Tile`/`TileSnapshot` chain to GDScript rendering,
replacing the hardcoded terrain-to-color switch in `_draw()`.

## Purpose
Phase 15 added hills and mountains to the map but they render as the flat green fallback — the color
switch in `_draw()` only handles "village" and "forest". With 4+ terrain types now in play, hardcoded
constants don't scale. Moving color to TOML makes every terrain type visually distinct and eliminates
GDScript's knowledge of terrain-specific colors entirely.

## Output
- `schema.rs`: `color: String` field on `TerrainDef` (serde default = "")
- `board.rs`: `color: String` field on `Tile`; `Tile::new()` default "#808080"; `Tile::from_def()` copies color
- `snapshot.rs`: `color: String` on `TileSnapshot`; `from_game_state()` reads from `tile_at()`;
  new `test_tile_snapshot_includes_color` unit test
- `loader.rs`: `test_terrain_registry_loads` updated to assert `flat.color == "#4a7c4e"`
- All 14 terrain TOMLs: `color = "..."` field added
- `game.gd`: tile color map built from `state["terrain"]`; hardcoded terrain-to-color if-else removed;
  `COLOR_FOREST` and `COLOR_VILLAGE` constants removed; `COLOR_FLAT` kept as fallback only
</objective>

    ## game.gd — replace hardcoded terrain-to-color switch with data-driven lookup

    In the constants section at the top, remove `COLOR_FOREST` and `COLOR_VILLAGE`,
    keeping only `COLOR_FLAT` as the fallback:

    Replace:
    ```gdscript
    const COLOR_FLAT      = Color(0.29, 0.49, 0.31)  # #4a7c4e
    const COLOR_FOREST    = Color(0.18, 0.35, 0.15)  # #2d5927
    const COLOR_VILLAGE   = Color(0.72, 0.60, 0.25)  # gold-tan
    ```

    With:
    ```gdscript
    const COLOR_FLAT = Color(0.29, 0.49, 0.31)  # #4a7c4e — fallback for hexes without color data
    ```

    In `_draw()`, replace the terrain query and hardcoded color switch:

    Remove:
    ```gdscript
    # 1. Terrain hexes
    for col in range(BOARD_COLS):
        for row in range(BOARD_ROWS):
            var center     = _tile_map.map_to_local(Vector2i(col, row)) + _tile_map.position
            var terrain_id = _core.get_terrain_at(col, row)
            var color: Color
            if terrain_id == "village":
                color = COLOR_VILLAGE
            elif terrain_id == "forest":
                color = COLOR_FOREST
            else:
                color = COLOR_FLAT
            draw_polygon(_hex_polygon(center, HEX_RADIUS), [color])
    ```

    Replace with:
    ```gdscript
    # Build per-hex color map from state snapshot (colors defined in terrain TOMLs)
    var tile_colors: Dictionary = {}
    for tile in state.get("terrain", []):
        var c: String = tile.get("color", "")
        tile_colors[Vector2i(int(tile["col"]), int(tile["row"]))] = \
            Color.html(c) if c != "" else COLOR_FLAT

    # 1. Terrain hexes
    for col in range(BOARD_COLS):
        for row in range(BOARD_ROWS):
            var center = _tile_map.map_to_local(Vector2i(col, row)) + _tile_map.position
            var color  = tile_colors.get(Vector2i(col, row), COLOR_FLAT)
            draw_polygon(_hex_polygon(center, HEX_RADIUS), [color])
    ```

    Note: `tile_colors` must be built BEFORE the terrain hex loop.
    The `state` variable is already parsed at the top of `_draw()` — no change needed there.
