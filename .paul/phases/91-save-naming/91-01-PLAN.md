<objective>
## Goal
Add display_name field to save files and a rename UI in the save management screen. Players can label saves with custom names (e.g., "Just before the major battle"). The save list shows display_name when present, falling back to auto-generated metadata.

## Purpose
Players managing multiple campaign saves need meaningful labels to distinguish them. Filenames are machine-generated timestamps — display_name provides human-readable identification.

## Output
- `display_name` field in save TOML [game] section
- Rename prompt in LOAD_SAVE screen (R key)
- Text input via love.textinput callback
- Save list shows display_name when available
</objective>

<task type="auto">
  <name>Task 1: display_name field in save/load and list</name>
  <files>norrust_love/save.lua</files>
  <action>
    **write_save:** Add `display_name = ""` to the data.game table (empty string default).

    **list_saves:** In the header parsing loop, extract display_name from [game] section:
    ```
    local dn = line:match('^display_name%s*=%s*"([^"]*)"')
    if dn then display_name = dn end
    ```
    Add `display_name` field to each save entry in the returned array.

    **New function save.update_display_name(filepath, name):**
    - Read entire save file text
    - If `display_name = "..."` line exists in [game] section, replace it
    - If not, insert `display_name = "name"` after the [game] header line
    - Write updated text back to file
    - This avoids full re-parse; just a targeted string replacement
  </action>
  <verify>luajit -bl save.lua /dev/null — no syntax errors</verify>
  <done>AC-1 satisfied: display_name field serialized in saves</done>
</task>

  <files>norrust_love/main.lua, norrust_love/draw.lua, norrust_love/input.lua</files>
  <action>

    - Add `save_renaming = false` and `save_rename_text = ""` to game_data table

    - Pass save_renaming and save_rename_text to draw ctx

    **draw.lua:**
    - In the LOAD_SAVE branch, modify save entry display:
      - If save has display_name (non-empty string), show it as primary label, with date/scenario as secondary dimmer text

    - When ctx.save_renaming is true, draw a text input bar at the bottom:
      - Dark background rectangle
      - "Rename: " label + ctx.save_rename_text + blinking cursor
      - "[Enter] Confirm  [Esc] Cancel" hint

    **input.lua:**
    - In LOAD_SAVE handler, add 'r' key:
      - Set game_data.save_renaming = true
      - Set game_data.save_rename_text to current display_name (or "")
    - When save_renaming is true, intercept keys before normal LOAD_SAVE handling:
      - 'return': call save.update_display_name(), set save_renaming = false, rescan list
      - 'escape': set save_renaming = false (cancel)
      - 'backspace': remove last character from save_rename_text
      - All other keys: ignore (text input comes via love.textinput)
  </action>
