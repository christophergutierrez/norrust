## Goal
Add vision_range to UnitDef/Unit and implement compute_visibility() returning HashSet<Hex> of visible hexes per faction. Range-based (not line-of-sight).

## Purpose
Foundation for fog of war — the engine must know which hexes each faction can see before FFI can filter snapshots or Lua can render fog.

## Output
- `visibility.rs` module with `compute_visibility(state, faction) -> HashSet<Hex>`
- `vision_range` field on UnitDef and Unit
- Unit tests proving visibility calculation is correct
</objective>

<task type="auto">
  <name>Task 2: Implement compute_visibility module</name>
  <files>norrust_core/src/visibility.rs, norrust_core/src/lib.rs</files>
  <action>
    Create norrust_core/src/visibility.rs:

    ```
    pub fn compute_visibility(state: &GameState, faction: u8) -> HashSet<Hex>
    ```

    Algorithm:
    1. Collect all units belonging to `faction`
    2. For each unit, determine effective vision range: if unit.vision_range > 0 use it, else use unit.movement
    3. Get the unit's position from state.positions
    4. For each hex on the board (iterate board.tiles keys or iterate all col/row within board dimensions), check if hex.distance(unit_pos) <= effective_vision_range
    5. If yes, insert hex into result HashSet

    Optimization note: iterate board hexes once, check against all faction units. For each hex, if ANY friendly unit is within range, it's visible. This avoids generating all hexes within range of each unit (which could include off-board hexes).

    Add `pub mod visibility;` to lib.rs.

    Add unit tests in visibility.rs:
    - test_basic_visibility: single unit on small board, verify exact visible hexes
    - test_vision_range_fallback: unit with vision_range=0 uses movement
    - test_multiple_units_union: two units combine their vision
    - test_enemy_not_counted: faction 1 units don't contribute to faction 0 visibility
  </action>
  <verify>cargo test --lib -p norrust_core -- visibility --skip balance 2>&1 | tail -10</verify>
  <done>AC-3 and AC-4 satisfied: compute_visibility returns correct HashSet with fallback logic</done>
</task>
