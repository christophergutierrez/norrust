<objective>
## Goal
Click any hex to see terrain info (type, defense %, movement cost, healing) in a sidebar panel. When a unit is selected, show that unit's specific terrain interaction on the clicked hex.

## Purpose
Foundation for tactical decision-making. Players need to understand terrain before the ghost movement and combat preview phases can be useful. Currently there is no way to see terrain stats — only the colored hex tiles.

## Output
- TileSnapshot extended with defense, movement_cost, healing fields
- New FFI function: norrust_get_unit_terrain_info() for unit-specific terrain stats
- New draw_terrain_panel() in main.lua showing terrain info in sidebar
- Click-hex interaction: empty hex shows terrain panel, unit click still shows unit panel
</objective>

## AC-1: TileSnapshot Includes Terrain Stats
```gherkin
Given a game with terrain loaded from scenario TOML
When norrust_get_state_json() is called
Then each tile in the terrain array includes defense (u32), movement_cost (u32), and healing (u32) fields
```

## AC-2: Unit-Terrain Info FFI
```gherkin
Given a unit on the board and a target hex with known terrain
When norrust_get_unit_terrain_info(engine, unit_id, col, row) is called
Then it returns JSON with the unit's effective defense % and movement cost on that terrain
And the defense fallback chain is: unit.defense[terrain_id] → tile.defense → unit.default_defense
```

## AC-3: Terrain Panel on Hex Click
```gherkin
Given the game is in PLAYING mode and no unit is selected
When the player clicks an empty hex on the board
Then a sidebar panel shows: terrain type name, defense %, movement cost, healing (if > 0)
```

## AC-4: Unit-Specific Terrain Info
```gherkin
Given a friendly unit is selected (selected_unit_id != -1)
When the player clicks an empty reachable hex
Then the terrain panel also shows the selected unit's specific defense % and movement cost on that terrain
And these may differ from the base terrain values
```

## AC-5: Panel Priority
```gherkin
Given the game is in PLAYING mode
When the player clicks a hex containing a unit
Then the unit panel is shown (existing behavior unchanged)
And the terrain panel is NOT shown simultaneously
When the player clicks an empty hex
Then the terrain panel is shown and the unit panel is cleared
```

  <name>Task 1: Extend TileSnapshot and add unit-terrain FFI</name>
  <files>norrust_core/src/snapshot.rs, norrust_core/src/ffi.rs, norrust_core/src/lib.rs</files>
  <action>
    1. In snapshot.rs, add three fields to TileSnapshot:
       - defense: u32
       - movement_cost: u32
       - healing: u32
       Populate them from tile_at() in StateSnapshot::from_game_state().

    2. In ffi.rs, add norrust_get_unit_terrain_info(engine, unit_id, col, row) -> *mut c_char:
       - Looks up the unit by ID in state.units
       - Looks up the tile at (col, row) via Hex::from_offset
       - Computes effective defense: unit.defense.get(terrain_id) → tile.defense → unit.default_defense
       - Computes effective movement cost: unit.movement_costs.get(terrain_id) → tile.movement_cost
       - Returns JSON: {"defense": N, "movement_cost": N, "terrain_id": "...", "base_defense": N, "base_movement_cost": N, "healing": N}
       - Returns null on invalid unit_id or hex

    3. Add unit test: test_tile_snapshot_includes_stats (verify defense/movement_cost/healing in snapshot)
    4. Add unit test: test_unit_terrain_info (verify unit-specific defense differs from base)
  </action>
  <verify>cargo test --workspace — all existing + new tests pass</verify>
  <done>AC-1 and AC-2 satisfied</done>
