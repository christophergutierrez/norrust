<objective>
## Goal
Add per-faction gold tracking to GameState. On EndTurn, the newly-active faction earns
2 gold per village it owns. Starting gold is 10 per faction. StateSnapshot exposes gold.
HUD displays the active faction's current gold.

## Purpose
Gold is the prerequisite for recruitment in Phase 21. This phase wires the income side of
the economy — village_owners already tracks ownership, so EndTurn income is a small addition.
Exposing gold in StateSnapshot gives AI clients the same data as GDScript.

## Output
- `norrust_core/src/game_state.rs`: `GameState.gold` field + EndTurn income + test
- `norrust_core/src/snapshot.rs`: `StateSnapshot.gold` field
- `norrust_client/scripts/game.gd`: HUD gold display
</objective>

## AC-1: Starting gold is 10 per faction
```gherkin
Given a freshly created GameState
When querying gold for faction 0 or faction 1
Then both are 10
```

## AC-2: Owned villages pay 2 gold per turn to their owner
```gherkin
Given faction 0 owns 1 village and faction 1 owns 0 villages
When EndTurn is applied (faction 0 ends turn, faction 1 becomes active)
Then faction 1's gold is unchanged (10)
When EndTurn is applied again (faction 1 ends turn, faction 0 becomes active)
Then faction 0's gold is 12 (10 + 2 for 1 village)
```

## AC-3: Gold exposed in StateSnapshot JSON
```gherkin
Given a GameState with gold [10, 10]
When get_state_json() is called
Then the JSON contains "gold":[10,10]
```

## AC-4: HUD shows active faction's gold
```gherkin
Given the game is running in PLAYING mode
When _draw() renders the HUD
Then the HUD text includes the active faction's gold amount (e.g. "10g")
```

    **2. Initialize gold in GameState::new() and new_seeded():**
    Both constructors initialize all fields. Add `gold: [10, 10]` to both.
    (Search for the struct literal that initializes `village_owners: HashMap::new()`)

    **3. Add village income in Action::EndTurn:**
    After the faction flip (`state.active_faction = 1 - state.active_faction;`),
    before the unit reset loop, insert:
    ```rust
    // Village income: newly-active faction earns 2 gold per owned village
    let active_i8 = state.active_faction as i8;
    let income = state.village_owners.values()
        .filter(|&&owner| owner == active_i8)
        .count() as u32 * 2;
    state.gold[state.active_faction as usize] += income;
    ```

    **4. Add test `test_village_income_adds_gold`:**
    ```rust
    #[test]
    fn test_village_income_adds_gold() {
        use crate::board::Tile;
        let board = Board::new(10, 10);
        let mut state = GameState::new(board);

        // Verify starting gold
        assert_eq!(state.gold, [10, 10]);

        // Give faction 0 ownership of one village directly
        let village_hex = Hex::from_offset(3, 3);
        state.village_owners.insert(village_hex, 0);

        // faction 0 ends turn → faction 1 becomes active → faction 1 gets income (0 villages = 0g)
        apply_action(&mut state, Action::EndTurn).unwrap();
        assert_eq!(state.active_faction, 1);
        assert_eq!(state.gold[1], 10, "faction 1 owns no villages, gold unchanged");

        // faction 1 ends turn → faction 0 becomes active → faction 0 gets income (1 village = 2g)
        apply_action(&mut state, Action::EndTurn).unwrap();
        assert_eq!(state.active_faction, 0);
        assert_eq!(state.gold[0], 12, "faction 0 owns 1 village: 10 + 2 = 12");
    }
    ```
  </action>
  <verify>cargo test test_village_income_adds_gold -- --nocapture 2>&1</verify>
  <done>AC-1 + AC-2 satisfied: starting gold 10, village pays 2 per owned village per turn</done>
