## Goal
Implement basic save/load for a single scenario: save unit positions, gold, turn, and board reference to a TOML file. Load reconstructs the same board state via existing FFI calls.

## Purpose
Foundation for persistent game state. Testable immediately: place units, move them, save, restart, load — units should be in correct positions.

- save.lua: TOML writer + save/load functions
- 2 new Rust FFI functions: set_turn, set_active_faction
- F5 to save, F9 to load
- Save files in Love2D save directory (~/.local/share/love/norrust/saves/)
</objective>

<task type="auto">
  <name>Task 2: save.lua — TOML writer + save/load functions</name>
  <files>norrust_love/save.lua, norrust_love/conf.lua</files>
  <action>
    Set Love2D identity in conf.lua: `t.identity = "norrust"` so save directory is ~/.local/share/love/norrust/

    Create norrust_love/save.lua with:

    1. `serialize_toml(data)` — converts a Lua table to TOML string:
       - [section] for simple key-value groups
       - [[array]] for arrays-of-tables (units)
       - String values quoted, numbers bare, booleans bare
       - Only needs to handle the save file structure (not generic TOML)

    2. `save.write_save(engine, norrust, scenario_board, scenarios_path)`:
       - Gets current state via norrust.get_state(engine)
       - Builds save table: {game = {board_path, turn, active_faction, gold_0, gold_1}, units = [...]}
       - Each unit: {id, def_id, faction, col, row}
       - Serializes to TOML string
       - Creates "saves/" directory via love.filesystem.createDirectory
       - Writes to saves/YYYY-MM-DD_HHMMSS_scenario.toml via love.filesystem.write
       - Returns filename on success, nil on failure

    3. `save.load_save(engine, norrust, filepath)`:
       - Reads TOML file via love.filesystem.read
       - Parses with toml_parser (extend if needed for [[units]] arrays-of-tables)
       - Calls norrust.load_board(engine, board_path, 42) to set up terrain
       - Calls norrust.place_unit_at for each unit
       - Calls norrust.set_faction_gold for each faction
       - Calls norrust.set_turn and norrust.set_active_faction
       - Returns parsed data on success, nil on failure

    NOTE: toml_parser.lua currently does NOT support [[arrays-of-tables]]. Either:
    (a) Extend toml_parser to handle [[section]] syntax, or
    (b) Use a simpler format where units are indexed: [unit_0], [unit_1], etc.
    Choose (a) — extend the parser — to stay consistent with scenario TOML files.
  </action>
  <verify>luajit -bl norrust_love/save.lua /dev/null succeeds</verify>
  <done>AC-1, AC-3 satisfied: save writes TOML, load reads it</done>
</task>
