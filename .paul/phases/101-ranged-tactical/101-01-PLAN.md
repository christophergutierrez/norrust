<objective>
## Goal
Add three tactical behaviors to per-unit decision making: (1) ranged units prefer attacking from distance 2 over melee range, (2) focus fire bonus for attacking wounded enemies, (3) badly wounded units retreat toward healing terrain instead of marching toward enemies.

## Purpose
The 1-ply lookahead picks the best evaluate_state score, but it misses tactical nuances: ranged units sometimes move adjacent when they could attack from range (no retaliation), the AI spreads damage across enemies instead of focusing kills, and wounded units march to their death instead of retreating to heal. These three bonuses steer plan_unit_action toward smarter choices.

## Output
Modified `plan_unit_action` in `norrust_core/src/ai.rs` with tactical scoring bonuses and wounded retreat logic.
</objective>

  <action>
    Modify the attack scoring loop in `plan_unit_action` to add two tactical bonuses to the evaluate_state score:

    1. **Ranged distance bonus**: After computing `score = evaluate_state(&sim, faction)`, check if the unit attacked from distance 2 using a ranged attack. If so, add a small bonus (e.g., `+2.0`) to the score. This is a tie-breaker: when two positions produce similar evaluate_state results, prefer the ranged position because it avoids melee retaliation on the opponent's turn (which 1-ply doesn't see).

    2. **Focus fire bonus**: After computing `score`, check the target enemy's HP ratio before the attack: `enemy_hp / enemy_max_hp`. Add a bonus of `(1.0 - hp_ratio) * 5.0` — this gives up to 5.0 bonus for attacking enemies at 0% HP (already dead shouldn't happen), ~2.5 for 50% HP enemies, 0 for full HP enemies. This steers the AI toward finishing wounded enemies.

    Both bonuses are added to the evaluate_state score BEFORE the `if score > best_score` comparison.

    Note: Read the enemy HP BEFORE the clone+simulate (from original state), since after simulation the enemy may be dead and we can't check its HP ratio.
  </action>

<task type="auto">
  <name>Task 2: Wounded unit retreat toward healing</name>
  <files>norrust_core/src/ai.rs</files>
  <action>
    Modify the move-only fallback in `plan_unit_action` to consider healing retreat for wounded units:

    1. Define a retreat threshold: `const RETREAT_HP_RATIO: f32 = 0.30;`

    2. In the `if !has_any_attack` block, before the march-toward-enemy fallback:
       a. Check if the unit's HP ratio (`unit.hp as f32 / unit.max_hp.max(1) as f32`) is below RETREAT_HP_RATIO
       b. If wounded: find the nearest reachable hex with healing > 0 (via `state.board.tile_at(hex).map(|t| t.healing > 0)`)
       c. If a healing hex is reachable: move there instead of marching toward enemies
       d. If no healing hex reachable but wounded: still march toward nearest healing hex (closest by distance among all board hexes with healing > 0, pick the reachable hex that minimizes distance to it)
       e. If not wounded: use existing march-toward-enemy logic (unchanged)

    3. Also apply retreat logic even when attacks ARE available: if the unit is below RETREAT_HP_RATIO AND no attack in the candidates would kill an enemy (i.e., all post-attack simulations still have the target alive), prefer retreating over attacking. Check by looking at whether any simulated attack results in the target being removed from sim.units. If no kill is possible, override the attack decision with retreat.

    Note: A wounded unit that CAN secure a kill should still attack (the kill is worth the risk). Only retreat when no kill is available.
  </action>
