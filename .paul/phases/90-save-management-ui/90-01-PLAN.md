<objective>
## Goal
Add a save management screen accessible from the main menu. Lists all save files reverse-chronological with metadata (campaign, scenario, turn, date). Player can select a save to load or delete it.

## Purpose
Currently F9 silently loads the most recent save with no UI. Players need to see and choose from their saves, especially when managing multiple campaigns or playthroughs.

## Output
- New game mode `LOAD_SAVE` with save list UI
- Save file scanning with metadata extraction from TOML headers
- Keyboard navigation: up/down to select, Enter to load, D to delete, Escape to go back
- Integrates with existing load_save() flow
</objective>

  <name>Task 1: Save scanning and metadata extraction</name>
  <files>norrust_love/save.lua</files>
  <action>
    Add `save.list_saves()` function that:
    - Calls `love.filesystem.getDirectoryItems("saves")` to enumerate .toml files
    - Sorts reverse-alphabetically (date-first naming = reverse-chronological)
    - For each file, reads only the [game] and [campaign] sections (first ~20 lines) using `love.filesystem.read()`
    - Extracts: board_path (→ scenario name), turn, campaign_file (if present)
    - Parses date/time from filename (`YYYY-MM-DD_HHMMSS_scenario.toml`)
    - Returns array of {filename, date_str, scenario, turn, campaign, filepath}

    Add `save.delete_save(filepath)` that calls `love.filesystem.remove(filepath)`.

    Keep parse_save_toml internal — list_saves should do lightweight header parsing only (read first 20 lines, extract key=value pairs for [game] and [campaign] sections).
  </action>
  <verify>Run `love . --generate-tiles` to syntax-check. Verify save.list_saves returns correct data by adding a print statement temporarily.</verify>

    - Enter in LOAD_SAVE: load selected save using existing F9 logic (extract into helper or inline), transition to PLAYING
    - 'd' in LOAD_SAVE: call save.delete_save() on selected filepath, rescan list, adjust idx
